# card

The sanctioned home for **card chrome** — the `rounded + border + bg + padding`
shell that ~9 bespoke `*-card.tsx` files each hand-rolled. `<Card>` renders that
shell once and bakes the Ctrl+A **select-scope** into its root element, so any
card automatically scopes selection to itself (a click into the card + Ctrl+A
selects only the card, not the surrounding pane) with zero per-consumer wiring.

- Polymorphic `as` (`div` | `button` | `a` | `li`, default `div`); `ref`
  forwards to the root DOM element (DnD consumers depend on it).
- `interactive` adds the hover affordance; `selected` adds the emphasis border.
- Default chrome is `rounded-md border border-border bg-card p-3`; pass
  `className` to override (cn uses `tailwind-merge`, so `rounded-lg` /
  `bg-muted/30` / a different padding cleanly replace the defaults).

The `no-adhoc-card` lint rule (in `lint/`) fingerprints ad-hoc card markup
(rounded + border + the `bg-card` surface token + padding on an intrinsic
`div`/`button`/`a`/`span`) and redirects it here. It keys on `bg-card`
specifically — the dedicated card-surface token — not the broader `bg-muted`/
`bg-background` that panels, rows, and chips use without being cards. Escape via
`<Card>`, the `p-card` token, or
`// eslint-disable-next-line card/no-adhoc-card -- <reason>`.
