# scroll-reveal

The sanctioned owner of "reveal an element on activation". A reveal fires on an
activation **transition** (`isActive` going false‚Üítrue while mounted) or on
explicit intent (`revealOnMount`, or the imperative `revealElement`) ‚Äî **never
because an element remounted already-active**. That distinction is the whole
point: background data churn (a live-state push remounting the selected row)
must never move the user's scroll.

- `useRevealOnActive(isActive, opts?)` ‚Äî returns a callback ref. Reveals on the
  false‚Üítrue transition. `revealOnMount` (a boolean, or a lazily-consumed
  one-shot function) opts into a single reveal when the element mounts
  already-active (deep-link / new-tab cases).
- `revealElement(el, opts?)` ‚Äî the imperative funnel for genuine event handlers
  and effects with a bespoke re-fire key. The only sanctioned wrapper over
  `el.scrollIntoView`; the `no-adhoc-scroll-into-view` lint rule routes every
  call site here.

Extend this primitive rather than hand-rolling `scrollIntoView` again.
