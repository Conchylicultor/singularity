# icon-button

`IconButton` is the curated standalone-icon-action path: it composes `Button`
(`aspect="icon"`) + `Tooltip`, injecting the mandatory `aria-label` + tooltip and
rendering a bare `<Icon/>`. Its height is ambient (set by the region's
`ControlSizeProvider` / slot `controlSize`) — it never sizes itself.

## Enforcement

`lint/prefer-icon-button.ts` (contributed by this plugin, registered repo-wide)
steers a standalone `<Button aspect="icon"><MdX/></Button>` toward
`<IconButton icon={…} label=… />`. It fires NARROWLY — only when the `<Button>`
has `aspect="icon"`, its only child (ignoring whitespace) is a single
react-icons glyph element, and it is NOT a render-target prop value
(`trigger={<Button…/>}` / `render={<Button…/>}`). This is the exact shape an
author hand-rolls when they skip `IconButton` and lose the `aria-label`. Bare
`<Button aspect="icon">` stays valid for triggers / text-glyph buttons /
stateful-indicator children; a justified one-off (e.g. a per-model glyph size
`IconButton` can't express) carries a per-site
`// eslint-disable-next-line icon-button/prefer-icon-button -- <reason>`.
