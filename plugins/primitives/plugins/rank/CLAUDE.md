# rank

## DO NOT use floats or integers for ordered lists

Use this plugin. `fractional-indexing` strings ("a0", "V0m", "Zz9‚Ä¶") can always
insert between any two neighbors, sort correctly under the `rank_text` PostgreSQL
domain (C collation = byte order), and never need a full-list rewrite.

## When to use this vs tree's `computeDrop`

- **`computeDrop` (`@plugins/primitives/plugins/tree/shared`)** ‚Äî client-side DnD:
  computes the rank for an item *moved between two existing neighbors*. Call it in
  `onDragEnd` / `onDrop` handlers.

- **`nextRankIn` / `nextRankUnder` (this plugin, server barrel)** ‚Äî server-side
  *inserts*: generates the rank for a new item appended at the end of a list.
  Both accept an optional `executor` (a `tx` from `db.transaction`) so they
  compose safely inside transactions.

```ts

// Rank.from(str)           ‚Äî wrap a raw string (e.g. from HTTP body)

// Shared / client ‚Äî Zod schema that accepts string | Rank and always outputs Rank
import { RankSchema } from "@plugins/primitives/plugins/rank/shared";
```

## Anti-patterns

- `rank: integer` / `rank: float` ‚Äî can't insert between adjacent integers; floats lose precision
- `ORDER BY createdAt` as a proxy for user order
- `MAX(rank) + 1` ‚Äî the integer trap
- Calling `generateKeyBetween` directly ‚Äî use `Rank.between()` instead
- Sorting with `.localeCompare()` ‚Äî TypeScript now prevents it; `Rank` has no such method
- Comparing `Rank` objects with `===` ‚Äî use `Rank.equals(a, b)`
