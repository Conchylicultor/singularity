# latest-ref

`useLatestRef(value)` returns a stable `useRef`-backed ref whose identity never
changes; `useEventCallback(fn)` returns a permanently-stable function. Read
`.current` only in callbacks / effects, never during render.

## Registered as known-stable for exhaustive-deps

Both `useLatestRef` and `useEventCallback` are listed in the `stableHooks`
setting of [`build-lint-config.ts`](../../../framework/plugins/tooling/plugins/lint/core/build-lint-config.ts)
(`settings["react-hooks"].stableHooks`), which our patch to
`eslint-plugin-react-hooks` (`patches/eslint-plugin-react-hooks@*.patch`) feeds
into `exhaustive-deps`. The returned ref / callback is therefore treated exactly
like a bare `useRef` return: **do NOT list it in a `useCallback`/`useMemo`/`useEffect`
dependency array** — it is neither required nor flagged. (A separate cleanup
warning can still fire when an effect's *cleanup* reads `someRef.current`; that is
stability-independent and is suppressed at the site with an inline disable.)
