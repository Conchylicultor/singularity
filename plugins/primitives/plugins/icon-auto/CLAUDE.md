# icon-auto

The `icon-auto` slot-icon sizing convention: icons rendered inside a text-embedded
slot (Badge / Row / LinkChip / ToggleChip / Breadcrumb `leading`+`actions`, etc.)
should track the slot's font-size instead of carrying a hardcoded `size-N`.

## The utility

The `@utility icon-auto { width: 1.15em; height: 1.15em; }` lives in
`plugins/framework/plugins/web-core/web/theme/app.css` (per the theme rule that
all `@utility`/token CSS lives in the web-core theme folder). `1.15em` reads as
slightly larger than cap height — next to `text-xs` (12px) it renders ~13.8px,
matching the dominant `size-3.5` (14px) better than a bare `1em`. The name is
registered in `web-core/web/theme/control-utilities.ts` (its twMerge conflict
group), kept in sync by the `app-css-utilities-in-sync` check.

This plugin owns only the **lint rule + convention** — the CSS and the twMerge
config stay in web-core, mirroring how `control-size` owns its lint rule while the
`control-*` utilities live in `app.css`.

## Enforcement — `lint/no-adhoc-slot-icon-size.ts`

Fires only when a **bare inline glyph** is passed to the `icon=`/`leading=` slot
of an **auto-sizing primitive** and carries a hardcoded `size-*`/`h-*`/`w-*`
class. ALL of these must hold:

1. Attribute name is `icon` or `leading`.
2. The value is a `JSXExpressionContainer` wrapping an inline `JSXElement`.
3. The element owning the attribute is one of the auto-sizing primitives —
   `{ Badge, Row, LinkChip, ToggleChip, Breadcrumb }` — matched by opening-element
   identifier name. **Keep `AUTO_SIZING_PARENTS` in the rule in sync** with the
   primitives whose slot containers apply the `icon-auto` utility.
4. The slotted element is a bare glyph: no children, and its tag is `svg` or a
   Capitalized component (`MdX`/`Icon`/…) — never a lowercase host (`span`/`div`).
5. Its (simple-literal) `className` carries a hardcoded size token.

```tsx
// flagged — bare glyph in an auto-sizing slot
<Row icon={<MdDelete className="size-4" />} />
<LinkChip leading={<svg className="h-4 w-4" />} />
// clean — slot auto-sizes it
<Row icon={<MdDelete />} />
```

Report-only (no autofix): removing the class vs. keeping a deliberate override is
a human call.

### Honest scope — a convention aid, NOT a guarantee

The rule is deliberately conservative to keep ZERO false positives, at the cost of
accepted false negatives. It catches the **bare-inline-glyph-in-an-auto-sizing-
slot** pattern (the bulk of real call sites) and nothing else:

- **Parent allow-list, not prop name.** It does NOT fire on custom wrappers like
  `<ToolButton icon={...} />` or `<BulletList icon={...} />` — those do not wire
  `icon-auto`. An aliased import of a real primitive (e.g. `RowPrimitive` for
  `Row`) is matched by tag name only, so it is NOT caught — accepted false
  negative.
- **Bare glyph only.** It does NOT fire on layout-box/spacer wrappers
  (`<span className="size-5" />`, `<span className="size-3"><MdCheck/></span>`) or
  a `<Badge>` used as a slot label — these have children or a host tag, where the
  hardcoded size is intentional.
- It does NOT catch `const ico = <X className="size-3" />; icon={ico}` — no
  variable tracing.
- It does NOT catch sizes hardcoded INSIDE a component the slot renders.
- The `actions=` slot is out of scope (often `<Button>`s with their own glyphs).
- A non-literal / dynamic `className` (opaque member access, etc.) is skipped.

Treat it as a nudge toward the convention, not a complete AST guarantee.
