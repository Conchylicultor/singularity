# surface-id

A leaf primitive holding the **stable per-surface-instance id** — the tab's
`tabId` — as a React context (`SurfaceIdContext`) plus a reader hook
(`useSurfaceTabId`). It answers "which surface am I rendered in?" without
touching `window.location`, which is wrong the moment two surfaces are visible
at once (desktop multi-window / keep-alive tabs).

It is its own leaf (rather than part of `pane`) for a boundary reason: the
graph already has `pane → icon-button → shortcuts`, so letting the `shortcuts`
primitive read the surface id by importing `pane` would close an import cycle.
Both `pane` (which provides the value via `PaneSurfaceProvider`) and consumers
like `shortcuts` and `scoped-store` import this leaf instead.

The value is supplied by `PaneSurfaceProvider` (in `pane`), which is mounted
once per surface by `TabSurface`.
