# custom-columns

User-defined custom columns for any DataView. Two stores:

- **Definitions** (per-surface schema `{ id, label, type }[]`) live in **config_v2**
  via `customColumnsExtraFields`, merged by the data-view host into view-core's
  `viewsDescriptor` next to `sortPresets`. Git-promotable, per-app-scopable,
  reactive — zero new registration machinery (mirrors `sortPresetsExtraFields`).
- **Values** (per-row user data keyed by `(dataViewId, rowKey, columnId)`) live in
  the `data_view_custom_values` DB table, surfaced by a push-mode param live
  resource (recomputed by the L4 DB change-feed on every write) + a single
  upsert/delete-on-empty endpoint.

The data-view host (the parent) owns the bridge that composes both into ordinary
`FieldDef[]` and appends them to `props.fields`, so custom columns flow through
every view + sort/filter/search for free. This child NEVER imports
`data-view`/`view-core` (that would form a cycle) — the host threads the resolved
config descriptor down into `useCustomColumnDefs(descriptor)`.
