# gallery

The **gallery** view child of the `data-view` primitive. Contributes one
`DataViewSlots.View("gallery")` entry: a responsive card grid rendered through
the shared `FieldDef` schema.

## What it renders

- **Default card** (free): a field-driven `<DataCard>` — title from the first
  `text` field (or `fields[0]`), the cover field as media, and the remaining
  fields as muted property rows. Wired to `onRowActivate`.
- **Custom card** (compose / replace): when `options.renderCard` is set, the
  gallery renders it as-is. A custom card **owns its own click handling** — it
  is not wrapped in `DataCard` and `onRowActivate` is not wired.

## Options

`options` (= `viewOptions.gallery`) is a `GalleryViewOptions<TRow>`:

- `renderCard?(row)` — compose a `<DataCard>` or fully replace the card.
- `coverField?` — override which field id is the cover (else the `FieldDef`
  with `cover: true`, else the first `media` field).
- `minCardWidth?` — grid min card width in px (default 200).

## Exports

- `DataCard` — composable card chrome with `media` / body / `actions` /
  `footer` regions, `group`-driven hover reveal, focus, and click→`onActivate`.
- `GalleryViewOptions` — the typed options interface.
- `galleryOptions<TRow>(o)` — builds a typed `["gallery", o]` entry for
  `viewOptions`.
