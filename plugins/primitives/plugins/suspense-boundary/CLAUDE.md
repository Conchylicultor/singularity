# suspense-boundary

Generic React Suspense primitive that wraps every slot contribution, mirroring
[`error-boundary`](../error-boundary/CLAUDE.md). When a contribution suspends —
most commonly because a child called `useConfig` (which suspends until config is
loaded rather than flashing default values) — this renders a small local spinner
in that slot instead of letting the thrown promise bubble up and blank a larger
region of the app.

It registers a `registerSlotItemMiddleware` at priority `200`, higher than the
error boundary's `100`. Middlewares apply highest-priority-innermost, so the
Suspense boundary sits *inside* the error boundary: the error boundary stays
outermost and still catches genuine render errors thrown from the suspended tree.

There is also an app-level `<Suspense>` in `web-core`'s `App.tsx` that covers
root consumers (e.g. the theme injector) not rendered through a slot.
