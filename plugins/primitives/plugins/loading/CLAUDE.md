# loading

The single home for the LOADING state. An orchestrator over existing leaves ‚Äî
nothing visual is re-implemented:

```tsx
<Loading />                               // variant="text" (default) ‚Üí <Placeholder>Loading‚Ä¶</Placeholder>
<Loading variant="spinner" label="‚Ä¶" />  // <Spinner/> + optional label, for small/inline areas
<Loading variant="rows" count={6} />     // skeleton list rows
<Loading variant="cards" count={8} />    // skeleton card grid (matches the gallery layout)
<Loading variant="block" className="h-4 w-32" /> // single shimmer block (the atom)
```

## Delay-before-show (why this exists)

Live-state resources usually settle over the WS in <100ms. Every `Loading`
variant mounts **invisible** and fades in only after ~120ms (pure CSS, see
`web/internal/loading.css` ‚Äî no JS timers): on the common warm path the data
arrives first and the node unmounts before it ever paints, so fast loads show
**no transient UI at all**. The skeleton/fade only appears on genuinely slow
loads (WS down ‚Üí HTTP fallback). Hand-rolled `Loading‚Ä¶` text can't do this ‚Äî
which is what the `no-adhoc-loading-text` lint rule enforces.

## Division of labor

- **`Loading`** ‚Äî the loading state (this plugin).
- **`Placeholder`** ‚Äî plain muted text for EMPTY and ERROR states (and the
  delegate for `variant="text"`).
- **`Spinner`** ‚Äî bare spinning icon for *in-progress actions* (inside a
  button); `variant="spinner"` composes it for loading areas.
- **DataView** ‚Äî pass `loading` and the active view renders its own skeleton
  shape; `emptyState` requires confirmed-empty.
- For gating on resource readiness (what to render *around* a Loading), see
  `<ResourceView>` / `matchResource` / `combineResources` in
  `plugins/primitives/plugins/live-state/CLAUDE.md`.

## Lint rules contributed

- `no-adhoc-loading-text` ‚Äî bans hand-rolled `Loading‚Ä¶` JSX text outside
  `<Loading>`/`<Placeholder>`. BURNDOWN allowlist in `lint/index.ts` ‚Äî
  migrate entries, never add.
- `no-shadcn-skeleton` ‚Äî the shadcn `components/ui/skeleton.tsx` was deleted
  (subsumed by `variant="block"`); this keeps the import path from being
  resurrected by a shadcn re-generation.
