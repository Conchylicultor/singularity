# icon-picker

Searchable, categorized icon picker over the full Material Design icon set
(react-icons/md). Owns the icon registry, the `SvgNode` storage format, and the
server-side SVG resolution. The `avatar` primitive composes it.

## Components

- `<IconPicker value onSelect />` â€” the icon block: a header, a search box, and a
  scrollable, categorized grid. Loads the ~2 000-icon registry lazily on first
  mount, so mount it only when visible (e.g. inside an open popover). It renders
  *only* the icon UI â€” surrounding chrome (popover, extra rows) is the caller's
  job. `onSelect` fires with `{ key, svgNodes }`: the MD icon key plus its
  extracted SVG child-tree, ready to store and render.

## The `SvgNode` storage format

`SvgNode` (in `core/`) is the JSON-serializable shape of an icon's `<svg>`
child-tree (`{ tag, attr, child[] }`). Storing it in a DB column lets any
consumer render the icon as raw `<svg>` with **no icon-module import** â€” the
2 000-icon `react-icons/md` bundle never ships to surfaces that only display.

- `extractSvgNodes(Icon)` (web) â€” pull the tree out of a react-icons component
  (used internally by `IconPicker`, exported for callers that have an
  `IconType` in hand).
- `resolveIconSvgNodes(key)` / `resolveIconSvgNodesJson(key)` (server) â€” look up
  the tree for a stored icon key against the prebuilt `ICON_SVG_MAP`, for
  backfills and resolvers that only have the key.

## The generated SVG map

`server/internal/icon-svg-map.generated.ts` is a static
`Record<key, SvgNode[]>` covering every MD icon. It is generated by
`scripts/gen-icon-svg-map.ts` and kept in sync by the `icon-svg-map-in-sync`
check (which hashes `icon-metadata.json` + the react-icons version). Regenerate
with `bun run plugins/primitives/plugins/icon-picker/scripts/gen-icon-svg-map.ts`
when either input changes.

## Why a separate primitive

The icon registry, search, SVG-extraction, and server resolution are generic â€”
any surface that lets a user pick an icon (avatars, page icons, app icons) needs
them. Keeping them here means `avatar` only adds its own concerns (the colored
disc and the color palette) on top, and future pickers reuse the same block.
