# radius

The corner-radius standard: every rounded corner reads from one shape token so
a Shape preset can re-soften the whole app together, plus the lint rule that
keeps it the single source of truth.

## The scale

Corner radius is **themeable** shape, so it routes through the `--radius` shape
token (owned by the `tokens/shape` token group). The token-driven `rounded-*`
scale ‚Äî `rounded-sm`, `rounded-md`, `rounded-lg`, `rounded-xl`, ‚Ä¶ ‚Äî resolves to
`--radius`-derived values, so picking a Shape preset (Sharp / Rounded / Pill)
rescales every corner at once.

Two shapes bypass the token and freeze a corner at a literal value:

- **bare `rounded`** ‚Äî Tailwind's static `0.25rem`, which does NOT read
  `--radius`. This was the `Row` primitive bug: list rows stayed at 0.25rem
  under every preset. Use `rounded-md` (or the step that fits).
- **arbitrary `rounded-[‚Ä¶]`** (e.g. `rounded-[2px]`) ‚Äî a hand-pinned literal.

`rounded-none`, `rounded-full`, and `rounded-checkbox` are intentional,
shape-preserving corners (a hard corner, a pill/circle, a small square) and are
always allowed.

`rounded-checkbox` is the sanctioned square corner for checkbox-class indicators:
`min(var(--radius), 3px)`. It **caps** the shape token at 3px rather than freezing
a flat literal ‚Äî so a sharp / 0-radius theme still yields a hard-cornered checkbox
(it adapts downward), but a generous Shape preset can never round the box past 3px
into a near-circle (which made multi-select look mutually-exclusive). This is
actually *more* theme-aware than shadcn's own checkbox, which hardcodes a flat
`rounded-[4px]`. It is owned as a custom `@utility` in ui-kit's `app.css`
(registered in `custom-utilities.ts`).
Prefer the `CheckboxIndicator` / `RadioIndicator` primitive
(`@plugins/primitives/plugins/selection-indicator/web`), which bakes in the
correct shape, over reaching for the class directly ‚Äî and never re-derive a square
corner with an arbitrary `rounded-[‚Ä¶]` literal.

## Enforcement

`lint/no-adhoc-radius.ts` fails `./singularity check` on any `className` (or
`cn`/`clsx` argument) carrying a bare `rounded` or an arbitrary `rounded-[‚Ä¶]`.
Reach for a token-driven scale step instead so Shape presets keep working.

The rule enforces repo-wide with no `ignores` allowlist ‚Äî every corner routes
through the token scale. The legacy offenders were all migrated; a genuinely
fixed literal escapes per-site via the disable comment below, not a central list.

Escape hatch (rare ‚Äî a genuinely fixed literal shape):

```ts
// eslint-disable-next-line radius/no-adhoc-radius -- reason
```
