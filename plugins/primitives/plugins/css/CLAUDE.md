# css

Umbrella for the composable layout primitives ‚Äî the `<Frame>` / `<Grid>` /
`<Cluster>` / `<Center>` / `<Overlay>` sub-plugins ‚Äî plus the lint gate that
keeps raw layout CSS out of feature code. Layout is "CSS-in-semantics": you
write the role (`leading` / `content` / a `gallery` grid / a `cluster` of
chips), the primitive owns the `flex ‚Ä¶ min-w-0 shrink-0` mechanics, so the
overlap/clip bug class becomes structurally unrepresentable.

## Enforcement

`lint/no-adhoc-layout.ts` fails `./singularity check` on any class-name carrying
a raw Tailwind layout utility ‚Äî flow/display (`flex*`, `grid*`, `basis-*`,
`col-span-*`, `row-span-*`), space-sharing (`shrink-*`, `grow-*`, `min-w-0`),
alignment (`items-*`, `justify-*`, `place-*`, `self-*`), positioning
(`absolute`, `fixed`, `sticky`, `inset-*`), and clipping (`overflow-*`). It does
**not** touch positioning *context* (`relative`/`static`), sizing
(`w-*`/`h-*`/`size-*`/`min-w-*` other than the `min-w-0` footgun), or non-flow
display (`block`/`hidden`/`inline`); spacing and `z-*` are owned by their own
rules. Compose layout through the primitives instead ‚Äî `<Frame>` for rows,
`<Grid>`/`<Cluster>`/`<Center>`/`<Overlay>` for the other modes, `<Stack>` /
`<Inset>` for 1-D flow, `<TruncatingText>` for the `min-w-0` truncation leaf.

The `ignores` allowlist in `lint/index.ts` has two tiers: **permanent** globs for
the layout primitives themselves (they own the raw mechanics) and a **burndown**
list of pre-rule offenders, grandfathered so the gate lands as `error` for new
code without a repo-wide rewrite. The burndown drains to 0 over time (the
`no-adhoc-spacing` playbook); do not add entries back. A genuinely-fixed one-off
escapes per-site via `// eslint-disable-next-line layout/no-adhoc-layout -- reason`.

## Plugin reference

- Description: Umbrella for global CSS layout primitives (named-slot rows, grids, clusters, overlays) with the shrink hierarchy baked into one place.
- Sub-plugins:
  - **`center`** ‚Äî Centering layout primitive: <Center axis> centers its content on one or both axes via a grid place-items box.
  - **`cluster`** ‚Äî Wrap-friendly chip group layout primitive: <Cluster> lays out a wrapping row of rigid identity chips that never individually shrink, delegating to Stack.
  - **`control-size`** ‚Äî Control-size standard: the shared control-* height scale and its enforcing lint rule (no-adhoc-control).
  - **`frame`** ‚Äî Named-slot row layout primitive: <Frame leading content meta trailing> lays four role slots on a CSS grid with the shrink hierarchy baked in ‚Äî rigid clusters never crush, content truncates last, meta truncates first. Callers write roles, never min-w-0/shrink-0/flex-1 mechanics.
  - **`grid`** ‚Äî Responsive/uniform grid layout primitive: <Grid minCellWidth> lays out a wrapping, equal-width card grid via a closed prop surface ‚Äî not a raw grid-template passthrough.
  - **`icon-auto`** ‚Äî icon-auto slot-icon sizing convention: the icon-auto @utility (em-based, in app.css) plus the no-adhoc-slot-icon-size lint rule.
  - **`overlay`** ‚Äî In-flow positioning layout primitive: <Overlay behind above clickThrough> paints full-bleed layers under/over its content within its own box, plus the click-through-toggle idiom.
  - **`radius`** ‚Äî Corner-radius standard: the token-driven rounded-* scale and its enforcing lint rule (no-adhoc-radius).
  - **`z-layers`** ‚Äî Semantic z-layer scale (z-base..z-max) and its enforcing lint rule (no-adhoc-zindex).
