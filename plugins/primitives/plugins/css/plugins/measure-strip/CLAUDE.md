# measure-strip

The sanctioned home for an **off-screen measurement strip** — a hidden flex row
used to measure children's natural widths before an overflow/collapse decision.
`<MeasureStrip>` self-portals to `document.body` and parks the row off-screen at
`-9999/-9999` with `opacity: 0` + `pointer-events: none`, so it reports true
natural widths without ever affecting layout.

## Why this primitive exists

Several width-driven UIs (the app tab bar, pane toolbars, `ResponsiveOverflow`)
need to know how wide their children *want* to be before deciding how many fit.
The idiom — render every child once in a hidden, body-portaled flex row and read
back `el.children` widths — was copy-pasted byte-identical across three sites
(only `gap` varied). Routing them all through one primitive removes the
duplication and the chance of one copy drifting.

## API

```tsx
<MeasureStrip ref={measureRef} gap={4} enabled={count > 0}>
  {children}
</MeasureStrip>
```

- **`ref`** — the consumer/hook OWNS the ref; the strip never creates it. Read
  the measured widths via `ref.current.children`.
- **`gap`** — gap between items, in px (match the rendered strip's gap).
- **`enabled`** — gates the portal (pass e.g. `count > 0`). Default `true`.
- **`children`** — rendered as-is, with **no implicit wrapping**. A consumer that
  needs one DOM node per item (for per-item width readout) wraps each child
  itself.
