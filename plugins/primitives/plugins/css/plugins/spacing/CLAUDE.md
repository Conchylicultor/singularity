# spacing

The layout-spacing standard: one closed set of gap/padding roles for every
spacing decision, the `<Stack>` / `<Inset>` primitives that consume them, and
the lint rule that keeps them the single source of truth.

## The scale

Spacing is **runtime-themeable** (it tightens with the density preset), so the
8-step ramp lives in the `density` token group as `--space-{2xs,xs,sm,md,lg,xl,2xl}`
runtime vars, exposed as the `gap-<step>` / `p-<step>` `@utility` families in
`plugins/primitives/plugins/css/plugins/ui-kit/web/theme/app.css`. `none` is the constant 0.

| Step  | Comfortable | Replaces (raw) |
| ----- | ----------- | -------------- |
| `none`| 0           | `*-0`          |
| `2xs` | 0.125rem    | `*-0.5`        |
| `xs`  | 0.25rem     | `*-1`          |
| `sm`  | 0.5rem      | `*-2`          |
| `md`  | 0.75rem     | `*-3`          |
| `lg`  | 1rem        | `*-4`          |
| `xl`  | 1.5rem      | `*-6`          |
| `2xl` | 2rem        | `*-8`          |

The values seed 1:1 with today's dominant raw usage at Comfortable, but diverge
at Cozy/Compact — so gaps and insets finally scale together with controls and
pads instead of staying fixed.

## The primitives

- **`<Stack direction gap align justify wrap as>`** — flex container with a gap
  from the ramp. Replaces `flex flex-col gap-*` and `space-y-*`. `direction`
  defaults to `col`; `gap` is required.
- **`<Inset pad x y t r b l as>`** — padding container. Replaces `p-*`/`px-*`/`py-*`.
  Props compose general→specific (`pad` then axis then side).

Both compose caller `className` last, so layout overrides win. Use the named
`gap-<step>` / `p-<step>` utilities directly for grids or third-party components
you don't want to wrap.

### Reaching the ramp from a `className`

- **`insetClass({pad,x,y,t,r,b,l})`** — the same resolver `<Inset>` uses (same
  records, same general→specific order), returning a class string. Use it when
  you can only pass a `className` and cannot wrap the element: Lexical's
  `<ContentEditable>`, `<Text>`, any third-party `className` prop. Use `<Inset>`
  whenever you own the element.

`insetClass` also exists so a step can be a **variable**. Tailwind emits an
`@utility` only when its literal token is found by the source scanner, so
`` `pl-${STEP}` `` compiles to nothing scannable and works only by accident
(some other file happens to spell `pl-md`). The step→class records live inside
this plugin as literals; call sites pass the step, never build the class.

## Enforcement

`lint/no-adhoc-spacing.ts` fails `./singularity check` on any class-name carrying
a raw Tailwind spacing utility — numeric (`gap-2`, `px-3`, `mt-4`, `space-y-2`)
or arbitrary (`gap-[7px]`, `p-[5px]`) — across gap, padding, margin, and
space-between. The named `*-<step>` utilities and `mx-auto`/`my-auto` are
allowed. Margins should usually be restructured into a Stack gap or Inset pad
rather than re-expressed as a named margin (there are intentionally no named
margin utilities).

A large BURNDOWN allowlist in `lint/index.ts` grandfathers files that pre-date
the rule; it drains to empty over time. A genuinely-fixed one-off escapes
per-site via `// eslint-disable-next-line spacing/no-adhoc-spacing -- reason`.
