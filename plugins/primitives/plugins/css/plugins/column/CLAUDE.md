# column

The named-slot **column** primitive ‚Äî the column twin of `<Frame>`. A vertical
stack of up to three **role slots** ‚Äî `header` / `body` / `footer` ‚Äî passed as
props (no `children`). Where `Frame` is the named-slot *row*, `Column` is the
named-slot *column*: it bakes the `rigid | flexible | rigid` fill policy into one
place so callers write **roles, never mechanics**.

## Why flex (not grid)

`Frame` uses CSS grid because the *row* shrink hierarchy (content truncates last,
meta first) needs track sizing. `Column`'s requirement is simpler and is the
textbook flex-column: a rigid header, a growing+scrolling body, a rigid footer ‚Äî
which is exactly what `<Scroll>`'s `fill` (`min-h-0 flex-1 overflow-y-auto`) was
built for. Flex keeps the body's `Scroll` composition **honest**: `fill` is
semantically correct in a flex column, whereas in a grid cell its `flex-1` would
be inert.

So `Column` owns `flex flex-col` + the rigid (`shrink-0`) header/footer wrappers,
and **delegates the scroll body to the existing `<Scroll>` primitive** ‚Äî
composition, exactly as `Frame` delegates truncation to `<TruncatingText>`.
`Scroll` stays the single owner of `overflow`; the `min-h-0` / `flex-1` fill pair
stays inside `Scroll` (or, with `scrollBody={false}`, on the plain flexible body
wrapper).

## The fill policy

- **`header`** ‚Üí `shrink-0` wrapper ‚Äî rigid, never crushed.
- **`body`** ‚Üí the single flexible region. By default wrapped in
  `<Scroll axis="y" fill>` so it absorbs the column's slack and scrolls.
- **`footer`** ‚Üí `shrink-0` wrapper ‚Äî rigid, never crushed.

Only present slots render ‚Äî an absent slot produces no region and no phantom gap.

## Props

- **`header`** ‚Äî rigid top region (`shrink-0`).
- **`body`** ‚Äî flexible middle region; scrolls by default.
- **`footer`** ‚Äî rigid bottom region (`shrink-0`).
- **`scrollBody`** ‚Äî wrap `body` in a `<Scroll axis="y" fill>` cell. Default
  `true`. Set `false` for a plain flexible region (`min-h-0 flex-1`) that manages
  its own overflow (e.g. an embedded `data-view`).
- **`fill`** ‚Äî emit the flex-child fill pair (`min-h-0 flex-1`) on the root so the
  column fills its own flex-col parent. Default `false`. Needed when `Column` is
  itself a flex child whose height is externally clamped.
- **`gap`** ‚Äî gap role from the spacing ramp between regions. Default `none`
  (regions are usually flush).
- **`as`** ‚Äî host element. Default `div`. **`className`** composes last;
  **`ref`** forwarded.
