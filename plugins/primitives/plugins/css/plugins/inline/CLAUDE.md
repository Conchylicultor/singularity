# inline

The inline-level flow row — an `inline-flex` row that sits **inline in a text run**,
baseline-aligned, flowing with the surrounding text instead of breaking the line. The
inline-level sibling of `<Stack>` (which is block-level `flex`).

Reach for `<Inline>` whenever a chip, icon, or small affordance must appear mid-text or
inside another element's text flow (a path chip in a transcript row, a lone status icon in a
sidebar row). Block-level `Stack`/`Frame`/`Cluster` would force a line break; `Inline` does
not.

## How it works

Delegates to `<Stack direction="row">` (same as `Cluster`) so the gap ramp and
align/justify semantics stay in one place. It overrides only the display:

- `inline-flex` overrides Stack's `flex` (tailwind-merge resolves the `display` group —
  later class wins).
- `align-baseline` (`vertical-align: baseline`) seats the box on the surrounding text
  baseline — the recipe `Badge` already uses for inline-in-text chips.

Carries **no `min-w-0`**: constrain the box with `max-w-full` in `className`; the truncation
leaf inside (a `<Text>` in a line container) owns `min-w-0`. This preserves the "exactly one
primitive owns `min-w-0`" invariant.

Single-line context: a non-wrapping `<Inline>` (the default — a row that is line-ish, not a
flow region) **inherits** the surrounding `SingleLine` contract rather than resetting it, so
chips inside a line container stay single-line. An `<Inline wrap>` is a flow region and resets
(its `Stack` delegate handles that). Because a non-wrapping `Inline` keeps a chip GROUP on one
line, it's also the fix for a multi-contribution chip slot (`<Inline><Slot.Render/></Inline>`).

## Props

- **`gap`** — spacing-ramp step (`none`…`2xl`). Required, mirrors `Stack`.
- **`align`** — cross-axis alignment. Default `center`.
- **`justify`** — main-axis distribution.
- **`wrap`** — allow children to wrap. **`as`** — host element, default `span`.
  **`className`** composes last; **`ref`** forwarded.
