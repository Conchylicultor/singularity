# cluster

The wrap-friendly chip group. A horizontal row of **rigid identity chips/tags**
that wraps to the next line when it runs out of width — the home for badge/tag
rows, filter-chip strips, and any "group of whole chips" layout.

## What Cluster adds over a bare `Stack`

`<Cluster>` delegates to `<Stack direction="row" wrap>` internally — it does
**not** reimplement flex, so the gap ramp and align/justify semantics stay
defined in exactly one place. The distinct export buys two things:

- **Intent.** The contract is that children are identity chips — rigid, wrapping,
  **never individually shrinking**. Cluster imposes no `min-w-0` anywhere
  (truncation is a leaf's altitude, never a chip group's), so a chip keeps its
  whole width and the group reflows by wrapping, never by crushing a chip.
- **A future policy home.** Because every chip group routes through this one
  primitive, Cluster can later compose `ResponsiveOverflow` (progressively hide
  chips that don't fit, with a "+N more") with **zero call-site change**.

## Props

- **`gap`** — spacing-ramp step, applied on both axes (wrap rows + chips). Default `sm`.
- **`align`** — `StackAlign`. Default `center`.
- **`justify`** — `StackJustify`.
- **`as`** — host element. Default `div`. **`className`** composes last.
