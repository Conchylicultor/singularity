# cursor-menu

Single-import wrapper for the **DropdownMenu + body-portaled zero-size anchor +
Content** skeleton — the cursor-menu analogue of the `popover` primitive. Pin a
context menu at a cursor point (`e.clientX`/`e.clientY`) without re-deriving the
buggy hand-rolled `position: fixed` trigger.

The trigger is portaled to `document.body` so its `position: fixed` resolves
against the real viewport, never a `transform`ed ancestor (e.g. the surface
backdrop's `transform-gpu`, which would otherwise become the containing block and
shift the menu off the cursor). `createPortal` relocates only the DOM node; the
trigger stays in the React tree under `<DropdownMenu>`, so base-ui Menu context is
intact, and `DropdownMenuContent` already self-portals + theme-forwards.
