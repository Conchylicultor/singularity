# error-boundary

Generic React error boundary primitive used to wrap plugin contributions
so a thrown render error is contained to one slot rather than crashing
the whole tree. The framework is domain-agnostic: it knows how to catch
an error, render a small fallback, and offer a Retry button.

## Reporter

Domain plugins (e.g. `crashes`) opt in by calling `registerBoundaryReporter(fn)`
once at mount time. When a boundary catches, it invokes the reporter
with the `BoundaryErrorReport` and stores whatever the reporter returns
as opaque `context` — passed through to action contributions. The
boundary itself never inspects that context.

## Actions

`ErrorBoundary.Action` is the slot for buttons rendered next to the
fallback's "Retry". Each action receives `{ report, context }`:

```tsx
ErrorBoundary.Action({
  component: ({ report, context }) => {
    const taskId = (context as MyContext)?.taskId ?? null;
    // …
  },
});
```

The `context` shape is a soft contract between whichever plugin owns
the registered reporter and whichever plugins contribute actions —
typically declared as a shared type in the reporter-owning plugin's
barrel.
