# code-api

Typed `defineEndpoint` contracts for the worktree code endpoints (`/api/code/*`:
tree, file, diff, image, push, commit).

These contracts live in their own leaf library plugin (no contributions) so both
sides of the code-viewing relationship can depend on them **downward**, without a
cycle:

- `code-explorer` owns the HTTP routes, server handlers, and the explorer UI —
  and it renders the conversation `file-pane`'s `FilePaneView`
  (`code-explorer → file-pane`).
- The conversation `file-pane` / `commits-graph` / `docs-button` consumers fetch
  these endpoints (`file-pane → code-api`).

If the contracts stayed in `code-explorer/core`, `file-pane` importing them would
close a `code-explorer ⇄ file-pane` import cycle. Keeping them here breaks it:
both plugins import `code-api`, which depends only on the endpoints primitive and
`EditedFileSchema` (a pure schema sink in the `code` plugin).
