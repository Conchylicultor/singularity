# marker-scan-safety

Contributes the repo-wide `no-adhoc-marker-scan` ESLint rule: a
`maskSource(src, { strings: false })` call — masking comments/regex but
**keeping string interiors** — is banned everywhere except a small, reviewed
allowlist of genuine token-in-string scanners.

## Why

Several build-time scanners masked with `{ strings: false }` and then located a
`defineX(...)` marker call to read its first string argument (an id, slot id,
route URL). Because string interiors stay visible, a `defineX("id")` written
INSIDE a string or template literal — a test fixture, a docs snippet, a codegen
template — is still present in the masked text, so the call-locating regex
matches it and its id is extracted as a **real contribution**: a silent false
positive.

The correct, proven contract for a **marker-value** scan is: mask FULLY, locate
the call over the full mask, and read the value back from the *original* source
by offset. Route these through
[`findMarkerCalls(src, "defineX")`](../../../../../../plugin-meta/plugins/parse-utils/CLAUDE.md)
(`@plugins/plugin-meta/plugins/parse-utils/core`) — it masks strings fully and
slices `argsText` from the original — or `markerCallSpans(maskSource(src), …)`
when the surrounding member/group context is also needed (pass a **full** mask
and read values from the original at the returned offsets). A string-embedded
call then vanishes from the masked text; a real call's blanked id is recovered
from the original.

## The rule

Fires on any `maskSource(..., { strings: false })` call (a `strings` property
whose value is the literal `false`), reporting on that property.

## Sanctioned exceptions (on the `ignores` allowlist)

`{ strings: false }` is correct in exactly two shapes, both allowlisted here:

1. **Token-in-string scan** — the token legitimately lives inside a real string
   literal with **no enclosing marker call** (an `/api/<prefix>` URL a caller
   passes to `fetch(...)`, a MIME string, a hardcoded path). Masking fully would
   erase the very thing being searched for. Prefer
   `grepCode({ maskStrings: false })`; the residual fixture false positive is
   inherent and mitigated by skipping test files. Example: the routes facet's
   `relate()` caller scan.
2. **Careful dual-mask** — statement structure is detected on a FULL mask, and
   only the final text is sliced from the strings-kept copy at identical
   offsets. Example: the plugin-boundaries statement splitter.

The remaining allowlist entries are the parse-utils unit tests that exercise the
`{ strings: false }` behavior itself.
