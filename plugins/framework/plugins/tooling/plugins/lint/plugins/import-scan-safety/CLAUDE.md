# import-scan-safety

Contributes the repo-wide `no-adhoc-import-scan` ESLint rule: a **global**
RegExp literal that scans for `import … from "…"` / `export … from "…"` / bare
`import "…"` statements is banned everywhere except the `findImports` primitive.

## Why

A hand-rolled import scanner that keeps string interiors
(`maskSource(src, { strings: false })` then regex) also matches an import
statement written INSIDE a string or template literal — a test fixture, a docs
snippet, a codegen template — reporting fixture data as a real import. Route
whole-file static-import scanning through
[`findImports`](../../../../../../plugin-meta/plugins/parse-utils/CLAUDE.md)
(`@plugins/plugin-meta/plugins/parse-utils/core`): it masks comments/regex AND
strings fully, matches the import structure against the masked text, and reads
each specifier back from the original source by offset. For a construct outside
its scope (dynamic `import()`, `require()`), mask fully with `maskSource(src)`
and read the quote span by offset.

The rule only fires on **global** (`/g`) regexes — the whole-file-scan tell.
Single-statement, anchored parsers (e.g. a trailing `from "…"$` matcher over one
already-isolated statement) are not the footgun and are not flagged. The
allowlist (`ignores`) contains only `find-imports.ts`, the one sanctioned home
for the scanner shape.
