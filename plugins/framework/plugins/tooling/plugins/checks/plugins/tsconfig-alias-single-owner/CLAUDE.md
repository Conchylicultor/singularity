# tsconfig-alias-single-owner

Fails if any `tsconfig*.json` redeclares a `paths` alias that `tsconfig.base.json`
owns (today `@plugins/*`). The alias is declared **once** in the root base config and
every other config inherits it via `extends` — base-relative resolution makes the same
entry correct at every depth. A local copy silently shadows the inherited one and breaks
bun's nearest-`tsconfig.json` module resolution in docgen / `barrel-import` (the
confusing `Cannot find module @plugins/...` far from the cause). This check makes that
regression impossible to reintroduce silently.

Uses `ts.readConfigFile` (parses JSONC, does **not** resolve `extends`) to read each
config's literal `paths`, so it sees only locally-declared aliases.
