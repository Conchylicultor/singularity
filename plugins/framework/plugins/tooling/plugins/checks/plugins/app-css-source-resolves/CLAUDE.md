# app-css-source-resolves

Pins the `@source` glob in
`plugins/primitives/plugins/css/plugins/ui-kit/web/theme/app.css` to the
repo-root `plugins/` directory.

`@source` is what tells Tailwind where to scan for class names. It is authored as
a relative path from app.css's own directory, so **relocating app.css silently
re-aims it** — exactly what happened when ui-kit moved into the `css/` umbrella
(two levels deeper) but kept its 6-deep `../` chain: Tailwind then only scanned
`plugins/primitives/plugins/` and every color/utility used elsewhere
(notification chips, conversation/task status) was never generated, rendering
those elements unstyled.

The file is **text-parsed** (no CSS import) to dodge runtime/boundary questions:
every `@source "<glob>"` is resolved against app.css's directory, and at least
one must land exactly on `<root>/plugins`. A wrong depth fails `./singularity
check` with the resolved target and the correct relative path to use.
