# app-css-utilities-in-sync

Keeps the custom `@utility` classes in
`plugins/framework/plugins/web-core/web/theme/app.css` in sync with
`web-core/web/theme/control-utilities.ts` — the canonical name list consumed by
the `lib/utils.ts` twMerge config. Both files are **text-parsed** (no TS import)
to dodge runtime/boundary questions:

- Forward: every name in `control-utilities.ts` (the `CONTROL_HEIGHT_UTILITIES` /
  `CONTROL_ICON_UTILITIES` / `PAD_UTILITIES` arrays + `ICON_AUTO_UTILITY`) must be
  declared as a `@utility` in `app.css`.
- Reverse: every `control-*` / `p-{chip,control,row}` / `icon-auto` `@utility`
  declared in `app.css` must be registered in `control-utilities.ts` (so its
  twMerge conflict group is configured).

A drift on either side fails `./singularity check` with the offending names and a
hint to re-sync the two files.
