# app-css-utilities-in-sync

Keeps the custom `@utility` classes in
`plugins/primitives/plugins/ui-kit/web/theme/app.css` in **total** sync with
`plugins/primitives/plugins/ui-kit/web/theme/custom-utilities.ts` — the single-source registry from which
the `lib/utils.ts` twMerge config is derived. Both files are **text-parsed** (no
TS import) to dodge runtime/boundary questions:

- Forward: every name in any `*_UTILITIES` array in `custom-utilities.ts` must be
  declared as a `@utility` in `app.css`.
- Reverse (total): every `@utility` declared in `app.css` must appear in some
  `*_UTILITIES` array. There is no allow-list — an unregistered custom utility is
  the silent-strip / fail-to-dedupe bug class (tailwind-merge misfiles a class it
  doesn't know about and drops it), so the check fails loudly on the first one.

A drift on either side fails `./singularity check` with the offending names and a
hint to re-sync the two files. To add a `@utility`: declare it in `app.css`, add
it to a `*_UTILITIES` array, and give that array a `CUSTOM_UTILITY_REGISTRY` entry
(`extend` a built-in group, a synthetic `group` + `conflictsWith`, or `standalone`
with a reason). twMerge wiring is then automatic — no `lib/utils.ts` edit.
