# pre-barrel-manifests-complete

Guards the **pre-barrel freeze invariant**: a `*.generated.ts` that a plugin
barrel imports at module-load (directly or transitively) MUST be regenerated
before the first barrel import in a build run, because Bun freezes a module on
its first `import()` and a later disk write cannot invalidate it.

The check walks every barrel (`web`/`server`/`central` + the web-sdk core seed),
DFS-follows their relative runtime imports, and asserts every reachable
`*.generated.ts` is either a registered pre-barrel manifest (`preBarrelManifests`
in `codegen/core/pre-barrel-manifests.ts`) or a registry-phase output (written in
the earlier `regenerateRegistryCodegen` phase). Fix a failure by registering the
file in `preBarrelManifests`, or by not importing it from a barrel.
