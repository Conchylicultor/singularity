# apps-paths-from-app-ref

Asserts that every `Apps.App` web contribution derives its `id` and `path`
from a named `AppRef` (`<app>.id` / `<app>.basePath`) produced by `defineApp`,
never from hand-written string or const literals. This keeps an app's base
path a single source of truth shared with the typed link builder
(`<route>.link(app, params)`), so the rail entry and deep links can never
drift apart.

The check reads the already-parsed contributions facet (no barrel import),
matching contributions whose slot is `Apps.App`, and validates that
`props.id`/`props.path` are `<ident>.id`/`<ident>.basePath` for the *same*
`ident`, where `ident` is a registered `export const <ident> = defineApp(...)`
binding found by scanning the repo. See `apps/agent-manager` and `apps/pages`
for the precedent.
