# tailwind-scan-covers-classes

Enforces the real invariant behind Tailwind styling: **every utility class used
in source must be present in the compiled CSS** (`used ⊆ emitted`).

Tailwind v4 only emits a class it sees in a *scanned* file, and the scan scope is
set by the `@source "<glob>"` directive in
`plugins/primitives/plugins/css/plugins/ui-kit/web/theme/app.css`. That glob is a
path **relative to app.css's own location**, and the Vite root is deep inside the
tree (`framework/web-core/web`), so the directory scan can only be re-pointed at
the whole `plugins/` subtree by this hand-written `../` chain. Relocate the
stylesheet and the chain silently re-aims — which is exactly how the css/ umbrella
move narrowed the scan to `plugins/primitives/plugins/` and dropped
`max-w-reading` (used only in `conversations/`), blowing out the reading column.

The check resolves every `@source` to its base directory and fails if any
tracked, scannable (`.ts/.tsx/.js/.css/.html`) file that **authors a utility
class** (`className`, `class=`, `cn(`, `@apply`) lies outside the scan scope. It
reads only out-of-scope files, so the correct case touches almost no disk.

This **supersedes the cause-shaped path-pin** (`@source must resolve to
<root>/plugins`): that only knew one failure mode (a wrong `../` depth) under one
assumption (all classes live under `plugins/`). This catches the *effect* —
whatever the cause: a relocated stylesheet, a second narrower `@source`, or a
`className` authored in a new top-level dir all surface as an unscanned file.

The check also requires `@import "tailwindcss" source(none);` — automatic
source detection must stay OFF. Auto-detection scans from the **vite root**,
and the artifact-mode global-css pass builds with the *repo root* as vite root:
on the main checkout, oxide's `walkdir` (which does not honor `.gitignore`)
descended into `.claude/worktrees/` — ~90 agent checkouts, millions of files —
turning the ~5s Tailwind pass into ~320s of cold readdir/stat I/O. With
`source(none)`, the declared `@source` list this check validates is the entire
scan surface, so the two halves together make the invariant exact:
`used ⊆ scanned = declared`.
