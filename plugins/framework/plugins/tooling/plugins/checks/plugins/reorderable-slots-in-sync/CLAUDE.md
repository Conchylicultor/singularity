# reorderable-slots-in-sync

Fails if `plugins/reorder/shared/reorderable-slots.generated.ts` drifts from the
current set of reorderable render slots (computed from the slots facet). Fix by
running `./singularity build` and committing the regenerated manifest.

**Zero-build discovery.** The failure also names, for each slot present in the
freshly-rendered set but absent from the committed manifest, the authored config
override it will owe (`config/<asPath(pluginId)>/<slotId>.jsonc`). It is free —
both manifest strings are already in hand, so the added set is a diff of what the
check computed anyway, with no second scan and no barrel import. This makes a
bare `./singularity check`, on a tree that has never been built, name exactly
which override file is about to be seeded (and which
`config:overrides-authored` will then hold open for review).
