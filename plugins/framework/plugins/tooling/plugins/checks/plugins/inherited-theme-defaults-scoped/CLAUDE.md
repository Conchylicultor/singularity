# inherited-theme-defaults-scoped

Enforces that **inherited, themed CSS defaults** (base `font-family`, default
text `color`, `letter-spacing`, …) are consumed at a **theme-scope root**
(`:root, [data-theme-scope]`), never on a bare root (`html`/`body`/`:root`/`*`)
**above** the per-app scope boundary.

## The bug it prevents

CSS inheritance passes the *computed* value down, not the `var()` reference. A
declaration like `html { font-family: var(--font-sans) }` resolves the var once
at `:root`; a **forked** app's `[data-theme-scope]` descendants then inherit that
already-computed desktop value and silently ignore the app's own scoped
`--font-sans` override (the override block is alive in the cascade but never
re-read). The visible symptom: opening one app's window changes the font of
every other window. Colors avoid this only because they're consumed *per element*
via utilities (`bg-background` → `var(--background)`), which re-read the var
inside each scope.

The fix is to re-declare the inherited default on the scope root itself, so each
`[data-theme-scope]` re-evaluates the `var()` within its own scope:

```css
:root, [data-theme-scope] { font-family: var(--font-sans); color: var(--foreground); }
```

This check makes the bad form impossible to merge for any **future** inherited
default — not just the known font/color cases — which is the "never again"
guarantee an enumerated CSS fix alone cannot give.

## What it flags

Scans `git ls-files 'plugins/**/*.css'` (comment-stripped, `@theme` interiors
masked). A rule is flagged when **all** hold:

- its selector references a bare root (`html`/`body`/`:root`/`*`) **and** does
  not also list a `[data-theme-scope]` selector (which would re-anchor it), and
- **(A)** it sets an inherited property (`INHERITED_PROPS` — `color`,
  `font-family`, `line-height`, `letter-spacing`, …) to a **themed** `var(--x)`
  value (`--x` is a live token-group var per `collectTokenGroupVars`), or
- **(B)** it `@apply`s an inherited+themed utility — base font-family
  (`font-sans|serif|mono`) or themed text color (`text-<token>` where `--<token>`
  is a token-group var; `text-sm` and friends map to non-token suffixes and are
  ignored).

Non-inherited paints (`background`, `border-color`) and per-element rules (any
selector without a bare root) are intentionally not flagged — they re-read the
var at each element and so respect the scope already. (A) is the comprehensive
guard; (B) covers the common `@apply` forms in actual use.

This is the machine enforcement of the scope model in
`framework/web-core/web/theme/CLAUDE.md` and the ui-kit `theme/CLAUDE.md`: a
subtree's theme is its nearest `data-theme-scope` ancestor, so any inherited
default must be re-anchored at that boundary.
