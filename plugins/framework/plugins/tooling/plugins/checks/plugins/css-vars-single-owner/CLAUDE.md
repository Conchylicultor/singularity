# css-vars-single-owner

Enforces that every token-group CSS var has exactly **one declaring owner**.
Token *value* resolution is a deliberate layered cascade (default → preset →
override); token *declaration* is the opposite — single-owner, and any overlap
is a real, ambiguous conflict (the cascade or `ThemeInjector`'s per-group
`<style>` blocks would resolve it silently / order-dependently).

Consumes the generated `TOKEN_GROUP_VARS` manifest (the real `defineTokenGroup`
descriptors, read at build time via their `ui.theme-engine.token-group` web-slot
contributions) and fails if:

- **(a) Cross-group collision** — any `--x` appears in 2+ token groups. Lists the
  var and the groups that claim it. Fix by renaming one group's schema key.
- **(b) Static-CSS overlap** — any token-group var is **declared** (`--x:`, not a
  `var(--x)` reference) in tracked CSS (`git ls-files 'plugins/**/*.css'`,
  comment-stripped), **excluding** declarations inside `@theme { … }` /
  `@theme inline { … }` blocks. Those are Tailwind's build-time utility-token
  layer (a defined lower-precedence position, not an ambiguous same-level
  runtime conflict); the `@theme` interiors are masked via `matchBracket` brace
  matching before the `--x:` scan. Fix by referencing the token via `var(--x)` —
  a token's only declaring owner is its token group (emitted by `ThemeInjector`).

This is the machine enforcement of the prose rule in
`framework/web-core/web/theme/CLAUDE.md` ("plugins consume tokens, never define
them"). The check is a pure function of tracked content + the generated
manifest.
