# Central Runtime

The **shared central runtime** ‚Äî a single long-running process that handles cross-worktree concerns (secrets, OAuth, cross-namespace notifications). Unlike the per-worktree `server/`, there is exactly one instance of central running for the entire host.

## RULES

- **NEVER modify anything in `central-core/` unless the user explicitly instructs so.** Changes here affect all worktrees simultaneously and cannot be sandboxed inside a worktree branch. Always get explicit approval before touching this directory.

## How It Works

1. `bin/index.ts` starts `Bun.serve({ unix: process.env.SOCKET_PATH })` on the Unix socket the gateway hands it.
2. Each plugin declares its routes via a `CentralPluginDefinition` (defined in `core/types.ts`).
3. `bin/plugins.ts` is a flat list of plugin imports ‚Äî structurally identical to the server and web registries.
4. At startup, plugins are topo-sorted by `dependsOn`, registered sequentially, then `onReady` runs concurrently.

## Path Aliases

- `@plugins/*` (plugin tree) ‚Äî declared **once** in the root `tsconfig.base.json` and inherited here via `extends`; never redeclared. Resolves relative to the base file (`<repo-root>/plugins/*`). The `tsconfig-alias-single-owner` check fails any tsconfig that re-adds a local copy.
