# attempt

Renders raw `att-<id>` strings (e.g. `att-1777756427-w4nu`) inline as
clickable chips. Agents write the bare id in their text — no tag wrapping
needed — and the active-data linkify primitive replaces matches at render
time.

Clicking opens the referenced attempt in `attemptPane` (`/a/:attemptId`).
The chip resolves the attempt's status and conversation count via
`attemptsResource`; falls back to the raw id if the attempt isn't in the
index yet.

The id pattern matches attempt ids derived from the worktree basename
(`att-<unix-seconds>-<4 base36 chars>`).

## Plugin reference

- Description: Renders raw `att-<id>` strings inline as clickable chips that open the attempt pane. Models emit the bare id, no tag wrapping needed.
- Exports (web):
  - Values: `AttemptChip`
- Contributes:
  - `ActiveData.Tag` `ATTEMPT_ID_RE` → `AttemptChip`
