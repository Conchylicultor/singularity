# plugin-link

Renders plugin hierarchy IDs in backtick-wrapped inline code as clickable
chips that open the plugin-view pane. When a model emits `` `tasks` `` or
`` `active-data.conv` ``, this plugin intercepts it at render time and
validates against the live plugin tree. If the text matches a known
`hierarchyId`, the `<code>` wrapper is replaced with a chip that opens
`plugin-view`; otherwise the original code appearance is preserved silently.

**Pattern**: broad kebab-case + dot-joined regex; validation happens in the
component via `GET /api/plugin-view/tree` (cached 60 s, TanStack Query key
`["plugin-view-tree"]`).
