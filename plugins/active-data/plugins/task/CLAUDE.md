# task

Renders `<task>prompt</task>` tags in assistant text as inline editable cards
with **Create** and **Launch** actions. Lets the model propose a task that the
user can immediately tweak and act on without leaving the transcript.

The card hosts a Lexical `PromptEditor` (paste-images plugin) seeded with the
tag's children. The host conversation's task is read via
`conversationPane.useData()` and used as the parent for the new task.

- **Create** — `POST /api/tasks` with `{ parentId, description }`. The card
  collapses to a small task chip linking to `taskDetailPane`.
- **Launch (Sonnet / Opus)** — pre-creates the task with the host as parent,
  then `LaunchButtons` posts `/api/conversations` with the resulting `taskId`
  and the (possibly edited) prompt. The card collapses to a `ConvChip`.

Multiple `<task>` instances in one assistant message are independent — each
gets a `useId()`-derived Lexical namespace.
