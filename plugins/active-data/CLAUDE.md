# active-data

Meta plugin for inline interactive widgets agents render via XML-like tags in
their assistant text output. Sub-plugins under `plugins/active-data/plugins/<tag>/`
each claim one tag name and ship the component that renders it.

The host (`assistant-text` in the JSONL viewer) calls
`useActiveDataRenderer()` and applies the returned function inside its
react-markdown component overrides — same shape as `linkifyChildren` from the
`file-links` primitive. Parsing is regex-based, flat (no nested tags), and
skips `<code>`, `<pre>`, and `<a>` so tags inside fenced blocks render literally.
Renderers needing the host conversation read it via `conversationPane.useData()`
directly; the slot does not pipe it.
