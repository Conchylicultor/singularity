# active-data

Meta plugin for inline interactive widgets agents render via XML-like tags in
their assistant text output. Sub-plugins under `plugins/active-data/plugins/<tag>/`
each claim one tag name and ship the component that renders it.

Hosts (currently `assistant-text` in the JSONL viewer) merge
`useActiveDataComponents()` into their react-markdown `components` map, paired
with `rehype-raw` so raw HTML in the markdown source reaches the components
map instead of being stripped during the mdast→hast pass. The contribution
shape exposes `attrs: Record<string, string>` and `children: string` to the
component; the adapter coerces react-markdown's `ReactNode` children into a
plain string before delegation. Tags inside fenced code blocks (`` ` ``,
``` ``` ```) render literally because react-markdown doesn't apply HTML
parsing inside `code`/`pre`. Renderers needing the host conversation read it
via `conversationPane.useData()` directly; the slot does not pipe it.
