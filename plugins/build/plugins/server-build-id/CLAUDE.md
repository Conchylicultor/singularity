# server-build-id

Server-only leaf that reads the `.build-id` baked into the served frontend
bundle (`${WEB_DIST_DIR}/.build-id`) **fresh on every call тАФ never memoized**.
`./singularity build` swaps the `dist` symlink (what the browser downloads)
*before* it restarts this backend, so a value cached at process start would
disagree with the currently-served bundle during the swapтЖТrestart window and
leave the "Server updated" reload button stuck (the tab loads the new bundle
while the server keeps reporting the old id). Reading the tiny dotfile per call
resolves through the same `dist` symlink the gateway serves from, so the reported
id always matches the bundle in the browser's hands. Extracted out from behind
the heavy `build/server` barrel so stale-tab detection can read the build id
without pulling `git-watcher`/`worktree` into its closure. Its only dependency is
`infra/paths`; a synchronous, boot-safe read (reports' boot-time backfill reads
it eagerly).
