# fields

Owner of the config_v2 **renderer-dispatch slot** `config-v2.fields.renderer`
and its shared renderer helpers. This plugin no longer hosts field-type
sub-plugins — every field type now lives in the global `fields/` matrix
(`plugins/fields/plugins/<type>/`). A type's config-render capability is a
`plugins/config` sub-plugin there that contributes its renderer into the slot
this barrel owns.

See [config v2 vision](../../../research/2026-05-16-config-v2-vision.md) for the
overall config design, and [fields/CLAUDE.md](../../../fields/CLAUDE.md) to add
a new field type.

## What this barrel owns

`web/index.ts` exports the slot machinery and the shared helpers a renderer
imports:

- `Fields` — the contribution factory; renderers call `Fields.Renderer(<R>)`.
- `FieldRenderer` — recursive dispatch into a sub-field (used by `list` /
  `object` renderers to recurse).
- `ConfigFieldContext` — the per-field render context.
- `FieldHeader` — the label/description block.
- `useLocalValue` — focus-aware local edit state for text-like inputs.
- `FieldRendererComponent` / `FieldRendererProps` — the renderer component type.

The slot id is `"config-v2.fields.renderer"` (`web/internal/slots.tsx`); dispatch
is by `field.type.id`. The slot is **frozen** — a `plugins/config` renderer
sub-plugin contributes into it without this barrel changing.

## Adding a field type

Field types are added under the `fields/` matrix, not here. See
[fields/CLAUDE.md](../../../fields/CLAUDE.md): create
`plugins/fields/plugins/<type>/` (token + identity in `core`, `Fields.Identity`
in `web`) plus a `plugins/config` sub-plugin whose `web/index.ts` contributes
`Fields.Renderer(<R>)` into the `config-v2.fields.renderer` slot owned here. The
renderer imports `FieldRendererComponent` (and any helper above) from
`@plugins/config_v2/plugins/fields/web`. `avatar`, `list`, and `reorder-tree` are
the config-only templates to mirror.
