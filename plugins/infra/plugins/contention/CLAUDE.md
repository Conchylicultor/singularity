# contention

Server-only infra primitive exposing a cached, cluster-wide system-contention snapshot via `getContentionSnapshot()`. It pairs the OS load average and core count (`os.loadavg()` / `os.cpus()` — every process on the box, including all git subprocesses) with the cluster-global Postgres backend counts read from `pg_stat_activity` (a cluster-wide view, so the worktree's own pool sees every backend across all worktree DB forks). The snapshot is memoized for ≤1s via a lazy `performance.now()` timestamp gate (not a poller) so a storm of slow ops collapses onto one read instead of self-amplifying the contention, and the pg read runs inside `runWithoutProfiling` so it never re-feeds the slow-op recorder it's captured for. Consumed by `debug/slow-ops` to stamp the box state onto each slow span.
