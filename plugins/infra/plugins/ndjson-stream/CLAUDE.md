# ndjson-stream

The shared NDJSON streaming primitive. On the server, `ndjsonResponse(produce)`
wraps a producer that calls `emit(frame)` into an `application/x-ndjson` chunked
`Response`, framing an unexpected throw as `{error}` and always closing the
stream. On the web, `readNdjson(route, url, init?)` consumes it as an async
generator, yielding one parsed JSON frame per line while guarding `res.ok` and
reporting failures via `EndpointError`.

This is the sanctioned home for streaming pull-endpoints that
`implement()`/`fetchEndpoint`'s single-JSON-response model can't express — e.g.
heavy fan-outs that must render rows progressively as they arrive, or survive
Bun's 10s idle timeout.
