# commits-graph

Toolbar chip showing how many commits the conversation's worktree branch is ahead of (and behind) `main`. Clicking opens a side pane that draws the chain of commits between the merge-base and HEAD as a single colored rail (VSCode-Git-Graph style, single-branch scope).

Counts are computed server-side via `git rev-list --left-right --count main...HEAD` and pushed live. Both resources refresh on two signals:

- **`pushesResource`** — re-runs git whenever a push lands for any attempt (and feeds the push count + the graph's pushed-sha set).
- **`git-watcher`'s `refHeadResource`** — re-runs git whenever `main` or this worktree's own branch advances, so a local commit / rebase / **sync-to-head** updates the chip even though no push happened. Each resource tracks its actively-subscribed attempts via `onFirstSubscribe`/`onLastUnsubscribe` and fans a ref advance out only to the chips currently on screen.
