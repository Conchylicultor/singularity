# exit-menu

A single ghost icon button rendered next to **Push & Exit** in the prompt
editor toolbar. Clicking it opens a dropdown of the various ways to leave a
conversation.

The menu is **slot-driven**: this plugin owns the `ExitMenu.Item` slot and
renders whatever contributes to it (sorted by `order`), wrapped in the
error-boundary middleware via `renderIsolated`. It never names individual
actions — adding a new exit action is just a new `ExitMenu.Item` contribution.

Current contributors (siblings under `conversation-view/plugins/`):

- `hold-and-exit` — Hold & Exit
- `exit` — Exit
- `drop-and-exit` — Drop & Exit (morphs to Complete & Exit / Exit by state)
- `drop-dependents` — Drop task + N dependents & Exit (hidden when none)

Each contributor renders its own `DropdownMenuItem` and may return `null` to
hide itself based on conversation state.
