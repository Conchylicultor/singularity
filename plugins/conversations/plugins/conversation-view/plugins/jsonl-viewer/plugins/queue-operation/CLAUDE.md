# queue-operation

Renders Claude Code prompt-queue events in the JSONL viewer. These are raw
`type:"queue-operation"` lines emitted when an item is **enqueued** (a queued
follow-up prompt, or a `<task-notification>` block from a completed background
task), **dequeued** (pulled off and sent to the agent on its next turn), or
**removed** (discarded without being sent).

Each operation renders as a compact one-line row with an icon and a label
(`Queued` / `Sent to agent` / `Removed from queue`). When the enqueued content
is a `<task-notification>` block, only its status + summary are surfaced as a
badge — the full XML stays available via the row's hover-only raw-JSON action.

The classification happens in the transcript parser
(`plugins/conversations/plugins/transcript-watcher/server/internal/parse-jsonl.ts`),
which emits `kind:"queue-operation"` events that this plugin's `EventRenderer`
matches.
