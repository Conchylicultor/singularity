# teammate-message

Renders messages relayed from OTHER Claude sessions in the JSONL viewer. The
harness drops a `<teammate-message teammate_id="…" color="…" summary="…">…</teammate-message>`
block into a `type:"user"` turn (sometimes wrapped in a pretty-printed JSON
envelope alongside the human's own typed text). These are not authored by the
human, so rendering them as a normal user bubble is misleading — instead they
render as a distinct inbound peer-message card: an auto-colored avatar disc
(keyed on the teammate id), the teammate's name, an optional summary note, and
a collapsible body (a markdown report, or a one-line summary for idle/status
JSON notifications).

The classification happens in the transcript parser
(`plugins/conversations/plugins/transcript-watcher/server/internal/parse-jsonl.ts`),
which decodes the relay forms via the `relay.ts` core helpers
(`unwrapRelayEnvelopes` / `extractTeammateMessages` / `stripRelayBoilerplate`)
and emits `kind:"teammate-message"` events that this plugin's `EventRenderer`
matches. Any human text bundled in the same turn survives and renders as a
normal user message.
