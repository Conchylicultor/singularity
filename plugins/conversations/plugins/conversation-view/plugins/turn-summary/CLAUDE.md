# turn-summary

Inline card rendered above the prompt input that shows a Haiku-generated summary of the latest exchange. Updates automatically after every assistant turn.

The card shows three things:

1. A one-line summary of what just happened.
2. A bulleted list of caveats / things to review or decide.
3. A bulleted list of suggested next actions.

The detail rows are collapsible; the one-liner is always visible.

## How generation fires

A persistent global trigger on `conversationTurnCompleted` (installed in this plugin's `onReady`) enqueues `turn-summary.generate` for every assistant turn across every conversation. The job is idempotent on `(conversationId, messageId)` â€” if a row already exists for the assistant message id from the event payload, the job no-ops. This keeps re-emitted events (e.g. after server restart) safe.

## Boundary notes

- Storage is plugin-owned (`turn_summaries` table, FK to `_conversations` with cascade-delete). One row per conversation, replaced on every new assistant turn.
- Rendering goes through the new `Conversation.AbovePromptInput` slot on the conversation-view host. Other plugins can contribute banners to the same slot.
- The Haiku call uses `runClaudePrint({ model: "haiku" })` from the `claude-cli` primitive â€” no Anthropic SDK plumbing, no API key. Failures are logged and swallowed; no row is written, so the card stays at the previous summary.
- Markdown bullet parsing is intentionally tolerant â€” Haiku occasionally drops headers or formats bullets with `*` instead of `-`. The parser recovers: missing sections become empty, dashless leading lines become a single item, and a totally malformed reply still fills `summary` so the user sees something.
