# conversation-preprompt

Records the preprompt a conversation's task was launched with as a snapshot on
the conversation, and surfaces it as a chip in the conversation header.

When a conversation is created, a job bound to the `conversationCreated` trigger
event resolves the launching task's selected preprompt and copies its id, title,
and full text into a `conversations_ext_preprompt` side-table (via the
entity-extensions primitive). The snapshot is intentional: the chip reflects
exactly what the agent was launched with, even if the config preprompt is later
edited or deleted.

The header chip (`Conversation.Header` slot) reads that snapshot from a
live-state resource keyed by conversation id and renders the preprompt title;
clicking it opens a popover with the full instruction text.

## Boundary notes

The recording is wired purely through the `conversationCreated` event binding —
this plugin is never imported by the load-bearing `conversations` lifecycle.
That keeps the dependency graph a DAG (conversations → conversation-preprompt,
never the reverse) and the feature fully opt-in and removable.
