# transcript-api

`GET /api/conversations/:id/transcript` → `{ paths: string[] }`.

A conversation is **not** one JSONL file. Claude Code relocates a live session into
a new id (a fork copies its ancestor's lines verbatim; a fresh daemon-hosted session
inherits nothing), and each id gets its own `<sessionId>.jsonl`. Singularity records
that succession as the conversation's **session chain**, and this endpoint returns
the chain's files in order, oldest → newest. Read them in that order and dedup by
`uuid` (first occurrence wins) to reconstruct the conversation — or just call
`readJsonlEventsFromChain` / `readTurnsFromChain`, which do exactly that.

- `[]` means no transcript has landed on disk yet (a warming-up conversation), not
  a failure. A missing conversation row is a `404`; a DB or filesystem failure is a
  `500`. The three are never conflated.
- The **last** path is the live tail — the file `claude --resume` appends to. It is
  the only one anything may write to; the ancestors are history the read path
  depends on.
