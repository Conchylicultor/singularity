# string-list / config

The **string-list** type's config capability: the `stringListField` factory
(core) and the textarea renderer (one item per line) contributed to
`config-v2.fields.renderer` (web).

The factory builds a config_v2 `FieldDef<string[]>` carrying the canonical
`fields/` string-list token (`@plugins/fields/plugins/string-list/core`, id
`"string-list"`); dispatch is by id string. The zod schema is a bare
`z.array(z.string())` — no per-item id/rank injection, so the stored value stays
a plain `string[]`.

The settings-pane renderer is intentionally minimal: a textarea with one item
per line. It is the secondary view — the primary editor is wherever the strings
are produced (e.g. the in-app drag UI for reorder). `FieldRendererComponent`
comes from the slot owner (`@plugins/config_v2/plugins/fields/web`).
