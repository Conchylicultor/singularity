# string-list / config

The **string-list** type's config capability: the `stringListField` factory
(core) and the drag-and-drop list renderer contributed to
`config-v2.fields.renderer` (web).

The factory builds a config_v2 `FieldDef<string[]>` carrying the canonical
`fields/` string-list token (`@plugins/fields/plugins/string-list/core`, id
`"string-list"`); dispatch is by id string. The zod schema is a bare
`z.array(z.string())` — no per-item id/rank injection, so the stored value stays
a plain `string[]`.

The settings-pane renderer is a sortable list of individual string rows: each
row is a text input bound to one entry plus a drag handle and a remove button,
with an "Add item" button that appends an empty entry. It composes the
`sortable-list` primitive (`SortableList` + `SortableItem`). Because the value
is a bare `string[]` (no per-item id), the renderer keeps a parallel array of
generated UUIDs positionally aligned to `value` for `SortableItem`'s stable-id
contract, reconciled whenever the external value's length changes. Per-row
edits use a focus-tracked local buffer flushed to `onChange` on blur; reorder
and remove flush immediately. `FieldRendererComponent` comes from the slot owner
(`@plugins/config_v2/plugins/fields/web`).
