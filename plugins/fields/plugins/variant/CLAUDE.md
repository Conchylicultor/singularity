# variant

The **variant** field type of the unified fields primitive — a
discriminated/polymorphic object whose `type` key selects which "variant" the
rest of the value's keys belong to. The value type is
`VariantValue = { type: string } & Record<string, unknown>`.

This barrel registers ONLY the `FieldIdentity` (label "Variant", `MdCallSplit`
icon) in the `fields.identity` registry. There is no `coerce` and no `extends`.

Storage is opaque/passthrough at the config boundary (mirroring a `reorder-tree`
node `{ type, ...payload }`): the config schema validates only the `type`
discriminant and preserves the payload verbatim. Per-type validation of the
payload happens downstream, against an injected per-type field registry.

The config-render capability — the `variantField` factory, the downstream
`validateVariant` helper, and the type-dispatched renderer contributed to the
frozen `config-v2.fields.renderer` slot — lives in the `plugins/config`
sub-plugin.
