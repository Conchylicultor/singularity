# object / config

The **object** type's config capability: the `objectField` factory (core) and
the collapsible sub-field renderer contributed to `config-v2.fields.renderer`
(web).

The factory builds a config_v2 `FieldDef<Record<string, unknown>>` carrying the
canonical `fields/` object token (`@plugins/fields/plugins/object/core`, id
`"object"`); dispatch is by id string. Its zod schema composes each sub-field's
`fieldSchemaWithDefault` into a passthrough `z.object`, so a key missing from a
stored document heals to that sub-field's default. The web renderer wraps the
sub-fields in a collapsible section (default expanded) and dispatches each one
via `FieldRenderer` — nested `objectField` instances recurse automatically.
`FieldRenderer` / `FieldRendererComponent` come from the slot owner
(`@plugins/config_v2/plugins/fields/web`).
