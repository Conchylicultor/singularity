# date / filter

The **date** type's `data-view.filter` capability.

Contributes `DataViewSlots.Filter({ match: "date", ... })` — the full date
operator set (`is` / `is before` / `is after` / `is on or before` /
`is on or after` / `is between` / `is within the past` / `is within the next` /
`is empty` / `is not empty`) plus the pure `predicate`s applied in the data-view
row pipeline. The `to` / upper bound is treated as end-of-day, so a row matches
when its timestamp falls within `[lo 00:00, hi 23:59:59.999]`.

## Relative anchors

Every scalar operand is a **`DateAnchor`** (`web/internal/date-anchor.ts`), a
JSON-serializable union resolved against `now` *at filter time*:

- `{ kind: "date"; iso }` — an absolute calendar day (the exact-date picker).
- `{ kind: "relative"; unit; amount }` — `now ± amount` units (signed: `<0` ago,
  `>0` from now), e.g. Today / Yesterday / Tomorrow / "3 days ago". A saved
  "due on or before Today" stays correct as the clock advances.

A legacy bare ISO string still resolves as an absolute day (backward compat), so
pre-existing stored operands keep working. `resolveAnchorDay(operand, now?)` is
the single resolver; `now` defaults to `Date.now()` and exists only so the unit
tests can pin it. `addUnits` shifts via `Date` setters, so month/year arithmetic
and DST stay calendar-correct.

The value inputs (`web/components/date-filter.tsx`) are a popover-driven anchor
chooser (presets + a custom-relative builder + an exact-date picker), composed
from the shared primitives (`InlinePopover`, `Row`, `Stack`/`Inset`, `Text`,
`SectionLabel`, `Button`).

## Within operators

`is within the past` / `is within the next` take a magnitude operand
`{ unit, amount }` (positive; default `1 week`); the direction is implied by the
operator. `withinRange(operand, "past"|"next", now?)` resolves the inclusive
`[lo, hi]` start-of-day window around today; an empty/invalid operand keeps the
row (incomplete rule), mirroring the scalar operators.
