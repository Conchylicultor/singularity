# dynamic-enum / config

The **dynamic-enum** type's config capability: the `dynamicEnumField` factory
(core) and the renderer contributed to `config-v2.fields.renderer` (web).

Relocated from `config_v2/plugins/fields/plugins/dynamic-enum`. The factory
builds a config_v2 `FieldDef<string>` carrying the canonical `fields/`
dynamic-enum token (`@plugins/fields/plugins/dynamic-enum/core`, id
`"dynamic-enum"`); dispatch is by id string. `FieldHeader` and the renderer slot
come from the slot owner (`@plugins/config_v2/plugins/fields/web`).

Options are not fixed: hosts contribute the live option set through the
`DynamicEnum.Options` slot (id `fields.dynamic-enum.options`), matched against
the field descriptor by reference equality. The renderer falls back to a plain
text input when no contribution matches.
