# dynamic-enum

The **dynamic-enum** (dynamic select) field type of the unified fields
primitive — a single-choice string value whose option set is not fixed at
declaration time but resolved at config-render time.

This barrel registers ONLY the `FieldIdentity` (label "Dynamic Select",
`MdArrowDropDownCircle` icon, identity `coerce` to string) in the
`fields.identity` registry. It is a standalone identity — it does NOT
`extend` enum, because enum's capabilities read `field.options`, which
dynamic-enum lacks.

The type's config capability lives in the `plugins/config` sub-plugin: the
`dynamicEnumField` factory, the renderer, and the `DynamicEnum.Options` slot
through which hosts contribute the live option set (matched by reference
equality on the field descriptor, with a text-input fallback when unmatched).
