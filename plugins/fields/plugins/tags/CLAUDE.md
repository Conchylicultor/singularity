# tags

The **tags** (multi-value) field type of the unified fields primitive — a *set*
of string labels per row, rather than a single scalar.

This barrel registers ONLY the `FieldIdentity` (label "Tags", `MdLabel` icon; NO
`extends`, NO `coerce`) in the `fields.identity` registry. Unlike a scalar type,
a `tags` field carries its values through `FieldDef.values: (row) => string[]`
(not `value`), so the data-view search/filter paths treat it as an array.

The type's capabilities live in dedicated sub-plugins under `plugins/`:

- `plugins/filter` — the multi-select `FilterContribution` (`data-view.filter`):
  one `ToggleChip` per `field.options`, toggling membership of a
  `selected: string[]`, plus an **array-aware** predicate that keeps a row when
  its tag set intersects the selected set (match-any).

It deliberately does **not** `extends enum`: `extends` would resolve enum's
*scalar* predicate, which tests one value's membership in the selected set — the
wrong semantics for a multi-valued row. `tags` stands alone with its own
array-aware filter.
