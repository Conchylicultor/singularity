# enum

The **enum** (select) field type of the unified fields primitive — a
single-choice value drawn from a fixed set of string options.

This barrel registers ONLY the `FieldIdentity` (label "Select", `MdList` icon,
identity `coerce` to string) in the `fields.identity` registry. The type's
capabilities live in dedicated sub-plugins under `plugins/`:

- `plugins/config` — the config-render capability: relocates the enum renderer
  into the matrix, contributing to the frozen `config-v2.fields.renderer` slot.
- `plugins/table` — the read-only chip `EnumCell` (`data-view.cell`), resolving
  the raw value to its option label via `field.options`.
- `plugins/filter` — the multi-select `FilterContribution` (`data-view.filter`):
  one `ToggleChip` per option, toggling membership of a `selected: string[]`.

The enum factory + token still live in `config_v2` for now
(`@plugins/config_v2/plugins/fields/plugins/enum/core`); config descriptors keep
importing `enumField` from there. Dispatch is by id string `"enum"`, so the
relocated renderer (whose `.type` is this `fields/` enum token, also id `"enum"`)
still matches descriptors built with config_v2's `enumField`. The two id-equal
tokens unify in a later task.
