# secret

The **secret** field type of the unified fields primitive — an encrypted
string value with set/not-set metadata, never stored in plaintext config.

This barrel registers ONLY the `FieldIdentity` (label "Secret", `MdKey` icon)
in the `fields.identity` registry. The value type is a plain `string`.

## Security invariant

The secret identity registers **NO `coerce`** and contributes **NO data-view
cell or filter** (no `plugins/table` / `plugins/filter` sub-plugin). Omitting
`coerce` plus having no cell contribution is what keeps a secret out of every
data-view projection: there is no path by which a secret value can become a
readable table cell. Do not add a `coerce` or any table/filter capability to
this type.

The config-render capability — the `secretField` factory, the password-input
renderer contributed to `config-v2.fields.renderer`, the encrypted
`FieldStorageProvider` (namespace `"config-fields"`), the live-state meta
resource (`"config-v2.secret-meta"`), and the central `readSecretConfig` reader
consumed by auth providers — lives in the `plugins/config` sub-plugin across its
four runtimes (core / web / server / central).
