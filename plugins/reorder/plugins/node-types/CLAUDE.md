# node-types

The reorder **node-type registry** ‚Äî the extensible vocabulary of node types a
reorder tree can hold, mirroring the `fields` primitive's `fields.identity`
registry. A node type owns its payload schema and its render; the core tree
format (`fields/reorder-tree/core`) only knows the structural `type`/`id`/`items`
fields.

## Mental model

- `node-types/core` ‚Äî **types only** (`ReorderNodeType<P>`,
  `ReorderNodeRenderProps<P>`), browser-safe, no registry values. Each node type
  is generic over its payload `P` and carries its own `schema: ZodType<P>`.
- `node-types/web` ‚Äî owns the `reorder.node-type` slot (`ReorderNodes.NodeType`)
  and the `useReorderNodeTypes()` read hook (a `Map<string, ReorderNodeType>`).
  Imports only `web-sdk/core` + `node-types/core`, so both consumers (the live
  reorder middleware and the `reorder-tree` config field) can import the barrel
  without a cycle.
- Built-in node types are sub-plugins, each contributing one
  `ReorderNodes.NodeType(...)` and importing `editor/web` for the shared
  sortable primitives:
  - `plugins/spacer` ‚Äî leaf node type (blank draggable gap), with an "Add Spacer"
    insert.
  - `plugins/header` ‚Äî the one container node type (labeled, collapsible box that
    renders its pre-rendered members as children). Container creation is
    config-only ‚Äî no `insert`.

## Adding a node type

1. Create `plugins/reorder/plugins/node-types/plugins/<type>/`.
2. `web/index.ts` default-exports a plugin contributing
   `ReorderNodes.NodeType({ nodeType: { type, container, schema, render, insert? } })`.
3. Run `./singularity build` ‚Äî the registry is autogenerated from the filesystem.

## Plugin reference

- Description: Reorder node-type registry: owns the reorder.node-type slot and the useReorderNodeTypes() read hook. Slot owner only ‚Äî contributes no node types itself.
- Web:
  - Slots: `ReorderNodes.NodeType` ‚Üê `reorder.node-types.header`, `reorder.node-types.spacer`
  - Exports: Values: `ReorderNodes`, `useReorderNodeTypes`
- Cross-plugin:
  - Imported by: `fields/reorder-tree/config`, `reorder`, `reorder/node-types/header`, `reorder/node-types/spacer`
- Core:
  - Exports: Types: `ReorderNodeRenderProps`, `ReorderNodeType`
- Sub-plugins:
  - **`header`** ‚Äî Header reorder node type: the one container type ‚Äî a labeled, collapsible box rendering its pre-rendered members. Owns the label/collapsed payload schema; collapse toggles via onPatch.
  - **`spacer`** ‚Äî Spacer reorder node type: a blank draggable gap (leaf), with an 'Add Spacer' insert affordance.
