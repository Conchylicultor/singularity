# prototypes

Prototypes app — a surface to browse, **Focus** on, **Compare**, and iterate on
throwaway UI design mockups. The mockup content + shared harness live in a
repo-root `prototypes/` directory (tracked, sibling to `research/`), authored
separately from this plugin: each mock is `meta.json` + an `App` component +
optional styles, mounted by `prototypes/_shared/harness.html`.

This top-level plugin is an empty namespace (create-app rule). All content lives
in sub-plugins:

- **`files`** — server-side raw file serving for `prototypes/*`, the live-state
  list + version resources, and the file-watcher that auto-reloads open iframes
  on edit. Owns the `/api/prototypes` route contract.
- **`shell`** — the `Apps.App` rail entry, the app layout, and pane registration.
- **`gallery`** — the gallery list pane and the Focus/Compare detail pane.

Design: `research/2026-06-16-global-prototypes-design-workflow.md`.
