# controls

## Keyboard transport (per-surface, focus-scoped)

Sonata's keyboard transport is **per-surface and focus-scoped** — there is no
module-level command bus. Both pieces mount as `Sonata.Effect`s inside
`SonataProvider` (so they read `useSonata()` and the surface id), and both are
gated on a song being open (`currentSongId != null`) so they are inert on the
library:

- **`TransportShortcuts`** (`components/transport-shortcuts.tsx`) — registers
  Space (play/pause), ↑ (speed up), ↓ (slow down) via `useSurfaceShortcuts`.
  Each shortcut is eligible only while ITS surface is the focused one and its
  handler closes over THAT surface's own transport verbs. Tempo step is
  `TEMPO_STEP` (`shortcuts.ts`).
- **`SeekHoldController`** (`seek-hold-controller.tsx`) — owns ←/→ via a raw
  window listener (it needs keyup + OS auto-repeat to tell a *tap* — jump one
  bar — from a *press-and-hold* — accelerating bar-by-bar scrub). Because it is
  outside the shortcut registry it enforces focus itself
  (`getFocusedSurfaceId() === thisSurface`) and the song gate, so an arrow hold
  in a foreground window can't scrub a background Sonata window.

This replaces the old `transport-store.ts` singleton bus, whose last-mounted
publisher ignored focus (Space in one window toggled the other).
