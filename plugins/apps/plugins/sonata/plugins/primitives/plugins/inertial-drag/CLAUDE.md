# inertial-drag

A reusable 1-D pointer-drag primitive with **release momentum**: grab a surface
and drag to scrub a scalar value, then flick and let go — the value coasts in the
drag direction under exponential friction and settles, iOS/Android fling style.

`useInertialDrag` (from the `web` barrel) returns pointer handlers to spread on a
surface plus a `phase` (`idle` / `dragging` / `flinging`) for cursor affordance.
The physics runs internally in pixel space so `friction` (a 1/s time-constant)
and `minVelocity` (px/s) have scale-invariant defaults; the consumer's
`unitsPerPixel` converts to value units only at emit time, hard-clamped to
`bounds` (a bound-hit stops the fling, no rubber-band bounce).

The pure, framework-free math lives in `core/`: `decay.ts` is the closed-form
exponential-friction deceleration model (sampled by elapsed wall-time so the
animation is frame-rate-independent), and `velocity-tracker.ts` is a
trailing-window pointer-velocity estimator (a drag-pause-release reports ≈0, so
it does not fling). Both are `bun test`-covered.
