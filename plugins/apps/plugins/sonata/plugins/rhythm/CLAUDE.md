# rhythm

The *when* of Sonata. A framework-free leaf that owns rhythm as a first-class
data type — the missing orthogonal axis next to `voicing` (which owns *which*
tones sound). It imports nothing and speaks only plain numbers, so the rhythm
circle primitive and the per-hand chord-grid grooves reuse one home.

A `RhythmPattern` is a rotating onset necklace: `subdivisions` evenly-spaced
pulses per bar, the `onsets` that strike, and a cyclic `rotation`. Every
operation is **pure** — inputs are never mutated; a fresh pattern (with fresh
arrays) is always returned.

`rotation` is stored *separately* from `onsets` so a rotated preset can still
name itself ("Son ⟳2") instead of collapsing to "Custom". A bead click maps back
through it: `rawIndex = (clicked − rotation) mod subdivisions`, then
`toggleOnset` flips it and sets `presetId = null`.

The presets in `presets.ts` are the literal onset arrays from
[rhythm-circle.com](https://rhythm-circle.com/)'s minified bundle. That site does
**no** resampling — selecting a preset snaps the ring's subdivision count to the
preset's native length and copies the array 1:1. Our `resample()` (proportional
`round(i · n / old)`, deduped) is an *addition* the site lacks: it adapts a
groove's shape across a subdivision change (e.g. Son 16 → 12) so the two hands
can run at independent pulse counts.

`findRhythm` throws on an unknown id — mirroring `findVoicing`, an unknown id is
a bug, not an absorbable empty value.
