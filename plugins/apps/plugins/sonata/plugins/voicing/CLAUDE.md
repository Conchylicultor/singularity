# voicing

Chord → notes for Sonata. A chord-authoring source produces timed
`ChordEvent`s (`{ data, start, end }` in quarter-note beats); a **voicing
strategy** derives the literal `Note[]` that the player sounds and draws — the
opposite direction of the MIDI source.

This is a shared leaf so multiple sources reuse one implementation. `chord-grid`
and the Ultimate Guitar source both call `findVoicing(id).voice(events, opts)`.
The caller owns its own `track` and note-id `idPrefix` via `VoicingOptions`, so
two sources never collide on track or note id. Pitch math is delegated to
`theory`'s `chordPitches`; this leaf only decides which tones sound and when.

The `Voicing` / `ChordEvent` shapes are the stable contract: when cross-source
voicing selection is wanted, this registry can be promoted to a slot without
changing those types.
