# voicing

Chord → notes for Sonata. A chord-authoring source produces timed
`ChordEvent`s (`{ data, start, end }` in quarter-note beats); the **voicing
engine** (`voiceChords(events, opts)`) derives the literal `Note[]` that the
player sounds and draws — the opposite direction of the MIDI source.

This is a shared leaf so multiple sources reuse one implementation. `chord-grid`
and the Ultimate Guitar source both call `voiceChords(events, opts)` (via
`reVoiceChords`). The caller owns its own `track` and note-id `idPrefix` via
`VoicingOptions`, so two sources never collide on track or note id. Pitch math is
delegated to `theory`'s `chordPitches`; this leaf only decides which tones sound
and when.

Two orthogonal axes drive emission:

- **Rhythm-grid** (the *when*) — `opts.rhythm`, a per-hand onset necklace
  resolved from the bar grid by `reVoiceChords`. Absent ⇒ the historical block
  path (one held chord per event, byte-for-byte).
- **Tone-order** (the *what*) — a per-hand `Figuration` (`opts.figuration`),
  defined in `figuration.ts`. A figuration is a pure
  `(FigurationContext) => StruckTone[]`; the registry (`FIGURATIONS`,
  `findFiguration`, `figurationsForHand`) is the generic collection API consumers
  use — never a named figuration object. `select` returning `[]` is a legitimate
  rest; an unknown id throws in `findFiguration`. Most patterns are a cyclic
  degree-sequence (`cyclicFiguration`); walking bass / harp roll supply `select`
  directly. The §5.4 fallback (a missing stacked-third substitutes the octave
  root) lives in the single degree resolver.

The `ChordEvent` / `Figuration` shapes are the stable contract: when cross-source
voicing selection is wanted, the figuration registry can be promoted to a slot
without changing those types.
