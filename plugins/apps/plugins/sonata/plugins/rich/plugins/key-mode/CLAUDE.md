# key-mode

Per-song key-source mode for Sonata. A song imported from MIDI may carry an
authored key signature (`score.meta.key`), which the score pipeline trusts. This
plugin lets the user **override** that authored key with auto-detection on a
per-song basis: when enabled, the song is treated as keyless, so the shell
re-runs Krumhansl key inference and re-spells notes / re-derives chords from the
inferred key.

## Design

- **State is DB-persisted per song** in `sonata_songs_ext_key_auto_detect` (an
  `entity-extensions` 1:1 side-table; an absent row reads as `false` = trust the
  authored key). A push live-state rollup (`keyAutoDetectResource`) serves all
  rows; the observer does the per-song lookup client-side.
- **The shell owns the in-memory flag**, not this plugin — the score pipeline
  (`inferKeys`) lives in the load-bearing shell, which can't import a feature
  plugin. The shell exposes a module-level store (`key-mode-store.ts`, mirroring
  `cursor-store`); this plugin's headless `KeyModeObserver` (a `Sonata.Effect`)
  reads the open song's persisted setting (gated on the resource resolving) and
  writes it into that store. Dependency arrow stays feature → shell.
- **The toggle UI lives in the key-readout panel** (which imports
  `saveKeyAutoDetect` from here and reads/sets the shell store directly). On
  toggle it sets the shell store optimistically (instant readout/spelling update)
  and calls `saveKeyAutoDetect` to persist; the observer re-affirms the same
  value on the next push.
