# app-cards

Contributes the launcher grid to `Home.Section`: one card per registered app
(read generically via `Apps.App.useContributions()`, excluding the Home app
itself via `useCurrentAppId()`).

The grid is a `DataView<ActiveApp>` (the shared `primitives/data-view` surface),
so it gets search and view-state persistence for free. Each app is one `text`
field (`name`) plus an **icon cover** supplied through the gallery default-card
`cover` producer — no custom card component.

The "New app" affordance is wired via the `creators` prop (a `CreateOption[]`):
the data-view host renders it as a toolbar button and — because
`viewOptions.gallery.showCreateCard: true` is set — also as a trailing dashed
"+" card at the end of the gallery grid. The `onSelect` handler is currently a
stub (no create-app flow exists yet; it will be filled in when one is designed).
`viewOptions` is passed as a plain literal rather than importing the gallery
child, per data-view's collection-consumer separation.
