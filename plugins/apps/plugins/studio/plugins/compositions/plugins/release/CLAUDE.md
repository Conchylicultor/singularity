# release

The release surface for a composition. **A release is a release *of* a
composition**, which is why this lives under `compositions/` rather than beside it.

It contributes two `CompositionDetail.Section`s — **Release** (target picker +
Run) and **Release history** (this composition's runs) — plus the `rel/:runId`
run-detail pane those history rows push, which hosts its own
`ReleaseDetail.Section` slot (info / logs / artifact).

## It used to be a sidebar app

It had a `Studio.Sidebar` entry and a `releasePane` whose first control was a
`Select` over `items.filter(category === "app")` — a second composition picker,
re-asking the question the Compositions list had already answered. Folding it into
the detail pane deleted that dropdown: the composition is the pane's `:id` param,
so only the target remains a choice.

`releaseDetailPane` survived the move unchanged (a run is a genuinely separate
entity); only its `defaultAncestors` was re-pointed at `compositionDetailPane`.
Its segment stays `rel/:runId` — segments are globally unique, and build's
run-detail already owns `r/:runId`.

## Two things worth knowing

- **Only `app` compositions are releasable.** The old Select enforced this by
  omission; now every composition row is reachable, so the section disables Run and
  says why when `category !== "app"` — the other categories are inspection lenses
  with no product to build.
- **The history is composition-scoped and unbounded.** The section is a
  server-delegated DataView over `queryReleaseHistory` (`POST
  /api/release/history/query`) — a keyset-paginated query scoped to this
  composition's runs, kept live by the `release.history-revision` tick. There is
  no 50-run cap and no client-side filter: a composition's full run history is
  browsable via infinite scroll. Each run's detail pane resolves its run by id
  through the `release.run` per-id resource, so an old run's detail resolves
  regardless of age.

The engine itself (`@plugins/release/core` — targets, endpoints, resources) is
top-level and untouched by this; only the Studio UI lives here.
