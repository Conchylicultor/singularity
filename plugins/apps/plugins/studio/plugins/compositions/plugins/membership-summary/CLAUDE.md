# membership-summary

The composition detail pane's headline numbers: how many plugins the active
draft bundles, out of how many exist, plus a badge per membership state.

`useActiveMembership()` re-resolves the draft's closure client-side on every
`updateActiveDraft`, so toggling one contributor moves these counts immediately —
this section is the cheapest read of "what did that toggle cost me". The same
membership map paints the Closure section's tint, so the two never disagree.

`BUNDLE_STATES` (entry / required / contributor / via-contributor) is what counts
as *shipped*; `available` and `excluded` are shown but excluded from the bundle
total.
