# contributors

The composition detail pane's contributor picker: the resolved `available`
frontier as toggle chips, each annotated with what selecting (or pruning) it
would cost in plugins.

The filter query is this section's own `useState`. It was hoisted to the parent
in the pre-section layout only because the editor was a child component of the
one big view; nothing outside this section reads it.

`resolved` is `resolveComposition(graph, flattenManifest(draft, manifests))` —
the draft's `extends` (e.g. a profile's self-improvement pack) must be flattened
against the full registry *before* resolving, mirroring what the store does.
Resolving the unflattened draft would silently under-report the frontier.
