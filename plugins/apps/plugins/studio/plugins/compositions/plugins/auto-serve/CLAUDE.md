# auto-serve

The composition detail pane's **Auto build & serve** section: a single
`ToggleChip` that flips this composition's `autoBuild` config flag, plus the live
serve-URL chip once it is on.

`autoBuild` is engine-opaque config metadata (dropped by `manifestItemToManifest`).
The CLI `compose-serve` stage reads it from **main's** (`singularity`) resolved
config and, for each activated composition, composes a per-composition frontend
dist + empty DB served at `http://<id>.localhost:9000` on every main build. The
toggle writes the `manifests` config via `setAutoBuild(id, on)`; because the stage
only reads main's config, toggling from a non-main worktree is inert until it
lands on main (the documented authoritative-worktree caveat).

The same toggle is mirrored as a trailing cell in the Compositions **list**
DataView — this section is its home in the per-composition detail pane, keyed by
the `comp/:id` route id so the write targets exactly this composition.
