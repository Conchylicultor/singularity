# draft-actions

The composition detail pane's persistence controls: an inline-editable name plus
Save / Delete / Clear for the active draft.

The pane's `:id` param is the composition being edited, so `save(draft, id)` is
always an in-place update — this section can never mint a new item. Creation is
the list pane's "New" button, which writes the config row first and then
navigates here. That is why there is no null-id branch and Delete is
unconditional.

Delete removes the config row, clears the active-composition store, and closes
this pane. `useClose()` returns null only for a root pane; `compositionDetailPane`
declares `defaultAncestors: [compositionsPane]` so it is never root — the
`openPane(compositionsPane, …, { mode: "root" })` fallback exists to make the
expression total, not because it is expected to fire.

Name edits go through `updateActiveDraft`, which patches the module-level store
and re-resolves membership client-side, so the sibling sections (Summary,
Closure) update as you type — before anything is saved.
