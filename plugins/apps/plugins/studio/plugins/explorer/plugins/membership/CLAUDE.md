# membership

Tints each explorer tree row by the inspected plugin's **membership state** in
the currently active composition (the working draft held by
`composition/web`'s active-composition store).

Contributes a `MembershipBand` into `Explorer.TreeRowBadge`. The band reads
`useActiveMembership()`:

- **No active composition → renders nothing** (no tint by default).
- Otherwise it looks up `membership.get(node.id)` and paints a translucent
  full-row band (`pointer-events-none absolute inset-0`) tinted by state:
  `entry` / `required` strong primary, `contributor` / `via-contributor` success,
  `available` faint info, `excluded` no band. The band is translucent so the row
  text stays legible and the selected `bg-accent` blends through.

## Compare color mode (Increment 3)

When the store is in **compare mode** (`useDiffMap()` is non-null — both an active
and a compareWith composition are set), the band tints by **diff state** instead
of membership: `only-a` / `only-b` / `both` / `neither`. The diff palette is the
themeable **categorical** scale (`bg-categorical-4/7/2`), deliberately distinct
from the single-composition `primary` / `success` / `info` membership tints so the
two modes are never confusable. `neither` gets no band. Diff mode takes precedence
over membership when active.

`DIFF_TINT` (state → tint class) and `DIFF_LEGEND` (ordered legend rows) are
exported from this plugin's barrel so the Compositions pane's diff legend and
delta chips reuse the **exact same colors** the tree paints — single source of
truth, no drift.

On row hover it also surfaces a small **"Show closure from here"** pin button
calling `pinAsRoot(node.id)` — pin any plugin and the whole tree re-tints by its
closure.

The membership map is derived **once** per (active, graph) change by the store,
so this per-row component only does a `Map.get`. The band relies on the `relative`
class on the explorer's `TreeRow` root (the single shared-code change this feature
makes).
