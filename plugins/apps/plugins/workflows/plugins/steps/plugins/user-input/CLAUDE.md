# user-input

Human-in-the-loop workflow step. At runtime the executor calls `ctx.waitFor` on
the engine's `userInputSubmitted` trigger event, which **suspends** the durable
run (the engine's run-job catches the suspend signal and marks the step/execution
`suspended`). The execution trace renders the configured prompt plus one input
per field; submitting POSTs to the engine's `submitStep` endpoint, which emits the
event. The waiting executor resumes and returns the submitted data as the step's
`output`.

Config: `{ prompt?: string; fields?: { name: string; label?: string }[] }`. An
empty `fields` list collects a single field named `"value"`.
