# http-request

HTTP-request step type for workflows. The executor interpolates the configured
`url` and `body` against the previous step's output (`{{ path }}` tokens),
parses the `headers` block (one `Key: Value` per line, split on the first
colon), and performs an SSRF-safe outbound call via `safeFetch`. The response is
emitted as the step's `output` — `{ status, ok, headers, body }`, where `body`
is parsed JSON when the response is JSON (or parseable), else the raw text — so
a downstream `branch` can route on it.

A non-2xx status is a valid, routable result and is returned, not thrown.
Network, timeout, and `SsrfError` failures propagate and fail the step loudly.

Config: `{ method?: string; url?: string; headers?: string; body?: string }`
(method defaults to `GET`; body is sent only for non-GET/HEAD methods).
