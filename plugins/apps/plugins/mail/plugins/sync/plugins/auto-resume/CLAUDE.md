# auto-resume

Auto-resumes a stuck Mail sync the instant the Gmail scope is (re)granted. When
sync hits a terminal auth error the account parks in `status = "error"` and the
main-only delta tick intentionally skips it, so nothing re-arms the sync after
the user reconnects Google — until now.

This is **web-only** by necessity: the reconnect/reconsent signal is pushed from
the central auth runtime to browsers alone (a worktree sync job has no
subscription path to it), so the only place to observe the edge is the web
runtime. A headless `Core.Root` listener watches `useGmailAccess().ready` and, on
its `false → true` edge, POSTs the existing `mailSyncEndpoint`
(`POST /api/mail/sync`) — reusing `kickSync` verbatim, with no new resume logic
and no import of `@plugins/auth/*` (the Gmail integration brokers the signal).
