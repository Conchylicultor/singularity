# mail-html

`<MailHtml>` ‚Äî the **privacy-safe email HTML renderer** used by the reading
pane. Email HTML is hostile third-party markup: it can carry scripts, tracking
pixels, and layout that fights the app. This component runs it through a fixed
four-stage pipeline before injecting it, and each stage exists for a reason.

```tsx
<MailHtml
  html={message.bodyHtml}
  showRemoteImages={userOptedIn}
  onRemoteImagesDetected={(present) => setHasRemote(present)}
  resolveCid={(cid) => inlineAttachmentUrl(cid)}  // undefined = not yet hydrated
/>
```

## Pipeline (`web/internal/process-email-html.ts`)

1. **Sanitize (DOMPurify).** Removes `<script>`, event-handler attributes
   (`onclick`, ‚Ä¶), `javascript:`/`vbscript:` URLs, and active/embedding elements
   (`<iframe>`/`<object>`/`<embed>`/`<form>`/`<style>` ‚Ä¶). `data:` URIs are
   blocked at the URI-allowlist level (defense-in-depth against
   `data:image/svg+xml`). **Why:** the message can be actively malicious ‚Äî this
   is the XSS boundary. `srcset` is forbidden so a responsive candidate can't
   smuggle an un-gated remote URL past the `src`-only image walk.
2. **Image gate.** For every `<img>` and inline `style` `url(http‚Ä¶)`:
   - `cid:` ‚Üí `resolveCid(bareContentId)`; sets the src if the inline attachment
     is available, else leaves a `data-cid` placeholder (the body re-processes
     when `resolveCid` changes, so it fills in as attachments hydrate).
   - remote `http(s)` ‚Üí **only** rewritten to the same-origin, SSRF-guarded proxy
     (`remote-images`) **when `showRemoteImages` is true**; otherwise the src is
     stripped and recorded. **Why:** a remote image is a tracking pixel ‚Äî loading
     it leaks the reader's IP, location, and open-time to the sender. Nothing
     remote loads until the user explicitly opts in per message. After the walk,
     `onRemoteImagesDetected(hadRemoteImage)` tells the pane whether to offer the
     "Display images" affordance.
   - `data:` images are dropped (belt-and-suspenders).
   - Links are forced to `target="_blank"` `rel="noopener noreferrer nofollow"`.
3. **Quoted-history collapse.** Detects the first reply boundary ‚Äî a
   `.gmail_quote`, a top-level `<blockquote>`, or a divider line ("On ‚Ä¶ wrote:",
   "-----Original Message-----", an Outlook "From:/Sent:" header block) ‚Äî and
   moves it plus following siblings behind a collapsed `‚Ä¢‚Ä¢‚Ä¢` toggle. **Why:**
   long quoted chains bury the actual reply; collapsing by default matches every
   webmail client.
4. **Scoped render.** The result is injected via `dangerouslySetInnerHTML` inside
   `.mail-html-body`, a self-contained style scope: images `max-width:100%`, a
   readable base font, and `overflow-x:auto` so wide tables scroll **inside their
   own box** and can never break the app layout.

## Testing

Pure string heuristics (`cid`, `remote-url`, `quote-boundary`) have a co-located
`bun:test` (`web/internal/helpers.test.ts`). The DOM-bound behavior ‚Äî DOMPurify
stripping, image gating, cid resolution, quote collapse ‚Äî is covered by the
jsdom test in `web/__tests__/mail-html.test.tsx` (DOMPurify needs a DOM).

Depends on `dompurify` (declared in `package.json`).
