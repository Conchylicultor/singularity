# inbox

The Mail app's inbox rendered as a **standard `data-view`** (the Notion-like
multi-view data surface), rather than the hand-rolled `thread-list`/`mailbox`
pair. A server-delegated `DataView` over `mail_threads` for the default mailbox
(the Gmail **INBOX** = "is not archived"), rendered as a Gmail-style list.

It sits **alongside** the current `thread-list`/`mailbox` (nothing removed yet);
a future task retires the deprecated views once this is validated.

## Server

- **`queryInbox`** (`POST /api/mail/inbox/query`) — one keyset page of INBOX
  threads. Near-verbatim of `all-conversations`' handler: resolves the single
  account, then a **server-fixed** scope `label_ids @> '["INBOX"]'` (via
  mail-core's tested `mailViewFilterSql(MAIL_SYSTEM_VIEWS[0].filter)`, the inbox
  system view) AND the compiled `FilterGroup` AND an ilike search over
  subject/snippet AND the null-aware keyset seek. The INBOX predicate is fixed,
  not a removable filter chip — a user must not be able to pull spam/trash/sent
  into "Inbox". Ordered by the requested `sort` with a `mail_threads.id` PK
  tiebreaker so the seek is strict.
- **`inboxRevisionResource`** — an independent coarse `${count}:${maxUpdatedMs}`
  revision tick over `mail_threads` (`mode:"push"`, scoped to `mail_threads`).
  **Distinct** from thread-list's `mail-threads-revision` — the two plugins do
  not couple (thread-list is slated for removal). On any thread write the
  DataView refetches its loaded window in place (no scroll reset).

## Web

- **`inboxPane`** (`segment: "mailbox"`, `/mail/mailbox`) — a `<DataView>` over
  `MailThread` with `views={["list"]}`, a rich two-line `InboxRow` via
  `viewOptions.list.renderRow`, and a leading star. `onRowActivate` opens the
  reading pane (`threadPane`) as a pushed Miller column. The paginated SQL query
  is the source of truth; the revision tick drives the in-place refetch.
- The field schema (`MAIL_INBOX_FIELDS`) powers the toolbar Sort pill (Subject /
  Date / Messages) and Filter pill (Unread / Starred / Important / Attachment).
  `sender`/`snippet` are display-only (inside `renderRow`), not fields — server
  search covers subject/snippet via ilike.

## Reachability

A new **Inbox** entry in the `Mail.Sidebar` slot, and bare `/mail` lands here
(the mail shell navigates to `/mail/mailbox` by URL — never importing `inboxPane`
— so `inbox → shell` stays one-way and acyclic).

## Boundaries

Consumes only barrels: `mail-core` (schema/types, `mailViewFilterSql`,
`MAIL_SYSTEM_VIEWS`, `resolveMailAccountId`, `_mailThreads`), the mail `shell`
(`Mail` slot), `reading-pane/web` (`threadPane`), and the data-view /
server-query / endpoint / live-state / pane primitives. It must **never** be
imported by the mail shell (that would cycle) — the landing repoint is by route
string only.
