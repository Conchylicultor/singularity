# gmail-api

A **stateless, typed Gmail REST API v1 client**. Every function takes an access
`token: string` as its first parameter and returns the raw Gmail JSON shape
(`core/internal/types.ts`). It calls `https://gmail.googleapis.com/gmail/v1/users/me/...`
directly and nothing else — no DB, no auth import, no per-account state.

## Why separate from `mail-core`

`mail-core` owns the persisted mailbox mirror and the `requireGmailToken()`
auth-wiring helper. This plugin is the opposite half: the pure transport. Keeping
it token-in / JSON-out means it never imports `@plugins/auth/*` or
`@plugins/integrations/plugins/gmail/*`, so there is **no import cycle** — the
sync engine resolves a token (via `mail-core`) and hands it to these functions.
It is also reusable beyond sync: compose/send and label mutations will land here
later behind the same token-param contract.

## Backoff & quota ("batched")

Google's JSON multipart batch endpoint is deprecated, so `batchGetMessages` is a
**concurrency-bounded parallel fan-out** (`createSemaphore(8)` from
`@plugins/packages/plugins/semaphore/core`) rather than a single batch request,
preserving input order via `Promise.all`. Every request flows through one
`gmailRequest` helper that wraps the fetch in `retryUntil`
(`@plugins/packages/plugins/retry/core`) with `withJitter(exponential(...))`
backoff and a 120s deadline. Retry classification:

- 2xx → return the parsed object (success).
- 429, 403 with a rate-limit `reason`
  (`rateLimitExceeded` / `userRateLimitExceeded` / `backendError`), or 5xx →
  return `null` so `retryUntil` backs off and retries.
- Any other non-ok status → throw, propagating immediately (no retry).

## Typed errors (`core/internal/errors.ts`)

- `GmailHistoryExpiredError` — `history.list` returned 404 (stale `historyId`);
  the caller must drop its watermark and full-resync. Thrown only on the history
  endpoint (`historyEndpoint: true`).
- `GmailApiError` — a non-retryable API failure carrying `status` and the
  parsed Gmail `reason`.
