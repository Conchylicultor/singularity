# mail

A Gmail-class mail client, delivered as a top-level Singularity app at `/mail`.

This top-level plugin is an empty namespace only (no slots, no logic). All
content lives in sub-plugins:

- `shell` — the `/mail` app entry, rail icon, layout, and the capability-driven
  landing pane.
- `mail-core` — the persisted mail data model (accounts, threads, messages,
  labels, attachments, drafts, sync-state, outbox) plus the Gmail token-wiring
  helper.

## Phase roadmap

See `research/2026-06-29-apps-gmail-client.md` for the full plan. This is
**phase 1**: app scaffold + data model + token wiring. Sync, the inbox list,
compose, triage, and search land in later phases.

## Auth

Mail **consumes** the Gmail integration
(`@plugins/integrations/plugins/gmail/{web,server}`) for connection state and
access tokens. It never re-implements OAuth and never imports `@plugins/auth/*`
directly — all Gmail auth flows through the integration's public API.
