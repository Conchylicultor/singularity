# exports

Extracts barrel exports metadata for all runtimes (core/web/server/central/shared):
per-symbol name + kind (type | value). Uses `relate()` to compute `consumers[]` for
each exported symbol by inverting the `apiUses` data already on `PluginNode` — the
same inversion that `buildSymbolConsumers()` in `plugin-view/tree-handler.ts` performs
at request time.

Dual-write step: `node.facets["exports"]` is populated alongside the existing monolithic
`node.exports` field. Doc output is byte-identical to the `renderExportsAt()` path in
`codegen/docgen.ts`. The `consumers[]` per symbol is bonus metadata not yet emitted in
`renderDoc()`, available for future consumers (e.g. `tree-handler.ts` in Step 5).
