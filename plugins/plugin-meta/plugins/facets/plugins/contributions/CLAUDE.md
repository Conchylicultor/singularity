# contributions

Extracts runtime slot contributions from each plugin's barrel imports. Reads
`def.contributions[]` from the module default export across all runtimes (in
practice only web barrels carry contributions).

Unlike static-analysis facets (commands, slots, resources), this facet needs the barrel-imported
modules — it reads the `importedModules` field of `ExtractContext`, which is populated by Pass 1
of `enrichPluginTreeDocs()`.

`extract()` collects `{ slotId, componentName, doc }` without display names.
`relate()` fills in `slotDisplayName` by reading `node.slots` (the monolithic
field, always populated before facets run) — `${groupName}.${memberName}` from
each `SlotDef`.

`renderDoc()` is implemented but not yet called (docgen still reads the monolithic
`node.runtimeContributions` until Step 5 of the facet migration).
