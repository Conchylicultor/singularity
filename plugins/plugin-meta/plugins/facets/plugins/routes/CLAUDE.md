# routes

Extracts HTTP routes (from `defineEndpoint()` calls in each plugin's `core/`) and WS
routes (literal keys in `server/` and `central/` barrels). Fixes the blindness of the
monolithic `parseRouteMap` which only captures quoted string keys — missing all
`[endpoint.route]` computed keys used by the `defineEndpoint` pattern.

`relate()` rewrites `node.endpointCallers` with accurate data (the monolithic
`apiPrefixToOwner` block in `buildPluginTree` operates on broken `httpRoutes` data,
so this fully replaces its result).

`renderDoc()` is implemented but not yet wired into `docgen.ts` — that's Step 5 of
the unified facet docgen plan. `node.server.httpRoutes` is still read by consumers
until Step 5.
