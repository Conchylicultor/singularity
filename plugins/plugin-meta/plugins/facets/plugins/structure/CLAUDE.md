# structure

Extracts each plugin's on-disk structure: its immediate sub-folders (tagged
`standard` against the repo's recognized folder set), stray top-level
`.ts`/`.tsx` source files, and whether it self-declares a composition root
(`package.json` `singularity.compositionRoot`).

`extract()` reads the plugin directory directly; the standard-folder set is
resolved once at module load via top-level await (`standardPluginDirs` is
async by signature only — it does pure synchronous fs reads). No `relate()` —
structure is purely per-plugin. `renderDoc()` surfaces only anomalies
(non-standard folders, loose files, composition root) into the plugin doc.

Single source of truth for structure metadata; the per-plugin detail section,
the Forge catalog "Structure" tab, and the PR diff renderer all read
`node.facets["structure"]`.
