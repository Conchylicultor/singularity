# db-schema

Extracts DB schema metadata per plugin: schema file paths (`dbFiles`), table definitions (`tables`), entity extensions (`entityExtensions`), and the reverse `extendedBy` cross-references.

`extract()` walks each plugin's `server/` directory to find schema files and parse `pgTable()` declarations. `relate()` resolves `defineExtension()` calls across the full plugin tree to populate `entityExtensions` on the extending plugin and `extendedBy` on the parent plugin.

Part of the unified facet-based docgen pipeline. See `research/2026-05-20-global-unified-facet-docgen.md`.
