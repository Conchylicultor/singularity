# dependencies

Adds two **deduped cargo-tree-style dependency trees** to the plugin detail pane
(`PluginViewSlots.Section`):

- **Depends on** — the inspected plugin's recursive forward dependencies (its footprint).
- **Used by** — the recursive reverse dependents (its blast radius / impact).

Both faces share one mechanism. The full bidirectional graph is already live
client-side via `useGraph()` from `@plugins/plugin-meta/plugins/composition/web`
(`useEnsureCompositionData()` populates it). The pure builder
`web/internal/build-dep-tree.ts` walks the four `EdgeGraph` adjacency maps and
renders a spanning tree where the **first** occurrence of a plugin expands fully and
every later occurrence is a marked `↑` leaf — collapsing DAG diamonds and breaking
any theoretical cycle while preserving a linear reading order (the `cargo tree` /
`npm ls` strategy). Both hard imports and soft slot-contributions are included; soft
rows carry a subtle `soft` badge. Clicking a row swaps the pane to that plugin.

Pure presentation: the builder is covered by `build-dep-tree.test.ts` (bun:test);
the renderer mirrors the sibling `sub-plugins` section's recursive row.
