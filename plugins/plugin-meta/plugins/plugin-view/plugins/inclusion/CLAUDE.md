# inclusion

Adds a **"Composition membership"** section to the plugin detail pane
(`PluginViewSlots.Section`), explaining the inspected plugin's relationship to the
currently active composition.

- **No active composition** → a muted placeholder plus a **"Show closure from
  here"** button (`pinAsRoot(node.id)`), so any plugin can become the closure root.
- **Active composition** → the membership-state badge (`entry` / `required` /
  `contributor` / `via-contributor` / `available` / `excluded`), the **why-included**
  path from `useInclusion()` rendered as hard/soft edge chips (origin → … → target,
  each node clickable to re-pin), and the **select / prune impact** counts + id
  lists from `useImpact()`.

All composition state comes from `composition/web` (the active-composition store +
the `useInclusion` / `useImpact` engine wrappers); this plugin is pure
presentation built from the `link-chip` / `badge` / `spacing` / `text` primitives.
