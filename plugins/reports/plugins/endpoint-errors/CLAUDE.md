# endpoint-errors

Files crash tasks for **bug-shaped** handled endpoint errors so agents can
investigate them — not just transient toasts.

Registers a handler on `endpointErrorSink` (the `endpoints` primitive's
`defineReportSink` inversion hook, the twin of `error-boundary`'s
`boundaryReportSink`). The primitive fires for every non-ok response; this
reporter filters to bug-shaped errors and calls `report()`:

- **File a crash for:** any `status >= 500`, and `400` responses whose body is a
  schema validation failure (`{ error: "Validation failed" | "Query validation
  failed", issues }`).
- **Skip:** 401 / 403 / 404 / 409 and non-validation 400s (legitimate control
  flow — e.g. the block editor's plain-text "no previous sibling to merge" 400).

Dedup: the crash fingerprint is `sha256(errorType + stack)`, and every
`EndpointError` shares the same throw-site stack — so all distinguishing info is
encoded into `errorType` (`EndpointError <status> <route> (<sorted issue
paths>)`). Repeated identical errors collapse to one task and bump its count.
