# noise-rules

Built-in noise classification rules for the crashes plugin.

Some crashes are structurally benign and high-frequency — the canonical example
is the `ResizeObserver loop completed with undelivered notifications` browser
warning. These are recorded and visible like any other crash, but classified as
"noise" so they don't bump the notification badge or pop a toast.

This plugin contributes one `CrashNoiseRule` per known low-signal pattern via
the generic `CrashNoiseRule` slot owned by the parent `crashes` plugin.
`recordCrash` enumerates all rules and stamps a `noise` boolean on the crash row
— it never names a specific rule.

## How to add a rule

Add another `CrashNoiseRule({ id, matches })` entry to the `contributions` array
in `server/index.ts`:

```ts
CrashNoiseRule({
  id: "my-benign-pattern",
  matches: ({ message, errorType, source, stack }) =>
    message.includes("…"),
}),
```

`matches` receives the crash's `source`, `errorType`, `message`, and `stack`.
Return `true` to mark the crash as noise. A thrown error in `matches` is caught
and treated as "no match", so a buggy rule never breaks the crash pipeline.
