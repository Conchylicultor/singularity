# editor-collab

The **content-agnostic** server side of per-block collaborative text ŌĆö the
only text pipeline (see the editor's ┬¦ "Per-block CRDT text (unconditional)").
It merges and stores opaque Yjs update bytes and broadcasts state; it never
interprets doc content (no runs, no Lexical, no decorator tokens; those live in
the editor's web runtime, so only a client can build a decorator-correct doc).
Originated as Plan B (`research/2026-07-07-page-per-block-crdt-plan-b.md`).

## Model

- `page_block_docs(block_id pk ŌåÆ page_blocks.id ON DELETE CASCADE, state bytea,
  updated_at)` ŌĆö one compacted `Y.encodeStateAsUpdate(doc)` per text block.
  Deliberately in the change-feed: the `doc-update` UPDATE is what pushes the
  live resource.
- `blockContentResource` (`page-block-doc`, keyed, params `{ blockId }`) ŌĆö a
  0-or-1-element array of `{ blockId, state: base64, updatedAt }`. Param-scoped
  to ONE block so only mounted editors subscribe (lazy content loading);
  `identityTable: "page_block_docs"` scopes recompute to the changed row.
  Hand-written loader (not `queryResource`): base64 must be encoded in JS ŌĆö
  Postgres `encode(ŌĆ”, 'base64')` folds lines at 76 chars, which would corrupt
  large states ŌĆö and `stateToBase64` is the single shared encoder with the
  doc-init response.
- `POST /api/blocks/:id/doc-init` (body: raw update bytes via `blob()`) ŌĆö the
  first-writer-wins seed and the ONLY row creator (`ON CONFLICT DO NOTHING`,
  then return the authoritative state as base64 JSON). A losing seeder merges
  the winner's state instead of duplicating text ŌĆö this closes the Yjs
  two-clients-seed-the-same-content hazard at one server chokepoint.
  404 when the block row doesn't exist (the `page_blocks` FK precondition,
  mapped from SQLSTATE 23503 instead of an opaque 500). Clients gate doc-init
  on the block being server-confirmed (Stage 4a: the provider's
  `markBlockRowConfirmed`), so a 404 in practice means the block was deleted
  concurrently ŌĆö callers drop the seed quietly, like the doc-update 409.
- `POST /api/blocks/:id/doc-update` (body: raw incremental update) ŌĆö CRDT merge
  under `SELECT ŌĆ” FOR UPDATE`; 409 if uninitialized (never auto-seeds).
