# cover

Links a page's **cover image** to its page block so the attachments orphan-sweep
never reclaims it.

A page stores its cover at the nested `data.cover.attachmentId` (see
`PageCoverSchema` in `page/editor/core`), which is outside the flat
`data.attachmentId` convention that `attachment-block` reconciles by default.
This plugin contributes a single `AttachmentBlock.Collector` that surfaces the
cover's attachment id, so the shared block↔attachment reconcile links it like
any other attachment-owning block. Gradient covers reference no attachment and
need no linking.

Server-only; the cover UI lives in the Pages app (`apps/pages/page-tree`).
