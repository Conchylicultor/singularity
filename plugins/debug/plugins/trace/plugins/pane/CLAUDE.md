# pane

**Debug → Slow Events** — one sidebar entry for all slowness. The root pane
(`/debug/traces`, bound to the engine's `traceListRoute`) is a slot-backed tab
host (`defineTabbedView` → `SlowEvents.View`) that owns no list logic of its own.
Three tabs register into the slot:

- **Events** (this plugin) — the durable trace list: a `DataView` over
  `GET /api/traces` (metadata only, hydrate-on-open + Refresh, no live resource —
  a trace is written exactly when the system is slow, so a change-feed push per
  write would pile on at the worst moment). Row activate opens the detail Gantt.
- **Aggregates** (`debug/slow-ops/pane`) — the ranked per-op slow_ops table.
- **Cluster** (`debug/slow-ops/cluster`) — the cross-worktree fan-out.

Per the collection-consumer rule the pane never imports the Aggregates/Cluster
components — slow-ops imports `SlowEvents` (the slot token) and contributes into
it; nothing in `trace/` imports slow-ops.

## Detail Gantt (`/debug/traces/x/:id`)

`GET /api/traces/:id` (404 → graceful not-found). One `GanttContainer`
(`totalMs = atMs − windowStartMs`) hosting, top to bottom: the **trip row**
(rendered here generically from `snapshot.trigger` — the pane names no class),
then one `<Trace.Lane.Dispatch>` per `snapshot.events` key **in insertion
(registry) order** — so `spans` (per-kind span lanes), `gates` (occupancy strip),
`contention` (footer card), and any future class each paint themselves. A
bar/chip click feeds the shared bottom **sticky detail strip** via the engine's
generic `TraceLaneProps.onSelect` channel — a class-agnostic `{ title, fields }`
descriptor the strip renders without knowing what produced it. Header: the
`Trace.TriggerSummary.Dispatch`.
