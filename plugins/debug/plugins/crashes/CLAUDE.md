# crashes

Debug app pane listing every recorded crash, including low-signal "noise" crashes
(e.g. ResizeObserver loop warnings) that are quieted in the notification bell but
never hidden here.

Opened from the Debug sidebar ("Crashes"). Reads the web-safe `crashesResource`
live-state handle from `@plugins/crashes/core`, so the list updates in real time as
new crashes are recorded.

Each row shows:

- the crash **source** (badge),
- a **noise** badge when the crash matched a built-in noise rule,
- a **loop** badge when the crash is part of a crash loop,
- a **×count** dedup multiplier when the same fingerprint recurred,
- the **last-seen** relative time,
- a **task →** link to the crash's filed task (`/tasks/t/<taskId>`) when present,
- and the error line (`errorType: message`, or just `message`), truncated.

All crashes are listed — noise is surfaced via the badge, never filtered out.
