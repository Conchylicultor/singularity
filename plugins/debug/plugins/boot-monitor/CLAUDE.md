# boot-monitor

Whole-boot monitor — the producer that makes a **slow server boot** a first-class
slow event: one `slow_ops` row (`operationKind: "boot"`, `operation:
"server-boot"`) plus one coherent trace (`kind: "boot"`, section validated by
`debug/trace/boot`) per over-budget boot. Follows the one-signal-one-plugin
debug-monitor precedent (`stall-monitor`, `read-set-shrink`); the job copies
`boot-budget`'s shape (per-worktree minute cron, `dedup: "singleton"`,
`maxAttempts: 3`, module-level per-boot dedup — process lifetime = boot epoch).

**Division of labor vs `boot-budget`:** boot-budget answers "*which hook* is
heavy" (one deduped report per over-budget span, per-phase budgets); this
monitor answers "was *this boot* slow as a whole" (one aggregate row + trace,
one `totalBootBudgetMs`). Neither replaces Debug → Boot Profile, the always-on
deep-dive — this is **trip-only**: within-budget boots mint nothing.

Key mechanics:

- **Boot-completeness guard.** `profilerStart` pushes a span only when its
  `end()` closure runs, so a `phase === "drainWarmups"` span in
  `getProfilingData().spans` is a deterministic "boot fully complete" signal;
  ticks are skipped (not marked) until it appears.
- **Evidence-first.** `captureTrace` runs before `recordSlowOp` (the documented
  contract in `record-slow-op.ts`), so the row/report carry the trace id.
- **Gateway handoff.** `POST /api/boot/gateway-report` (body =
  `BootGatewaySchema` from `trace/boot/core`, all fields optional for version
  skew; no auth — the gateway dials the unix socket directly, the same trust
  surface as `/api/health/ready`) stores the gateway's spawn/readiness stamps in
  a module-level box the job folds into the section at mint time. Ordering holds
  by construction: the POST lands ~100 ms after the proxy swap, the mint at the
  next minute tick. No POST (older/unrestarted gateway) → `section.gateway` is
  simply absent.
- The trip metric is the profiler's own `totalDurationMs` — deterministic and
  gateway-version-independent; the gateway wait is section *evidence*, never the
  trip metric.
