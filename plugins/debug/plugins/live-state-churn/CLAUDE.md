# live-state-churn

Umbrella for live-state churn debugging — the two halves of working with no-op
(empty-diff) live-state pushes:

- **`monitor`** — the *detect* half: an in-process accumulator fed by every keyed
  live-state push, plus a scheduled job that files `live-state-noop` reports for
  resources sustaining a high no-op rate, surfaced in Debug → Reports.
- **`emit`** — _(forthcoming)_ the *emit* half: a debug affordance that generates
  synthetic no-op pushes on demand so churn-driven render/DOM bugs reproduce
  deterministically.

The parent is a pure container (no core/web/server barrels); each child owns its
own barrels.
