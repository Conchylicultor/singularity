# boot-profile

The **Boot Profile** Debug page renders the real browser boot timeline as a
Gantt, reusing the Gantt primitives from `debug/profiling`. It reads the
per-page-load trace from the `primitives/perfs/boot-trace` client store
(`getBootTrace()`) and decomposes boot into these phases:

- `navigation` — Navigation Timing, decomposed across the 0→first-byte window:
  unload/redirect, DNS, TCP connect (each shown only when non-trivial),
  request→first-byte (TTFB), response download, DOM interactive. This attributes
  a slow **document** response (cold/restarting backend, contention) to the right
  sub-phase instead of leaving a blank gap before TTFB.
- `scripts` — module eval, plugin load, createRoot.
- `main-thread` — Long Tasks (≥50ms): the bundle parse/compile/eval +
  plugin-chunk fan-out + first React render that block the thread between
  bytes-arrived and first paint. This fills the historical **0→first-span blind
  spot** — JS instrumentation can't run while the thread is blocked, so these
  come from the Long Tasks API. Clipped to the boot window (≤ FCP / first commit).
- `assets` — scripts/stylesheets from Resource Timing, biggest first, as a
  **request** (wait) + **download** (work) split via `WaitWorkRow`. Capped to the
  top-N by transfer size; the remainder rolls up into one summary row (count +
  bytes) so nothing is silently dropped.
- `boot-tasks` — boot tasks + the boot-snapshot fetch.
- `resources` — each boot-critical resource shipped in the boot snapshot,
  rendered with a **wait** segment (network + queue) and a solid **work**
  segment (server loader/read time) via the custom `WaitWorkRow`.
- `paint` — first-paint, first-contentful-paint, first React commit.

A summary strip surfaces two tables: **milestones** (TTFB, response end, plugins
loaded, boot tasks done, first React commit, first-contentful-paint) and **boot
cost** (JS shipped + chunk count, biggest chunk, main-thread busy before paint) —
the levers behind the pre-paint window. The trace is ephemeral (current tab's
boot); **Reload & re-measure** re-runs boot, **Refresh** re-reads the store.
