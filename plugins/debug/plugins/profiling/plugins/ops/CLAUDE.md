# ops

Op contention profiling sub-plugin for Debug > Profiling. It owns the **endpoints
and the Gantt** over the ops the [`op-log`](../op-log/CLAUDE.md) plugin records —
it no longer reads any file itself. `readOpRecords()` merges `op-log.jsonl` with
the two frozen legacy files and returns one `OpRecord[]` covering all three kinds
(`build`, `push`, `check`); this plugin windows, groups, and projects it onto the
wire.

## The render model

An op is **one bar** spanning `startMs → startMs + totalMs`, colored by `kind`,
with each entry of `waits[]` painted as an overlay segment at its own true offset
*inside* that span.

This is deliberately **not** the old `[wait][hold]` head-to-tail split. Waits are
interleaved with real work, so a build reads as
`[build-lock][…migrations/codegen…][duress-valve][host-grant][…heavy work…]`, and
one op may carry several waits of the same kind (a build re-queues for the host
grant across requeue cycles). A renderer must handle N segments at arbitrary
offsets and must not assume they tile the bar.

That interleaving is also why an op's span is `requestedAt → requestedAt +
totalMs` and **never** `waitMs + holdMs`: that sum omits every work gap between
the waits, and using it silently truncates the Gantt's time axis.

## Grouping

`canonicalWorktree()` is load-bearing: ops for one worktree carry *different*
identifiers — builds log the basename (`att-x`), while pushes fall back to the
branch (`claude-web/att-x`) when `SINGULARITY_WORKTREE` is unset for the push CLI.
Canonicalizing to the bare basename is what puts a worktree's push, build, and
check bars on a single row instead of several.

Row labels resolve through `resolveWorktreeTitles()` off the worktree id (= the
attempt id, by the basename invariant), so build-only rows — which carry no
`conversationId` — still get a human title.

## Reconciler

The `onReady` hook calls `finalizeOrphanedOps(isWorktreeOpActive)` — **one**
reconciler for all three kinds, replacing the two near-identical ones this plugin
used to own. `isMain()`-gated: the op log is a single global file, so gating stops
every worktree backend from writing it concurrently.
