# slow-ops

Debug app pane showing a **global, ranked overview** of slow operations — the
aggregate counterpart to the per-crash list in Debug → Reports. Slow ops are
metrics that often hide a *structural* issue (several slow routes bottlenecked on
one query), so this view ranks them by **aggregate impact** (total time across all
occurrences) rather than scattering them.

Opened from the Debug sidebar ("Slow Ops"). Reads the web-safe `slowOpsResource`
live-state handle from `@plugins/slow-ops/core`, so the table updates in real time
as new slow operations are recorded.

Each row shows the operation (its `operationKind` badge + operation label), the
occurrence **count**, the **total**, **max**, and **last** durations (ms), and the
**last-seen** relative time. Beneath the operation, the per-operation **caller
breakdown** is rendered inline (`↳ kind:label ×count (total ms)`, sorted by caller
total time) — the attribution showing which request/loader issued the slow op.

Default sort is by **total time descending**; every numeric column is sortable.
