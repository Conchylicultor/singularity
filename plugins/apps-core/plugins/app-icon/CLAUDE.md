# app-icon

Canonical, serializable identity of an app's icon. One descriptor authored once
per app, rendered everywhere an app's glyph appears (rail, tab bar, window
chrome, launcher).

## `AppIcon` ‚Äî the serializable descriptor

`AppIcon` (in `core/`) is a plain, JSON-serializable discriminated union:

```ts
type AppIcon = { kind: "md"; svgNodes: SvgNode[] };
```

It carries the icon's extracted `SvgNode` tree, not a React component ‚Äî so a
non-React consumer (a generated favicon, a Tauri window-icon, a server-side
render) can read the same value the UI does. The `kind` discriminant is the
seam: a future `{ kind: "image"; src }` variant drops in with one extra render
branch and zero changes to existing `kind: "md"` authors.

## Composes `icon-picker`

The `SvgNode` storage format, the extraction from a react-icons component, and
the raw-`<svg>` renderer all live in the [`icon-picker`](../../../primitives/plugins/icon-picker/CLAUDE.md)
primitive ‚Äî the same primitive `avatar` composes. `app-icon` adds only the
app-level descriptor and the author-time helper on top.

## Authoring

- `mdAppIcon(Icon)` ‚Äî build an `AppIcon` from a tree-shaken react-icons
  component: `icon: mdAppIcon(MdHome)`. Only the picked icon's SVG ships, never
  the full react-icons bundle.
- `DEFAULT_APP_ICON` ‚Äî fallback glyph for tabs/windows whose owning app cannot
  be resolved.

## Two render forms

- `<AppIconView icon className />` ‚Äî direct render of an `AppIcon` to raw
  `<svg>`. Use this wherever you hold an `AppIcon` and render it inline.
- `appIconComponent(icon)` ‚Äî adapter for generic icon-prop boundaries whose
  `icon` prop is typed `ComponentType<{ className? }>` (e.g. the `ui/tab-bar`
  `Tab` primitive, `IconButton`). Memoized on the stable `AppIcon` object so the
  returned component identity is stable (no per-render remount).
