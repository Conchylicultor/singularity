# engine

## Retention

`entity_versions` is an unbounded-growth firehose — one row per edit of a
versioned entity, coalesced per ~10-min session window but never bounded over
time, and with no FK to any consumer (see `internal/retention.ts`). A 30-day TTL
sweep (`entityVersionsRetention`, `perWorktree`, nightly) reclaims it, matching
Notion's free-tier page-history semantics.

- **Pinned rows are swept too.** The sweep is unscoped: the only pinned rows are
  the machine-created "Before restore" undo points, which are no more precious
  than an auto-snapshot. Sparing them would leave one immortal row per restore
  click and keep the table unbounded. A future user-curated "named version"
  feature should protect off a new `user_created` flag, not off `pinned`.
- **Accepted semantic:** a page untouched for >30 days loses its entire timeline
  — the version dialog goes empty and restore is impossible. Intended: versions
  are only useful as *past* states, and nothing is lost that the live entity
  doesn't still hold.
