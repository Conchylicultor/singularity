# Redline — UX Design

AI-powered architectural drawing tool. An agent models buildings in 3D using CAD primitives. The architect reviews 2D/3D views and gives feedback by drawing color-coded annotations. The agent interprets feedback and iterates.

---

## Core Loop

```
Architect describes building  →  Agent generates 3D model
                                        ↓
                               Views: plan, section, elevation, 3D
                                        ↓
                              Architect draws feedback (colored layers)
                                        ↓
                              Agent interprets & iterates
                                        ↓
                                     (repeat)
```

Both side panels are collapsible (Cmd+1 / Cmd+2). The agent bar is always visible.

### Project Panel (left)

Three sections stacked vertically:

**Views** — tree of all available views:
```

    Section B-B
```

Click a view to open it. Right-click for "Open in split" / "Replace quadrant."

### Viewport Area (center)

Three layout modes, switchable via segmented control (top-right corner) or keys 1/2/4:

**Single** — one view fills the viewport. Best for focused annotation.

**Split** — two views side by side, draggable divider. Good for plan vs. 3D comparison.

**Quad** — 2×2 grid, the default review layout:
```
┌─────────────────┬─────────────────┐
│   Floor Plan    │  3D Perspective  │
│                 │                  │
├─────────────────┼─────────────────┤
│   Section A-A   │  East Elevation  │
│                 │                  │
└─────────────────┴─────────────────┘
```

Each quadrant has a label dropdown to swap view type. Double-click to maximize; Esc to return.

**Viewport header** (28px per view): view type label, floor-level or section-cut selector, zoom indicator.

### Feedback Panel (right)

Color-coded feedback cards (see Feedback System below). "+" button at top to create a new one.

### Agent Bar (bottom)

- Text input: "Describe what you want..." (or "Add instructions alongside feedback...")
- Action button: "Generate" (initial) or "Apply Feedback (N)" (during iteration)
- Status indicator: idle / generating (animated pulse) / done
- Cmd+Enter submits

## Agent Interaction Flow

### Starting a Project

Three entry points, same destination:

1. **Text prompt** (primary) — describe the building in natural language. Agent generates initial 3D model.
2. **Sketch start** — draw rough floor plan freehand on blank canvas, then "Generate from Sketch."
3. **Parameters** — structured form (building type, stories, footprint, room list).

### Generation State

While the agent works:
- Agent bar shows animated status: "Generating initial model..." or "Applying feedback (3 items)..."
- Viewport dims slightly (10% overlay). Pan/zoom still works, annotation is disabled.
- Geometry appears progressively — lines and surfaces stream in, giving the feeling the agent is "drawing."
- On completion: overlay clears, toast notification, model snaps to final state.

### Triggering Iteration

After annotating, the architect has three equivalent paths:
1. Click "Apply Feedback (N)" button in agent bar
2. Type additional text in agent bar + hit submit (combines text with annotations)
3. Cmd+Enter

The agent receives **all unresolved feedback items** (structured data + text) plus any new text prompt.

### Version History

Every iteration = a new version, listed in the project panel:

```
History
  v5  "Added dormer windows"         now
  v4  "Widened kitchen, moved stairs" 2min
  v3  "Applied 3 feedback items"      8min
  v2  "Generated from sketch"         12min
  v1  "Initial generation"            15min
```

- Click a version → read-only view with yellow banner: "Viewing v3 — Click 'Restore' to continue from here."
- Shift+click two versions → diff view (removed=red, added=green, unchanged=gray).
- Restoring an old version creates a new version (v6 from v3). Skipped versions stay accessible but dimmed.

## Feedback System

The core innovation. Every feedback item is a **(drawing + text)** pair in a **unique color**.

### Creating Feedback

1. **"+" button** (or press F) → new card appears in feedback panel with auto-assigned color from palette: coral, teal, amber, violet, lime, sky, rose, indigo (high contrast on both light/dark backgrounds)
2. **Draw** — viewport enters annotation mode (crosshair cursor). Everything drawn appears in the active feedback's color, on the current view.
3. **Write** — text input on the feedback card (auto-focused). Describe the intent: "Move this wall 2 feet south."
4. **Continue** — press "+" for another feedback item (new color), click an existing card to resume its color, or Esc to exit annotation mode.

### Annotation Tools

Floating toolbar appears in annotation mode:

| Tool | Key | Behavior |
|------|-----|----------|
| Freehand | D | Default. Draw strokes freely. |
| Line | L | Click two points. For alignments, dimensions. |
| Arrow | A | Click two points, arrowhead at end. "Move this there." |
| Circle | C | Click center + drag radius. Circle problem areas. |
| Rectangle | R | Click corner + drag. Mark zones. |
| Text | T | Click to place a text label on the viewport. |
| Eraser | E | Erases strokes from the active feedback only. |

All tools draw at 2px stroke width (at current zoom). Annotations are vector data — crisp at any zoom.

### Feedback Cards

```
┌──────────────────────────────────────┐
│ ● Feedback #3                    [×] │
│                                      │
│ "Move the kitchen island closer to   │
│  the south wall, roughly aligned     │
│  with the window."                   │
│                                      │
│ ┌────────────┐                  [👁] │
│ │ [thumbnail] │  Floor Plan L1       │
│ └────────────┘                       │
└──────────────────────────────────────┘
```

- **Color dot** — filled circle in the feedback color
- **Title** — auto "Feedback #N", click to rename
- **Text** — the architect's description, editable anytime
- **Thumbnail** — cropped preview of annotation strokes; click to zoom viewport there
- **Eye toggle** — show/hide this feedback's annotations
- **[×]** — delete feedback + its annotations
- Active card has highlighted border in its color

### Resolving Feedback

After an agent iteration, each card gains a **checkmark button**:
- Click → marked "Resolved"
- Resolved cards collapse to a "Resolved" section at the bottom
- Annotations become hidden (toggleable)
- Color is freed for reuse
- "Resolve All" button for batch resolution

### What the Agent Receives

Structured data per feedback item:
- Text description
- Per-view annotation strokes (type, coordinates) with view context (which view, which level, which section cut)
- The view's projection parameters so the agent can map 2D annotations back to 3D model space
- Fallback: rasterized screenshot of annotated views for visual interpretation of ambiguous feedback

## View System

### View Types

| View | What it shows | Defined by |
|------|---------------|------------|
| **Floor Plan** | Horizontal cut at 1.2m above floor level | Floor level selection |
| **Section** | Vertical cut through the building | Cut line drawn on floor plan |
| **Elevation** | Orthographic exterior face projection | Direction (N/S/E/W) or elevation line |
| **3D Perspective** | Freely orbitable 3D view | Camera position |
| **Site Plan** | Top-down, building footprint on lot | (automatic) |

### Navigation per View

| View | Pan | Zoom | Orbit |
|------|-----|------|-------|
| Floor Plan | Drag / Space+drag | Scroll / Pinch | — |
| Section | Drag / Space+drag | Scroll / Pinch | — |
| Elevation | Drag / Space+drag | Scroll / Pinch | — |
| 3D | Shift+Right-drag | Scroll / Pinch | Right-drag |

Cmd+0 = fit all (any view). 3D-specific: "G" toggles ghost mode (semi-transparent walls).

### Section & Elevation Cuts

Section cut lines are interactive objects on the floor plan:
- Dashed line with labels at each end ("A" — "A")
- Direction arrows show viewing side
- Draggable — section view updates in real time

To create: press S → click two points on plan → click to set viewing direction → new section view appears in view tree.

Elevation lines work the same way around the building perimeter.

### Annotations Are View-Specific

Annotations drawn on "Floor Plan Level 1" appear only on that view. This is intentional — architectural feedback is view-specific. The feedback card lists which views have annotations for that item, and clicking those labels navigates there.

## Golden Path

1. **New project** → name it, type a prompt: "Modern two-story home, 2500 sq ft, 4 bedrooms, open-plan ground floor, flat roof."
2. **Agent generates** → geometry appears progressively in quad view.
3. **Review** → study the quad view. Kitchen too small, staircase blocks sight line.
4. **Feedback round:**
   - Press F → coral color. Draw arrows on plan showing kitchen expanding east. Type: "Enlarge kitchen — extend east wall ~4 feet."
   - Press F → teal color. Circle staircase, draw arrow to NW corner. Type: "Relocate staircase to open sight line from entry to rear garden."
   - Click "Apply Feedback (2)."
5. **Agent iterates** → model updates. Kitchen looks great (resolve coral). Staircase moved but hallway too narrow (keep teal, add note).
6. **Next round** → type in agent bar: "Add powder room under staircase." Hit Apply. Agent addresses both teal annotation and text prompt.
7. **Repeat** until satisfied.
8. **Export** (future) → DXF, PDF sheets, 3D formats.

## Visual Design

- **Palette:** Neutral UI (slate grays, white) — model linework and feedback colors must pop
- **Density:** Architect-grade, compact. Think Figma/Rhino, not consumer web. Tooltips on icons.
- **Typography:** Clean sans-serif (Inter), monospace for dimensions
- **Dark mode:** Dark viewport (#1a1a1a), light lines. Feedback colors work on both backgrounds.
- **Cursors:** Arrow (navigate), crosshair (annotate), grab hand (pan), orbit icon (3D rotate)

## Open Questions

- **Dimensions display:** Auto-generated room size labels on plans? Toggle-able?
- **Conflicting feedback:** Agent should flag contradictions and ask for clarification rather than silently picking one.
- **Orphaned annotations:** If agent removes geometry that had feedback on it, annotations float in space. Card gets a warning: "Referenced geometry has changed."
- **Scale & units:** Project-level setting (imperial/metric). Dimensions and grid snap follow the setting.
- **Multi-user:** Out of scope for v1. Color system is single-architect. Multi-user would need per-user color ranges.
- **Print/PDF:** Each view exportable as scaled PDF with title block. Future feature but view proportions should be print-ready.
