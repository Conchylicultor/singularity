# Unified Barrel Type Registry (v3)

## Context

Every plugin can have barrel directories (`web/`, `server/`, `check/`, `lint/`, etc.) — subdirectories with an `index.ts` entry point. Today, web/server/central each have specialized codegen with different formats, while check and lint use ad-hoc filesystem walks + dynamic imports. Adding a new barrel type requires editing hardcoded lists and writing custom discovery code.

Goal: one codegen, one generated format, zero config per barrel type.

## Changes from v2

- **Lazy imports everywhere** — not just web. Error isolation via `Promise.allSettled` applies to all runtimes (a broken plugin shouldn't crash the process). Async cost in Bun for local imports is negligible.
- **`dependsOn` everywhere** — referenced by `pluginPath` string, resolved after loading. Free data; consumers ignore it if they don't need ordering.
- **Zero flags** — `defineBarrelType("dir")` takes one argument. No `lazy`, no `dependsOn`, no `type` config.
- **Web/server/central specialized codegen eliminated** — all barrel types share one format.

## Design

### 1. Convention-based barrel detection

A directory is a barrel if it has an `index.ts` and its name isn't `plugins` or `node_modules`. No hardcoded barrel list.

**`findAllPluginDirs`** in `plugin-tree.ts` (line 549):

```ts
const EXCLUDED_DIRS = new Set(["plugins", "node_modules"]);

function hasAnyBarrel(dir: string): boolean {
  let entries;
  try { entries = readdirSync(dir, { withFileTypes: true }); } catch { return false; }
  return entries.some(
    e => e.isDirectory() && !EXCLUDED_DIRS.has(e.name) && existsSync(join(dir, e.name, "index.ts"))
  );
}
```

Replaces the 7 hardcoded `existsSync` checks. Same change in `findPluginDirs` in `allow-default-project.ts`.

### 2. `defineBarrelType()` — one arg, zero config

New file: `plugins/plugin-meta/plugins/plugin-tree/core/internal/barrel-types.ts`

```ts
export interface BarrelTypeDef {
  readonly dir: string;
  readonly _brand: "BarrelTypeDef";
}

export function defineBarrelType(dir: string): BarrelTypeDef {
  return { dir, _brand: "BarrelTypeDef" };
}

export function isBarrelTypeDef(value: unknown): value is BarrelTypeDef {
  return typeof value === "object" && value !== null &&
    (value as BarrelTypeDef)._brand === "BarrelTypeDef";
}
```

Re-export from `plugin-tree/core/index.ts`.

### 3. Barrel type declarations

Each barrel-owning plugin declares its barrel type in its `core/` barrel:

```ts
// plugins/framework/plugins/web-sdk/core/index.ts
export const webBarrelType = defineBarrelType("web");

// plugins/framework/plugins/server-core/core/index.ts
export const serverBarrelType = defineBarrelType("server");

// plugins/framework/plugins/central-core/core/index.ts
export const centralBarrelType = defineBarrelType("central");

// plugins/framework/plugins/tooling/plugins/checks/core/index.ts
export const checkBarrelType = defineBarrelType("check");

// plugins/framework/plugins/tooling/plugins/lint/core/index.ts
export const lintBarrelType = defineBarrelType("lint");
```

Adding a new barrel type (e.g. `guard/`) = one `defineBarrelType("guard")` call in the owning plugin's `core/`.

### 4. One generated format for all barrel types

The codegen discovers `BarrelTypeDef` exports by scanning `core/` barrels, then generates one file per barrel type at `<ownerDir>/core/<dir>.generated.ts`.

Every generated file has the same shape:

```ts
// <dir>.generated.ts (autogenerated)
export const <dir>Entries = [
  { pluginPath: "foo", hierarchyPath: "foo",
    loader: () => import("@plugins/foo/<dir>"), dependsOn: ["bar"] },
  { pluginPath: "bar", hierarchyPath: "bar",
    loader: () => import("@plugins/bar/<dir>"), dependsOn: [] },
] as const;
```

- **`loader`** — lazy `import()` for error isolation. `Promise.allSettled` catches broken plugins.
- **`dependsOn`** — `pluginPath` strings referencing other entries. Computed by scanning cross-plugin import edges at build time. Resolved to actual values by the consumer after loading.
- **`pluginPath`** — relative path under `plugins/`, e.g. `"conversations/plugins/conversation-view"`.
- **`hierarchyPath`** — dot-separated hierarchy, e.g. `"conversations/conversation-view"`.

### 5. Codegen implementation

**Discovery** — scan `core/index.ts` files for `defineBarrelType` calls:

```ts
async function discoverBarrelTypes(root: string): Promise<(BarrelTypeDef & { ownerDir: string })[]> {
  const tree = buildPluginTree(resolve(root, "plugins"));
  const out: (BarrelTypeDef & { ownerDir: string })[] = [];
  for (const node of tree.byDir.values()) {
    const coreBarrel = join(node.dir, "core", "index.ts");
    if (!existsSync(coreBarrel)) continue;
    if (!readFileSync(coreBarrel, "utf8").includes("defineBarrelType")) continue;
    const mod = await import(coreBarrel);
    for (const value of Object.values(mod)) {
      if (isBarrelTypeDef(value)) out.push({ ...value, ownerDir: node.dir });
    }
  }
  return out;
}
```

**Registry generation** — one function for all barrel types:

```ts
function renderBarrelRegistry(opts: { root: string; barrelType: BarrelTypeDef & { ownerDir: string } }): string {
  const { barrelType } = opts;
  const dir = barrelType.dir;
  const tree = buildPluginTree(resolve(opts.root, "plugins"));

  // Collect entries — plugins that have <dir>/index.ts with export default
  const entries: { alias: string; importPath: string; pluginPath: string; hierarchyPath: string }[] = [];
  for (const node of tree.byDir.values()) {
    const barrel = join(node.dir, dir, "index.ts");
    if (!existsSync(barrel) || !hasDefaultExport(barrel)) continue;
    entries.push({
      alias: `_${entries.length}`,
      importPath: `@plugins/${node.path}/${dir}`,
      pluginPath: node.path,
      hierarchyPath: node.hierarchyId.replaceAll(".", "/"),
    });
  }
  entries.sort((a, b) => a.importPath.localeCompare(b.importPath));

  // Compute dependsOn — cross-plugin import edges within <dir>/ directories
  const importPathToPluginPath = new Map(entries.map(e => [e.importPath, e.pluginPath]));
  const deps = new Map<string, string[]>();
  for (const e of entries) {
    const pluginDir = join(resolve(opts.root, "plugins"), e.pluginPath);
    const imports = collectCrossPluginImports(pluginDir, dir);
    const resolved = [...imports]
      .map(imp => importPathToPluginPath.get(imp))
      .filter((p): p is string => p !== undefined && p !== e.pluginPath);
    if (resolved.length) deps.set(e.pluginPath, resolved.sort());
  }

  const exportName = `${dir}Entries`;
  return [
    HEADER, "",
    ...entries.map(e => {
      const d = deps.get(e.pluginPath) ?? [];
      return `  { pluginPath: ${JSON.stringify(e.pluginPath)}, hierarchyPath: ${JSON.stringify(e.hierarchyPath)}, loader: () => import(${JSON.stringify(e.importPath)}), dependsOn: ${JSON.stringify(d)} },`;
    }),
  ].join("\n")
    .replace(HEADER, `${HEADER}\n\nexport const ${exportName} = [\n`)
    + "\n] as const;\n";
}
```

**Main entry point** — replaces the current runtime-specific `generatePluginRegistry`:

```ts
export async function generatePluginRegistry(opts: { root: string }): Promise<void> {
  const barrelTypes = await discoverBarrelTypes(opts.root);
  for (const bt of barrelTypes) {
    const file = join(bt.ownerDir, "core", `${bt.dir}.generated.ts`);
    const next = renderBarrelRegistry({ root: opts.root, barrelType: bt });
    const existing = existsSync(file) ? readFileSync(file, "utf8") : "";
    if (next !== existing) writeFileSync(file, next);
  }
}
```

The existing `RUNTIMES` config, `renderPluginRegistry`, and the web vs server format branching are all deleted — replaced by the single `renderBarrelRegistry`.

### 6. Consumer migration

All consumers follow the same pattern: import entries, load via `Promise.allSettled`, resolve `dependsOn` strings.

**Shared helper** (optional — consumers can inline this):

```ts
async function loadBarrelEntries<T>(
  entries: readonly { pluginPath: string; hierarchyPath: string; loader: () => Promise<{ default: T }>; dependsOn: readonly string[] }[]
): Promise<{ pluginPath: string; hierarchyPath: string; value: T; dependsOn: string[] }[]> {
  const results = await Promise.allSettled(
    entries.map(async (e) => ({
      pluginPath: e.pluginPath,
      hierarchyPath: e.hierarchyPath,
      value: (await e.loader()).default,
      dependsOn: [...e.dependsOn],
    }))
  );
  for (const r of results) {
    if (r.status === "rejected") console.warn(`[barrel] failed to load plugin: ${r.reason}`);
  }
  return results.filter(r => r.status === "fulfilled").map(r => r.value);
}
```

#### Web

**Before:**
```ts
// web-core/web/loader.ts
import { pluginEntries } from "./plugins.generated";

const results = await Promise.allSettled(pluginEntries.map(e => e.loader()));
const plugins = results
  .filter(r => r.status === "fulfilled")
  .map(r => { r.value.default._hierarchyPath = ...; return r.value.default; });
```

**After:**
```ts
// web-core/web/loader.ts
import { webEntries } from "@plugins/framework/plugins/web-sdk/core/web.generated";

const loaded = await loadBarrelEntries(webEntries);
const plugins = loaded.map(e => {
  e.value._hierarchyPath = e.hierarchyPath;
  return e.value;
});
```

#### Server

**Before:**
```ts
// server-core/bin/index.ts
import { plugins } from "./plugins.generated";
// plugins already have dependsOn and _hierarchyPath mutated by generated file

```

**After:**
```ts
// server-core/bin/index.ts
import { serverEntries } from "../core/server.generated";

const loaded = await loadBarrelEntries(serverEntries);
const byPath = new Map(loaded.map(e => [e.pluginPath, e.value]));
for (const e of loaded) {
  e.value._hierarchyPath = e.hierarchyPath;
  e.value.dependsOn = e.dependsOn.map(p => byPath.get(p)).filter(Boolean);
}
const ordered = topoSortPlugins(loaded.map(e => e.value));
```

### 7. `plugins-registry-in-sync` check

The sync check discovers barrel types the same way the codegen does, then compares:

```ts
const barrelTypes = await discoverBarrelTypes(root);
for (const bt of barrelTypes) {
  const file = join(bt.ownerDir, "core", `${bt.dir}.generated.ts`);
  const expected = renderBarrelRegistry({ root, barrelType: bt });
  if (!existsSync(file) || readFileSync(file, "utf8") !== expected) {
    return { ok: false, message: `${relative(root, file)} out of sync`, hint: "Run `./singularity build`" };
  }
}
```

The current web/server/central-specific logic is replaced by this single generic loop.

### 8. Boundary checker

`boundaries/core/resolve.ts` line 5: `const RUNTIMES = new Set(["web", "server", "central", "core", "shared"])` stays for now. This defines valid cross-plugin import targets — an import-rule concern, not a detection concern. Check/lint barrels aren't imported cross-plugin. A follow-up could derive this from barrel type metadata.

### 9. `PluginNode` changes

- `PluginNode.exports`: widen key type from `Record<Runtime | "core" | "shared", BarrelExport[]>` to `Record<string, BarrelExport[]>`. `collectPlugin` discovers barrel dirs dynamically and parses exports for all of them.
- `PluginNode.runtimes`: stays `Record<Runtime, boolean>` for backward compat — derived from exports.

## Adding a new barrel type — the workflow

1. Add `whatever/index.ts` to any plugin → tree walker detects it automatically
2. In the owning framework plugin: `export const whateverBarrelType = defineBarrelType("whatever")`
3. `./singularity build` → codegen generates `whatever.generated.ts`
4. Write a consumer that imports from the generated file

No hardcoded lists. No custom discovery code. No codegen flags.

## Verification

1. `./singularity build` generates all 5 `.generated.ts` files
2. `./singularity check` passes — `plugins-registry-in-sync` covers all generated files
3. App loads correctly at `http://<worktree>.localhost:9000` — all web plugins render
4. Server starts — all server plugins initialize in correct order (dependsOn resolved)
5. ESLint discovers lint rules (same scoping behavior)
6. All existing checks run and pass
7. `git diff $(git merge-base HEAD main)` for review

## Tradeoffs

- New barrel instances (e.g. adding `check/index.ts` to a plugin) require `./singularity build`. The `plugins-registry-in-sync` check catches drift.
- Lazy imports add one microtask per plugin at startup. Negligible for local imports in Bun; meaningful error isolation for all runtimes.
- `dependsOn` is computed for all barrel types even if unused. Build-time cost is incremental (scanning .ts files for import edges); no runtime cost if ignored.
