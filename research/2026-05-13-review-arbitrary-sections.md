# Review: Arbitrary User-Defined Sections

## Context

The review file list currently hardcodes a two-group split: "main files" (flat rows, no header) and "Auto-generated" (collapsible, collapsed by default). The user wants:

1. Every group — including the default one — gets a collapsible header with per-section stats
2. All sections expanded by default
3. Fully dynamic user-defined sections manageable from Settings, backed by a DB table with a custom Config.Section UI

## Approach

Each section is a row in a plugin-owned `review_sections` DB table. The Settings pane shows a custom component for creating/editing/deleting sections. The review view reads sections via a live-state resource and groups files accordingly. Files not matching any section appear in a default "Changes" group.

Follows the established `quick-prompts` / `prompt-templates` pattern: DB table → CRUD routes → `defineResource` → `useResource` → Config.Section component.

## Changes

### 1. DB table — `server/internal/tables.ts`

```ts
pgTable("review_sections", {
  id:        text("id").primaryKey(),
  name:      text("name").notNull(),
  patterns:  jsonb("patterns").$type<string[]>().notNull().default([]),
  rank:      rankText("rank").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
});
```

Each row is a section. `patterns` is a JSONB `string[]` — path patterns using the existing `matchesPattern` logic (prefix match by default, `**/` for suffix match).

### 2. Shared resource — `shared/resources.ts`

```ts
const ReviewSectionSchema = z.object({
  id: z.string(),
  name: z.string(),
  patterns: z.array(z.string()),
  rank: RankSchema,
});
export const reviewSectionsResource = resourceDescriptor<ReviewSection[]>(...);
```

### 3. Server — routes + resource

- `server/internal/resources.ts` — `defineResource` with `mode: "push"`, queries `review_sections` ordered by rank
- CRUD routes:
  - `GET /api/review-sections` — list
  - `POST /api/review-sections` — create (`{ name, patterns }`)
  - `PATCH /api/review-sections/:id` — update name/patterns
  - `DELETE /api/review-sections/:id` — delete
- Each mutation calls `resource.notify()`
- `server/index.ts` — register routes + `Resource.Declare`

### 4. Default sections — `server/internal/seed.ts`

On server boot, if the `review_sections` table is empty, seed default rows:
- "Auto-generated" with patterns `["**/CLAUDE.md", "docs/plugins-compact.md", "docs/plugins-details.md", "server/src/db/migrations/"]`

### 5. Settings UI — `web/components/review-sections-settings.tsx`

Config.Section contribution in `web/index.ts`:
```ts
Config.Section({
  id: "review-sections",
  title: "Review Sections",
  description: "File groupings shown in the review pane.",
  component: ReviewSectionsSettings,
})
```

The component:
- Lists sections from `useResource(reviewSectionsResource)`
- Each section row: editable name, editable patterns (textarea, one per line), delete button
- "Add section" button at the bottom
- Saves on blur (same pattern as quick-prompts)

### 6. Classification logic — `web/core-files.ts`

Remove `isAutoGenerated`. Add:

- `classifyFile(path, sections): string | null` — returns the `id` of the first section whose patterns match, or `null`
- `groupBySection(files, sections): FileSection[]` — groups files; default section (unmatched, label "Changes") first, then matched sections in rank order. Empty sections omitted.

`matchesPattern` stays (private, unchanged). `getFileWarningLevel` stays (orthogonal).

### 7. Review view — `web/components/review-view.tsx`

- Read sections via `useResource(reviewSectionsResource)`
- Replace `main`/`generated` split with `groupBySection(sorted, sections)`
- Replace `AutoGeneratedSection` with generic `FileSectionBlock` — `Collapsible` with `defaultOpen={true}`, shows section name + file count + additions/deletions
- Default "Changes" section uses the same `FileSectionBlock`
- `ToolbarRow` unchanged (totals across all files, expand/collapse-all controls file diffs)

### 8. Config cleanup — `shared/config.ts`

Remove `autoGeneratedPaths` from `reviewConfig`. Keep `safePaths` and `carefulPaths` (orthogonal warning-level concern).
