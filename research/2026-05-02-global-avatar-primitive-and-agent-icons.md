# Avatar primitive + agent icons

## Context

Today, conversations launched by an Agent are marked with a tiny violet "Agent" chip in the sidebar row (`plugins/agents/web/components/agent-chip-row.tsx`). It is identical for every agent — there is no way to tell *which* agent launched a conversation at a glance — and it sits alongside the relative time as a secondary label.

We want to mirror the visual language of standard chat apps (Slack DMs reference): each row has a circular avatar on the leftmost edge, color-coded and icon-bearing. For agent-launched conversations the avatar reflects the agent's identity (icon + color), so users can differentiate agents at a glance. For non-agent conversations the slot is reserved but blank — keeping every row's title aligned.

The avatar is also useful beyond conversation rows: agent detail pages (header), the conversation pane header (toolbar swap), and any future surface that renders a Conversation/Agent. So we ship it as a stand-alone primitive plugin.

## Approach

Three pieces:

1. **New primitive plugin `avatar`** under `plugins/primitives/plugins/avatar/` — pure web. Exposes a curated icon registry (string keys → react-icons/md components), a color palette (full-saturation Tailwind classes), an `<Avatar>` renderer, an optional status dot overlay, and an `<AvatarPicker>` popover.
2. **Agents schema** gets two new nullable text columns: `icon` (string key into the registry) and `iconColor` (palette key). Agent detail page renders an `<AvatarPicker>` next to the editable name field.
3. **Conversation row + toolbar** consume the avatar. Item plugin defines a new `Item.Avatar` slot rendered as the leftmost column with reserved width (so all rows align). Agents plugin contributes an `AgentAvatarRow` component that resolves the conversation's launching agent (via `agentLaunchesResource` + `taskId`) and renders the avatar with its status dot overlaid. For non-agent conversations the slot resolves empty and we render a blank-width spacer. The conversation pane header (`AgentChipToolbar`) is also rewritten to render the avatar to the left of the title.

The existing `<ConvStatusDot>` component is no longer rendered as a leftmost element — it becomes a small badge overlaid on the avatar (or on the empty placeholder, for non-agent rows).

```

        ├── avatar-picker.tsx        # popover: icon grid + color row
        └── icons.ts                 # AVATAR_ICONS registry (string → IconType)
        └── colors.ts                # AVATAR_COLORS palette
```

`web/index.ts` exports:

- `Avatar`, `AvatarPicker` components
- Types: `AvatarSpec` (`{ icon: string | null; color: string | null }`), `AvatarSize` (`"xs" | "sm" | "md" | "lg"`)
- Helpers: `resolveAvatarIcon(key)` (with fallback to `MdPrecisionManufacturing`), `colorBgClass(key)` (deterministic fallback when null), `DEFAULT_AGENT_AVATAR`
- The plugin definition has empty `contributions` (a pure primitive).

The status dot overlay is part of `<Avatar>` — pass an optional `statusDot` prop (color class) and it renders as an absolute-positioned `size-2` circle bottom-right with a ring matching the surface bg, like Slack.

The icon registry is a curated map from `react-icons/md` (project standard, see `package.json`):

```ts
import { MdPrecisionManufacturing, MdRocket, MdBugReport, ... } from "react-icons/md";
export const AVATAR_ICONS = {
  robot: MdPrecisionManufacturing,
  rocket: MdRocket,
  bug: MdBugReport,
  // ~30 curated icons
} as const;
export const DEFAULT_AGENT_AVATAR = { icon: "robot", color: "violet" } satisfies AvatarSpec;
```

The color palette duplicates the Tailwind hue keys from `plugins/conversations/plugins/conversation-category/web/internal/colors.ts` but at full saturation (`bg-violet-500`, `bg-sky-500`, …) since avatars are filled circles, not chip backgrounds. We do **not** import that file — keeping the avatar primitive standalone — but we keep keys aligned so the two systems read consistently.

### Agents schema + API

```ts
icon: text("icon"),
iconColor: text("icon_color"),
```

`plugins/agents/shared/schemas.ts` — extend `AgentSchema` with `icon: z.string().nullable()` and `iconColor: z.string().nullable()`. Migration is auto-generated by `./singularity build`.

`plugins/agents/server/handle-patch.ts` (or wherever `PATCH /api/agents/:id` lives) — accept the two new fields in the patch body. The existing pattern already updates whatever validated fields are present.

### Agent detail page

`plugins/agents/web/components/agent-detail.tsx` — replace the status-only header (`<AgentStatus + name input>`) with `<AvatarPicker + name input>`. The picker:

- Renders an `<Avatar>` button (size `lg`) showing the current `agent.icon` / `agent.iconColor` (falling back to `DEFAULT_AGENT_AVATAR`).
- Click opens a popover with an icon grid (~30 cells, MdRocket etc.) and a color swatch row (10 swatches).
- Selecting either patches the agent immediately via the existing `save({ icon, iconColor })` callback. No autosave/blur dance needed — selection is the commit.

`AgentStatus` (the small running indicator) is moved into the avatar overlay (re-using the avatar's `statusDot` prop) so it's not lost.

### Conversation item slot

`plugins/conversations/plugins/conversation-ui/plugins/item/web/slots.ts` — add a new slot:

```ts
Avatar: defineSlot<{ component: ComponentType<{ conv: ConversationItemConv }> }>(),
```

`plugins/conversations/plugins/conversation-ui/plugins/item/web/components/conversation-item.tsx` — extend `ConversationItemConv` with `taskId?: string | null` (already on the underlying `Conversation` and `ConversationSummary`). Replace the leftmost `<ConvStatusDot>` column with an `<AvatarSlot>` that:

- Iterates `Item.Avatar.useContributions()`.
- Renders the **first** contribution that returns non-null.
- Falls back to a blank `<Avatar />` (no icon, no color, just an empty rounded placeholder) so all rows align by width.
- The status dot overlays the avatar (pass `statusDot={CONV_STATUS_DOT[conv.status]}` to `<Avatar>`).

Inline layout: same change — avatar on the left, status dot overlaid.

The existing `Item.Chips` slot stays untouched; `<AgentChipRow>` is **removed** entirely (no longer needed; the avatar carries the signal).

### Agents plugin wiring

`plugins/agents/web/index.ts`:

- Drop the `Item.Chips({ component: AgentChipRow })` contribution (and delete `agent-chip-row.tsx`).
- Add `Item.Avatar({ component: AgentAvatarRow })`.

`plugins/agents/web/components/agent-avatar-row.tsx` (new):

```tsx
export function AgentAvatarRow({ conv }: { conv: ConversationItemConv }) {
  if (conv.kind !== "agent" || !conv.taskId) return null;
  const { data: launches } = useResource(agentLaunchesResource);
  const { data: agents } = useResource(agentsResource);
  const launch = launches?.find((l) => l.taskId === conv.taskId);
  const agent = agents?.find((a) => a.id === launch?.agentId);
  if (!agent) return null;
  return <Avatar icon={agent.icon ?? DEFAULT_AGENT_AVATAR.icon}
                 color={agent.iconColor ?? DEFAULT_AGENT_AVATAR.color}
                 size="sm"
                 statusDot={CONV_STATUS_DOT[conv.status]} />;
}
```

### Conversation pane header (toolbar swap)

`plugins/agents/web/components/agent-chip-toolbar.tsx` — rewrite to render an `<Avatar size="sm">` followed by the agent name, contributed at `position: "left"` so it lands left of the title (already the current contribution position — verify in `index.ts`).


Order: place near the other primitives (live-state, networking, pane).

- Migrating the conversation-category color palette to share a common color primitive.
