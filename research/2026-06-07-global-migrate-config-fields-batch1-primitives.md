# Migrate config field plugins into `fields/` â€” Batch 1 (primitives)

## Context

This is **task 5** of the unified-`fields/` migration
([research/2026-06-06-global-unified-fields-primitive.md](./2026-06-06-global-unified-fields-primitive.md),
stage S3). Tasks 1â€“4 already landed: `fields/core` owns the `FieldType` token,
`config_v2/core` re-exports it via a temporary shim, and every primitive type
already has a canonical token + identity + table/filter capabilities under
`plugins/fields/plugins/<type>/`.

What is **still** in the old location is the *config* capability of the
primitive types â€” the editable-scalar **factory** (`textField`, `boolField`, â€¦)
and the **config renderer** â€” bundled in two legacy plugins:

- `plugins/config_v2/plugins/fields/plugins/primitives/` (text + bool + int + float, one plugin)
- `plugins/config_v2/plugins/fields/plugins/multiline-text/`

Batch 1 relocates these into the matrix layout `fields/plugins/<type>/plugins/config/`,
updates the **26 factory import sites**, and deletes the two legacy plugins. It
establishes the complete relocation pattern (factory **and** renderer, importers
updated) that batches 2â€“3 will follow.

> Note: the enum migration (task 3) was a *partial* validation â€” it relocated only
> the renderer and left `enumField` + a duplicate `enumFieldType` token in
> config_v2. Batch 1 deliberately goes further (per the task brief and the research
> doc's dependency graph, line 226 `fields/<t>/config/core (factory; +importers)`).
> Enum's leftover factory is **out of scope** (batch 2).

### Invariants (must stay byte-identical â€” behavior-preserving)

- Dispatch slot id: `"config-v2.fields.renderer"` (unchanged, owned by `config_v2/plugins/fields`).
- Dispatch key: `field.type.id` â†’ the strings `"text"`, `"bool"`, `"int"`, `"float"`, `"multiline-text"`.
- Renderer registration: `Fields.Renderer(<Renderer>)` with `<Renderer>.type = <token>`.
- Each factory keeps its exact name, signature, default value, zod schema, and `meta` shape.
- No live-state resources / storage-provider keys exist for these types (verified) â€” nothing else to freeze.

### Key design decisions

1. **Token unification.** The relocated factory imports the **canonical** token from
   `@plugins/fields/plugins/<type>/core` (the one already used by identity/table/filter).
   The duplicate `defineFieldType` calls in the legacy `primitives`/`multiline-text`
   cores are deleted. No external code imports those tokens directly (verified), so
   only the factories move.
2. **Factory + `FieldDef` interface live in the config-capability core**
   (`fields/plugins/<type>/plugins/config/core`), not the type core â€” keeps the
   `config_v2/core` + `zod` dependency inside the config capability so table/filter
   stay config-free. (Matches research doc dependency graph.)
3. **Shared renderer helpers extracted to shared homes** (user-confirmed):
   - `FieldHeader`, `useLocalValue` â†’ `config_v2/plugins/fields/web` (the slot owner
     every renderer already imports `Fields`/`FieldRendererComponent` from).
   - `pickMeta` â†’ `config_v2/core` (home of `FieldMeta`/`FieldDef`).
   - `multiline-text` drops its inline copies and adopts the shared helpers
     (markup + impl are byte-identical, so behavior-preserving).

### Step 2 â€” Create the 5 config sub-plugins

For each type `T` in **text, bool, int, float, multiline-text**, under the existing
`plugins/fields/plugins/<T>/` create `plugins/config/` with:

```
plugins/fields/plugins/<T>/plugins/config/
  package.json              # name "@singularity/plugin-fields-<T>-config", private, "singularity": { "collapsed": true }
  CLAUDE.md                 # prose only â€” build inserts the AUTOGENERATED block
  core/
    index.ts               # export { <T>Field, type <T>FieldDef } from "./internal/<T>"
    internal/<T>.ts        # factory + FieldDef interface
  web/
    index.ts               # contributions: [Fields.Renderer(<T>Renderer)]
    components/<T>-renderer.tsx
```

**Factory core (`core/internal/<T>.ts`)** â€” relocate the factory + `<T>FieldDef`
interface from the legacy core, changing only the token source:

```ts
import { z } from "zod";
import { type FieldDef, type FieldMeta, pickMeta } from "@plugins/config_v2/core";
import { textFieldType } from "@plugins/fields/plugins/text/core";   // canonical token

export interface TextFieldDef extends FieldDef<string> {
  readonly type: typeof textFieldType;
}
export function textField(opts?: FieldMeta & { default?: string }): TextFieldDef {
  return Object.freeze({ type: textFieldType, schema: z.string(),
    defaultValue: opts?.default ?? "", meta: pickMeta(opts) });
}
```

- `bool` â†’ `boolFieldType`, `z.boolean()`, default `false`, `BoolFieldDef`.
- `int` â†’ `intFieldType`, `z.number().int()` + optional `.min/.max`, default `0`,
  `IntFieldDef` with `min?/max?/step?`.
- `float` â†’ `floatFieldType`, `z.number()` + optional `.min/.max`, default `0`,
  `FloatFieldDef` with `min?/max?/step?`.
- `multiline-text` â†’ `multilineTextFieldType`, `z.string()`, default `""`,
  `MultilineTextFieldDef` with `rows?`; now uses shared `pickMeta` (identical result).

**Renderer (`web/components/<T>-renderer.tsx`)** â€” relocate verbatim, repointing imports:

```ts
import { Fields, type FieldRendererComponent, FieldHeader, useLocalValue }
  from "@plugins/config_v2/plugins/fields/web";
import { textFieldType } from "@plugins/fields/plugins/text/core";   // for .type
import { type IntFieldDef } from "../../core";                        // (int/float/multiline only)
```

- `bool` renderer needs only `FieldHeader` (no `useLocalValue`).
- `int`/`float` read `min/max/step` from their config-core `FieldDef`; keep the exact
  on-blur logic (`Math.round` for int, raw `Number` for float; `step ?? 1` / `step ?? "any"`).
- `multiline-text` renderer uses shared `FieldHeader` + `useLocalValue` and keeps its
  `<textarea>` markup, `rows ?? 4`.

**Web barrel (`web/index.ts`)** â€” mirror enum config:

```ts
import type { PluginDefinition } from "@plugins/framework/plugins/web-sdk/core";
import { Fields } from "@plugins/config_v2/plugins/fields/web";
import { TextRenderer } from "./components/text-renderer";
export default {
  name: "Fields: Text Config",
  description: "Text field type: config-render capability (renderer for config-v2.fields.renderer) + textField factory.",
  contributions: [Fields.Renderer(TextRenderer)],
} satisfies PluginDefinition;
```

`package.json` names: `@singularity/plugin-fields-{text,bool,int,float,multiline-text}-config`.
