# Dissolve the top-level `e2e/` folder into per-plugin `e2e/` runtimes

**Date:** 2026-07-23 · **Category:** global

## Context

`e2e/` is a flat, repo-root folder of 29 Playwright `.mjs` scripts. It is the last
significant chunk of feature-specific code living outside `plugins/` — a plugin's
tests sit three directories away from the plugin they verify, discoverable only
by grepping. That violates the repo's organising principle (every feature is a
self-contained plugin) and it has visibly rotted, because nothing in the toolchain
can see these files:

- **5 scripts hardcode `../plugins/page/plugins/editor/node_modules/yjs/…`**, an
  import path that only resolves because `e2e/` happens to be a sibling of
  `plugins/`.
- **4 scripts default their target URL to a long-dead ephemeral worktree host**
  (`att-1781283277-ilxk.localhost:9000`, `claude-1776940724-olee.localhost:9000`, …).
  They cannot run as written.
- The `arg()` argv parser is byte-identical in 22 files; the chromium launch +
  `1400×900` viewport in 23; a verbatim "open a fresh blank Pages doc" flow in 14;
  `detectOsColorScheme()` in 2; a ~50-line profiler-report printer in 2. The
  pass/fail `check()` logger is reimplemented per script in **two mutually
  incompatible shapes**, with four different exit conventions.
- No script closes its browser in a `finally`, so a mid-script throw leaks a
  Chromium process.

**Outcome.** `e2e/` ceases to exist. Each test lives in its owning plugin at
`plugins/<path>/e2e/<name>.ts`. A new `e2e-harness` tooling plugin owns the shared
helpers, the two generic tools (`screenshot`, `perf`), and the chromium
provisioning. Scripts become `.ts` so `type-check` and `plugin-boundaries` cover
them — the rot above becomes impossible to reintroduce silently. Invocation stays
plain `bun <path>` (no CLI runner).

## Decisions

| Question | Decision | Why |
|---|---|---|
| Script + helper language | **`.ts`** | Bun runs `.ts` natively. `plugin-boundaries` walks only `.ts`/`.tsx`, and `type-check` only sees tsconfig-included files — `.mjs` helpers would be an entirely unchecked seam. Precedent: `plugins/primitives/plugins/css/plugins/layout-harness/web/internal/measure-page.ts` already drives Playwright from typed TS with `declare global` for browser-side globals. |
| Cross-plugin helper import | **`@plugins/…/<plugin>/e2e` barrel** | The only sanctioned grammar. Relative `../` escapes violate R8; workspace-name imports violate R7; `.mjs` + alias relies on bun resolving tsconfig `paths` from a `.mjs` importer — unverified, and unverifiable here (no `node_modules`). |
| Harness plugin | **`plugins/framework/plugins/tooling/plugins/e2e-harness/`** | Barrel reads `…/e2e-harness/e2e`; plugin name and runtime folder stay distinct. |
| Helpers folder | the plugin's own **`e2e/`**, not `core/` | `server-core/tsconfig.json` globs `**/plugins/*/core` with **no tooling exclusion**, so a `core/` here lands in the server TS program (`lib: ES2023`, no DOM) — `page.evaluate(() => document…)` bodies would fail to compile. |
| `e2e` as a boundary runtime | **`e2e: ["e2e", "core"]`** | Load-bearing, not optional: `runtimes` is default-deny, so without it *every* e2e→harness import is blocked. The policy is also exactly right — an e2e script may use other plugins' `core` types and `e2e` flow helpers, and is denied `web`/`server` (it drives the browser; it must not import app code). R12 then also forbids reaching into `shared/`. |
| `e2e` as a known plugin dir | fixed-set entry in `standardPluginDirs`, **not** `defineCollectedDir` | A collected dir generates an (empty) registry and drags in `collected-dir-tsconfig-coverage`. `e2e` is a structural convention like `bin`/`scripts`, not a contribution registry. |
| `check()` shape | one `Report` with `.ok()` and `.eq()` | The two in-tree shapes — `check(name, ok, detail)` and `check(label, actual, expected)` — are not safely overloadable on arity. Two named methods share one failure counter and one exit path. |
| Blank-Pages-doc flow | `plugins/page/plugins/editor/e2e/` | `framework/tooling` must not know the Pages app's copy (the literal `"Blank page"`). |
| Yjs block-doc decode | `plugins/page/plugins/editor-collab/e2e/` | It owns `page_block_docs`, `doc-init`/`doc-update`, and the `root`/`XmlText` layout. |
| Profiler report printer | `plugins/debug/plugins/render-profiler/core/` | Pure formatting over a type that plugin already exports; no Playwright, no DOM — a genuine public API, consumed by two different plugins' scripts. |

Cross-plugin specifier: `@plugins/framework/plugins/tooling/plugins/e2e-harness/e2e`.

### Helper API

**`args.ts`** — replaces 22 identical `arg()` copies and 3 divergent missing-arg endings.
```ts
export function arg(name: string): string | undefined;
export function arg(name: string, fallback: string): string;
export function numArg(name: string, fallback: number): number;
export function flag(name: string): boolean;
export function requireArg(name: string, usage: string): string;  // prints usage, exit 2
export function usage(line: string): never;
```

**`target.ts`** — structurally kills the dead-hostname class.
```ts
export function baseUrl(): string;              // --base | --url | --origin → $SINGULARITY_E2E_BASE
                                                // → http://${basename(REPO_ROOT)}.localhost:9000 → throw
export function pathUrl(path: string): string;
```
The default is *derived, never literal*: `basename(REPO_ROOT)` via
`@plugins/infra/plugins/paths/core` — the same derivation `test/bun-preload.ts`
uses for `SINGULARITY_WORKTREE` and the gateway uses for the namespace. Every
script then runs argument-free against its own worktree deploy.

**`browser.ts`**
```ts
export const DEFAULT_VIEWPORT: { width: 1400; height: 900 };
export interface Session { context: BrowserContext; page: Page; captured: Captured; label: string }
export interface Harness { browser: Browser; session(opts?: SessionOptions): Promise<Session> }
export function withBrowser<T>(run: (h: Harness) => Promise<T>): Promise<T>;   // finally { browser.close() }
export function boot(page: Page, url: string, opts?: { marker?: string; timeoutMs?: number; settleMs?: number }): Promise<void>;
```
`session()` folds in the context+page+capture wiring the two-context CRDT scripts
repeat verbatim for A and B. `boot()` replaces the ubiquitous
`goto` + `waitForTimeout(4000)` with the *polling* settle that
`deep-link-restore.mjs` and `screenshot.mjs` already independently reinvented —
fixed sleeps are the top flake source. `marker` keeps it app-agnostic.

**`capture.ts`**
```ts
export interface Captured { pageErrors: string[]; consoleErrors: string[]; failedRequests: string[] }
export function capture(page: Page, label?: string): Captured;
```

**`report.ts`**
```ts
export interface Report {
  ok(name: string, condition: boolean, detail?: string): void;
  eq(name: string, actual: unknown, expected: unknown): void;   // prints got/want; JSON.stringify equality
  fail(name: string, detail?: string): void;
  note(line: string): void;
  readonly failures: readonly string[];
  finish(): never;                                              // summary + exit 0/1
}
export function report(title?: string): Report;
```

**`shots.ts`** — `snap(page, outPrefix, name)` writes `${out}-${name}.png` and logs `wrote …` (~20 scripts).
**`color-scheme.ts`** — `detectOsColorScheme()`, lifted verbatim from `screenshot.mjs`.

### Domain barrels

`plugins/page/plugins/editor/e2e/index.ts` (+ `e2e/support/blank-page.ts`) — the
goto-`/pages` → click **"Blank page"** → wait → click-contenteditable flow is
verbatim in 14 scripts, and `.replace(/ /g," ").trim()` in ~10:
```ts
export interface BlankDoc { pageUrl: string; pageId: string; block: Locator; blockId: string }
export function openBlankPage(page: Page, base: string, opts?: { settleMs?: number }): Promise<BlankDoc>;
export function editableBlocks(page: Page): Locator;   // '[data-block-id] [contenteditable="true"]'
export function blockIdOf(block: Locator): Promise<string>;
export function blockText(block: Locator): Promise<string>;
export function caretState(block: Locator): Promise<CaretState>;
```

`plugins/page/plugins/editor-collab/e2e/index.ts` (+ `e2e/support/ydoc.ts`):
```ts
export function blockDocText(stateB64: string): string;   // Y update → lexical XmlText root → plain text
export function fetchBlockDoc(base: string, blockId: string): Promise<{ state: string } | undefined>;
```
**This deletes the 5 hardcoded `node_modules/yjs` paths for free.** Both
`editor/package.json` and `editor-collab/package.json` already declare
`"yjs": "^13.6.20"`, so a bare `import * as Y from "yjs"` inside either plugin's
tree resolves by ordinary walk-up. `crdt-restore-verify` imports `blockDocText`
from the collab barrel, so `history/package.json` needs no yjs dep. **No plugin
`package.json` is edited by this plan.**

`plugins/debug/plugins/render-profiler/core/` — add
`formatProfilerReport(report: ProfilerReport, opts?: { top?: number }): string`
(the `HOOK_LABEL` map + initiators/remounts loop, duplicated across two plugins'
scripts). Both scripts also switch from the hardcoded `"__reactRenderProfiler"`
string to the already-exported `RENDER_PROFILER_GLOBAL`.
