# Task meta-plugin ‚Äî initial slice

## Context

Singularity needs a first-class concept of **tasks**: a nested todo list of things agents work on, each task potentially having multiple **attempts** (tries at solving it), each attempt potentially spanning multiple conversations (to support future branching). This plan covers the first slice: schema + an empty task pane reachable from the sidebar. The pane will later be composed of independent sub-panes, so this plan also defines the meta-plugin extension surface (a slot that child plugins will contribute to), even though no child plugin exists yet.

Model the plugin after [`plugins/stats/`](../plugins/stats/) (cleanest "sidebar + route + slot-for-children" reference) and [`plugins/conversations/plugins/conversation-view/plugins/code/`](../plugins/conversations/plugins/conversation-view/plugins/code/) (reference for a meta-plugin with its own slot and server API).

## Schema

New file: `plugins/tasks/server/schema.ts` (drizzle-orm + drizzle-zod, mirroring `plugins/conversations/server/schema.ts:1-40`).

```ts
export const tasks = pgTable("tasks", {
  id: text("id").primaryKey(),
  parentId: text("parent_id").references((): AnyPgColumn => tasks.id, { onDelete: "cascade" }),
  title: text("title").notNull(),
  description: text("description"),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
});

export const taskAttempts = pgTable("task_attempts", {
  id: text("id").primaryKey(),
  taskId: text("task_id").notNull().references(() => tasks.id, { onDelete: "cascade" }),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
});
```

Plus `createSelectSchema` + exported `Task` / `TaskAttempt` zod types.

**Conversations ‚Üí attempt link.** To allow multiple conversations per attempt (future branching), add a nullable FK on the existing `conversations` table in `plugins/conversations/server/schema.ts`:

```ts
taskAttemptId: text("task_attempt_id").references(() => taskAttempts.id, { onDelete: "set null" }),
```

Cross-plugin import is fine (conversations already imports nothing from tasks today; tasks is a new plugin, so the FK direction must be conversations ‚Üí tasks to avoid a cycle ‚Äî do it by importing `taskAttempts` from `@plugins/tasks/server/schema` inside conversations/schema.ts).

Barrel export: add `export * from "@plugins/tasks/server/schema"` to `server/src/db/schema.ts`.

Migration is auto-generated by `./singularity build` (drizzle-kit) ‚Äî do NOT run drizzle-kit manually (per root `CLAUDE.md`).

### Frontend wiring

```ts

```

`plugins/tasks/web/index.ts` (modeled on `plugins/stats/web/index.ts:1-25`):
- `Shell.Sidebar` contribution: title "Tasks", icon (e.g. `MdChecklist` / `MdTaskAlt`), group `"System"`, `onClick` ‚Üí `Shell.OpenPane(tasksPane())`
- `Shell.Route` contribution: pattern `/tasks`, `resolve: () => tasksPane()`

`components/tasks-panel.tsx`: empty container that calls `Tasks.PanePanel.useContributions()` and maps them (stub layout ‚Äî placeholder text "No tasks yet" when no contributions). Keeps the sub-pane composition surface live so future child plugins light it up without touching the host.
