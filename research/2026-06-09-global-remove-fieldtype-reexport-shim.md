# Remove the temporary FieldType re-export shim, re-enable strict boundaries (task 8)

## Context

This is the **final task (S4 / task 8)** of the unified `fields/` primitive migration
(`research/2026-06-06-global-unified-fields-primitive.md`). The `FieldType` token now lives
in `fields/core`; during the gradual migration (S1–S3) `config_v2/core` was allowed to
*temporarily re-export* `defineFieldType` / `FieldType` from `fields/core` — a sanctioned,
documented exception to the no-cross-plugin-re-export rule — so the ~13 config field-type
plugins and their importers could move to `@plugins/fields/core` one batch at a time.

Those batches are done. Only **three** importers still pull `FieldType` through the
`config_v2/core` shim, and **zero** still pull `defineFieldType`. Once those three are
re-pointed at `@plugins/fields/core` directly, the shim and its boundary-check exception
can be deleted and the strict re-export rule re-enabled, closing the migration.
