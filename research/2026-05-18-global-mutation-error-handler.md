# Global Mutation Error Handler — Cleanup

## Context

A `MutationCache.subscribe` listener was added to `toaster-root.tsx` (shell layer) to auto-toast on unhandled mutation errors. The initial implementation has three structural problems that this plan fixes:

1. **Double-toast**: `useEndpointMutation`'s `onError` callback and the global toast are independent — both fire if a caller provides `onError`. The caller has no way to know they need to also set `meta.suppressError`.
2. **Layering smell**: `toaster-root.tsx` (shell) imports `EndpointError` from `infra/endpoints` to extract `body.message`. Shell should not know the internal shape of a transport error class.
3. **Discoverability**: The `suppressError` flag and the global behavior are undocumented in `endpoints/CLAUDE.md`, so agents writing mutations won't find them.

## Plan

### 1. Add `getEndpointErrorMessage` to the endpoints plugin

**File:** `plugins/infra/plugins/endpoints/web/internal/fetch-endpoint.ts`

Add a named export alongside `EndpointError`:

```ts
export function getEndpointErrorMessage(error: unknown): string {
  if (error instanceof EndpointError) {
    const { body } = error;
    if (
      body &&
      typeof body === "object" &&
      "message" in body &&
      typeof (body as { message: unknown }).message === "string"
    ) {
      return (body as { message: string }).message;
    }
    return error.message;
  }
  return error instanceof Error ? error.message : String(error);
}
```

**File:** `plugins/infra/plugins/endpoints/web/index.ts`

Add to the existing export line:
```ts
export { fetchEndpoint, EndpointError, getEndpointErrorMessage } from "./internal/fetch-endpoint";
```

This moves the extraction logic into the plugin that owns `EndpointError`. The toaster calls an opaque function; it never inspects `body` itself.

### 2. Auto-suppress the global toast when `onError` is provided

**File:** `plugins/infra/plugins/endpoints/web/internal/use-endpoint-mutation.ts`

Add `meta` to the opts type and wire auto-suppression:

```ts
opts?: {
  invalidates?: EndpointDef<any, any, any, any, any>[];
  onSuccess?: (data: TResponse extends void ? void : TResponse) => void;
  onError?: (err: EndpointError) => void;
  meta?: { suppressError?: boolean };  // explicit override; auto-derived from onError if absent
},
```

Inside `useMutation(...)`:

```ts
const suppressError = opts?.meta?.suppressError ?? (opts?.onError !== undefined);

return useMutation<...>({
  meta: { suppressError },
  mutationFn: ...,
  onSuccess: ...,
  onError: opts?.onError,
});
```

**Precedence:** explicit `meta.suppressError` wins; if absent, `onError !== undefined` decides; if absent, `false` → global toast fires.

The `Register` augmentation for `mutationMeta` stays in this file (it already lives here after the initial implementation), but move the comment to be the canonical documentation anchor.

---

### 3. Remove `EndpointError` import from `toaster-root.tsx`

**File:** `plugins/shell/plugins/toaster/web/components/toaster-root.tsx`

Replace:
```ts
import { EndpointError } from "@plugins/infra/plugins/endpoints/web";

function mutationErrorMessage(error: unknown): string { ... }
```

With:
```ts
import { getEndpointErrorMessage } from "@plugins/infra/plugins/endpoints/web";
```

And in the subscriber:
```ts
toast.error(getEndpointErrorMessage(event.action.error));
```

The local `mutationErrorMessage` helper is deleted — its logic now lives in the endpoints plugin.

### 4. Document in `endpoints/CLAUDE.md`

Add a **Global error handling** section before the `AUTOGENERATED:BEGIN` marker:

```markdown
## Global error handling

Every `useEndpointMutation` call is covered by a global toast safety net in `shell/toaster`.
If a mutation errors and no local `onError` is provided, `getEndpointErrorMessage` is used
to extract a human-readable string (prefers `body.message`, falls back to `"HTTP <status>"`),
and `toast.error` fires automatically.

**When `onError` is provided**, the global toast is silenced automatically — no extra step needed.

**To silence the toast without a local handler** (e.g., you show the error inline via `mutation.error`):
```ts
useEndpointMutation(endpoint, { meta: { suppressError: true } });
```

**To keep the global toast AND run a local side-effect** (e.g., analytics):
```ts
useEndpointMutation(endpoint, {
  onError: (err) => analytics.track(err),
  meta: { suppressError: false },  // explicit: don't auto-suppress
});
```
```

## Verification

1. `./singularity build` — TypeScript must compile with no errors; plugin boundary check must pass.
2. Browser: trigger a mutation with no `onError` → toast appears with server's `body.message`.
3. Browser: trigger a mutation with `onError` set → only the local handler fires, no toast.
4. Browser: trigger a mutation with `onError` + `meta: { suppressError: false }` → both the handler AND the toast fire.
5. Browser: trigger a mutation with `meta: { suppressError: true }`, no `onError` → no toast, `mutation.error` populated.
