# Resources facet — render sub-plugins (Facets v3, Phase 3 slice)

## Context

A **facet** is a self-contained slice of plugin metadata. Per the v3 design
(`research/2026-06-02-global-facets-rendering-separation-v3.md`), each facet must own
its four rendering surfaces — **doc**, **detail**, **diff**, **catalog** — so that
adding/removing a facet touches no consumer. Build-time rendering (`renderDoc`) lives in
`facet/`; the three **browser** renderers live in `plugins/render-{diff,detail,catalog}/web`,
discovered by web slots. Pure projections (`toComparable`) live in `core/`.

The `exports`, `slots`, `contributions`, `cross-refs`, and `routes` facets already have
their three render sub-plugins (commits `3b2255f19`, `80ce428d5`). The **`resources`**
facet does not — its rendering still lives in consumers:
- diff: `resourceStrings()` in `compute-plugin-diff.ts:80-84`
- detail: the resources block in `public-api-section.tsx:75-89`
- catalog: `ResourcesTable` registered via the legacy `Catalog.Category` slot

This plan builds the resources facet's three browser renderers, mirroring the routes/cross-refs
precedent byte-for-byte, and moves its `toComparable` projection into `core/`. This is the
Phase-3 slice for resources only — **purely additive**. Flipping consumers to generic iteration
and deleting the old `*Strings()`/`Catalog.Category`/`publicApi` paths is Phase 4, out of scope here.

### Why this is safe / additive
- Registration is automatic: `./singularity build` regenerates `web.generated.ts` from the
  folder tree. No manual registry edits.
- `Catalog.FacetTable` contributions are registered but **not yet rendered** by `catalog-view.tsx`
  (it still iterates only `Catalog.Category`). So the new catalog table sits dormant until Phase 4.5
  — no duplicate "Resources" tab appears. This matches the current routes/cross-refs state.
- The diff slot (`PluginChanges.DiffRenderer`) and detail slot (`PluginView.Section`) likewise
  coexist with the legacy consumer paths until Phase 4.

## Facet data shape (already extracted)

`node.facets["resources"]` is `ResourceFacetData = { server: ResourceDef[]; central: ResourceDef[] }`,
`ResourceDef = { key: string; mode: string }` (mode defaults to `"push"`; entries deduped by key,
sorted alphabetically). Source: `facets/plugins/resources/core/types.ts`,
`facets/plugins/resources/facet/index.ts`.

## Changes

All paths under `plugins/plugin-meta/plugins/facets/plugins/resources/`.

### 1. `core/to-comparable.ts` (NEW) + export from `core/index.ts`
Mirror the legacy `resourceStrings()` (server before central, `"key (mode)"` format):
```ts
import type { ResourceFacetData } from "./types";

/** Diff projection: one "key (mode)" string per resource, server before central.
 *  Mirrors the legacy resourceStrings() (compute-plugin-diff.ts) for identical diff output. */
export function resourcesToComparable(data: ResourceFacetData): string[] {
  return [...data.server, ...data.central].map((r) => `${r.key} (${r.mode})`);
}
```
Add `export { resourcesToComparable } from "./to-comparable";` to `core/index.ts`.

### 2. `plugins/render-diff/` (NEW)
- `package.json` — name `@singularity/plugin-plugin-meta-facets-resources-render-diff`, the
  standard `version/private/description` fields.
- `CLAUDE.md` — prose only (build inserts the AUTOGENERATED block).
- `web/index.ts` — contribute `PluginChangesSlots.DiffRenderer({ facetId: "resources",
  label: "Resources", toComparable: (data) => resourcesToComparable(data as ResourceFacetData) })`.
  Imports `PluginChangesSlots` from `@plugins/review/plugins/plugin-changes/web`, and
  `resourcesToComparable`/`ResourceFacetData` from the facet `core`.

### 3. `plugins/render-detail/` (NEW)
- `package.json`, `CLAUDE.md` (prose).
- `web/index.ts` — contribute `PluginViewSlots.Section({ id: "resources", label: "Resources",
  component: ResourcesDetailSection })`.
- `web/components/resources-detail-section.tsx` — read `node.facets?.["resources"] as
  ResourceFacetData | undefined`; return `null` if absent or both arrays empty. Render a flat
  merged list (`[...server, ...central]`) inside `<Section title="Resources" count={...}>`
  (`Section` from `@plugins/plugin-meta/plugins/plugin-view/web`). Each row: `<code>` key,
  muted mode, and a right-aligned runtime tag (`server`/`central`) — mirroring how
  `routes-detail-section.tsx` shows `r.runtime`. Uses the server/central split the facet provides
  (richer than the old flattened `public-api-section.tsx` block). Type-only import of
  `ResourceFacetData` from the facet `core` (erased at compile time; never value-import
  `facets/core`, which drags `fs`/`path` into the bundle).

### 4. `plugins/render-catalog/` (NEW)
- `package.json`, `CLAUDE.md` (prose).
- `web/index.ts` — contribute `Catalog.FacetTable(resourcesFacetTable)` (`Catalog` from
  `@plugins/apps/plugins/forge/plugins/catalog/web`).
- `web/resources-facet-table.tsx` — `defineFacetTable<ResourceRow>({ facetId: "resources",
  label: "Resources", icon: MdStorage, columns, rows, rowKey })`. `ResourceRow = { plugin:
  PluginNode; key: string; mode: string; runtime: "server" | "central" }`. `rows(entries)`
  flattens each entry's `data.server` (runtime `"server"`) then `data.central` (runtime
  `"central"`). Columns: key (`<code>`), mode (muted), runtime (muted mono), plugin
  (`<PluginChip hierarchyId={...} />`). `rowKey = (r) => \`${r.plugin.hierarchyId}:${r.runtime}:${r.key}\``.
  Imports `defineFacetTable`/`FacetTableEntry`/`PluginChip` from the catalog `web` barrel,
  `ColumnDef` from `@plugins/primitives/plugins/data-table/web`, `PluginNode` from
  `@plugins/plugin-meta/plugins/plugin-view/core`, `MdStorage` from `react-icons/md`.
