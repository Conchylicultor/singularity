# Slots Facet Migration

## Context

The facet system is migrating plugin metadata extraction from a monolithic `collectPlugin()` function to self-contained facet sub-plugins. Steps 1 (infrastructure) and 2 (commands facet) are complete. The `slots` facet is the next natural candidate — it uses the exact same `parseDefineGroup` helper as commands, reads `web/slots.ts` instead of `web/commands.ts`, and all types/helpers are already exported from `plugin-tree/core`.

No `relate()` needed yet — the `contributors[]` computation depends on the `contributions` facet which doesn't exist yet.

## Part 1: Type the Facet interface

### Add context types to `facets/core/facets.ts`

```typescript
export interface ExtractContext { dir: string }
export interface RenderDocContext { bodyIndent: string }
```

`RelateContext` left as `unknown` — typing it requires `PluginTree` from `plugin-tree/core`, which would create a cycle. No facet uses `relate()` yet, so this can wait.

### Add `createFacet<T>()` factory to `facets/core/facets.ts`

Eliminates `unknown` at every facet definition site. The generic erases at the collection boundary via a single centralized `as Facet` cast:

```typescript
export function createFacet<T>(impl: {
  def: FacetDef<T>;
  extract: (ctx: ExtractContext) => T;
  relate?: (ctx: unknown) => void;
  renderDoc: (data: T, ctx: RenderDocContext) => string[];
}): Facet {
  return impl as Facet;
}
```

### Update `Facet` interface

Replace `unknown` contexts with the new types (data stays `unknown` — that's the erasure):

```typescript
export interface Facet {
  def: FacetDef<unknown>;
  extract: (ctx: ExtractContext) => unknown;
  relate?: (ctx: unknown) => void;
  renderDoc: (data: unknown, ctx: RenderDocContext) => string[];
}
```

### Update barrel

Export `ExtractContext`, `RenderDocContext`, `createFacet` from `facets/core/index.ts`.

### Update caller in `plugin-tree.ts`

`enrichPluginTreeDocs()` line 1093 already passes `{ dir: node.dir }` — no change needed, it matches `ExtractContext`. The `renderDoc` call site doesn't exist yet (Step 5).

### Migrate commands facet

Update `commands/facet/index.ts` to use `createFacet<CommandDef[]>()`. Remove the `as { dir: string }` and `as CommandDef[]` casts.

## Part 2: Create slots facet

### Create 3 files

All under `plugins/plugin-meta/plugins/facets/plugins/slots/`:

**`package.json`**:
```json
{
  "name": "@singularity/plugin-plugin-meta-facets-slots",
  "version": "0.0.1",
  "private": true
}
```

**`CLAUDE.md`**:
```markdown
# slots

Extracts `defineSlot()` definitions from each plugin's `web/slots.ts`.
No `relate()` yet — slot contributors will be wired once the `contributions`
facet exists.
```

**`facet/index.ts`** — uses `createFacet<SlotDef[]>()`, zero casts:

```typescript
import { join } from "path";
import { createFacet, defineFacet } from "@plugins/plugin-meta/plugins/facets/core";
import { type SlotDef, readIfExists, stripTypes, parseDefineGroup } from "@plugins/plugin-meta/plugins/plugin-tree/core";

export const slotsFacetDef = defineFacet<SlotDef[]>("slots");

export default createFacet<SlotDef[]>({
  def: slotsFacetDef,

  extract(ctx) {
    const src = readIfExists(join(ctx.dir, "web", "slots.ts"));
    if (!src) return [];
    return parseDefineGroup(stripTypes(src), "defineSlot",
      (memberName, slotId, groupName) => ({ memberName, slotId, groupName }));
  },

  renderDoc(data, ctx) {
    if (data.length === 0) return [];
    const subIndent = `${ctx.bodyIndent}  `;
    return [
      `${subIndent}- Slots: ${data.map(s => `\`${s.groupName}.${s.memberName}\``).join(", ")}`,
    ];
  },
});
```

The `renderDoc` output must match `docgen.ts` line 41 exactly: `` `${s.groupName}.${s.memberName}` ``.

### Nothing else changes

- `collectPlugin()` continues populating `node.slots` (monolithic field)
- `docgen.ts` continues reading `node.slots`
- The facet independently stores the same data in `node.facets["slots"]`
- Doc output is byte-identical
