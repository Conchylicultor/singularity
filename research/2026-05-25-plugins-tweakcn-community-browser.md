# tweakcn Community Theme Browser

## Context

The tweakcn plugin currently supports importing themes by manually pasting a theme ID or URL. There's no way to discover themes from within the app.

tweakcn's theme ecosystem has two tiers (zero overlap between them):
- **Registry** (36 curated themes): Stored in `public/r/registry.json` in the tweakcn GitHub repo. Human-readable slugs (`catppuccin`, `claude`, `vercel`, etc.) with full CSS variable data in OKLCH format. Fetchable via GitHub raw content — stable.
- **Community** (~485 user-submitted themes): CUID IDs, accessible via Next.js Server Actions (fragile action IDs change on deploys).

Both are importable via the same `GET https://tweakcn.com/r/themes/{id}` endpoint. The existing `convertTweakcnTheme` function handles the `cssVars: { theme, light, dark }` format from both sources.

**Goal**: A single unified browsable gallery inside the theme-customizer pane with mini mockup previews (tweakcn's card style), search/tags, and one-click apply.

## Approach: Single unified bundled catalog

Ship one committed JSON file merging both sources (~521 themes). Generated by a dev script that fetches registry.json from GitHub + community themes via Server Actions. The bundled catalog is the single data source at runtime — always works, no fragile dependencies.

- Registry themes are marked `source: "registry"` and get a subtle "curated" badge
- Community themes have rich tags for filtering
- Server can refresh the registry portion at runtime (stable GitHub URL); community portion only via dev script re-run

## Data Sources

Both sources are fetched by the dev script and merged into a single `catalog.json`.

### Registry (stable, refreshable at runtime)

```
GET https://raw.githubusercontent.com/jnsahaj/tweakcn/main/public/r/registry.json
```

Returns an array of theme objects with `cssVars: { theme, light, dark }` — the exact format `convertTweakcnTheme` already consumes.

### Community (dev script only, fragile action IDs)

Fetched via tweakcn's Next.js Server Actions (paginated, 20/page). Returns themes with `styles: { light, dark }` (flat — mode-independent values like radius/fonts are present in both light and dark). The dev script reconstructs the `cssVars.theme` layer by extracting mode-independent keys.

## Data Model

### `CatalogTheme` (unified type — single entry point for both sources)

```ts
export interface CatalogTheme {
  id: string;            // slug for registry ("catppuccin"), CUID for community
  name: string;          // display name
  tags: string[];        // community: rich tags (47 categories); registry: empty or inferred
  source: "registry" | "community";
  likeCount?: number;    // community only, used for default sort
  author?: string;       // community only
  cssVars: {
    theme: Record<string, string>;   // mode-independent (fonts, radius, shadows)
    light: Record<string, string>;   // light mode CSS vars (oklch)
    dark: Record<string, string>;    // dark mode CSS vars (oklch)
  };
}
```

For community themes, the dev script reconstructs `cssVars.theme` from the flat `styles` by extracting mode-independent keys (radius, font-sans, font-mono, font-serif, spacing). For registry themes, `cssVars` comes directly from the source JSON — no transformation needed.

## Server Endpoints

### `GET /api/tweakcn/community/catalog`

Returns the unified catalog. Server reads from the bundled `catalog.json`.

```ts
response: z.object({
  themes: z.array(CatalogThemeSchema),
})
```

### `POST /api/tweakcn/community/apply`

Applies a theme from the catalog. Looks up by ID, converts `cssVars` → presets, upserts into `tweakcn_themes`, returns the stored theme.

```ts
body: z.object({ themeId: z.string() })
response: TweakcnThemeSchema
```

Flow:
1. Look up theme in catalog by `id`
2. Call `convertTweakcnTheme(theme.cssVars)` (reuses existing shared function)
3. Upsert into `_tweakcnThemes` table (same pattern as existing `handle-import.ts`)
4. Return the stored `TweakcnTheme`

## Web UI

### Theme Card (mini mockup — replicating tweakcn style)

tweakcn's own `ThemePreview` component uses:
- Colored background (the theme's `background` color)
- 6 vertical color bars top-right: primary, secondary, accent, muted, border, card
- Theme name bottom-left in the theme's font

We replicate this approach:

```
┌─────────────────────────────┐
│ bg: theme.background        │
│                ┃┃┃┃┃┃       │  ← 6 color bars (w-2 h-8)
│                             │     primary, secondary, accent,
│  "Theme Name"               │     muted, border, card
│  ᵗᵃᵍˢ                      │
└─────────────────────────────┘
```

- Card: `rounded-lg border overflow-hidden h-24 w-full cursor-pointer`
- Background: inline `style={{ backgroundColor: theme.cssVars.light.background }}`
- Color bars: 6 thin vertical rectangles using border-radius from theme
- Theme name: `text-sm font-medium` in `color: theme.cssVars.light.foreground`
- Tags: 1-2 tag pills truncated (if community theme)
- Hover: scale + shadow transition
- Applying state: opacity-50 + spinner

## Apply Flow

1. User clicks a theme card
2. Web: `applyMutation.mutate({ body: { themeId: theme.id } })`
3. Server `handle-apply.ts`:
   - Looks up theme in merged catalog by `id`
   - Calls `convertTweakcnTheme(theme.cssVars)` → gets `Record<groupId, { light, dark }>`
   - Upserts into `_tweakcnThemes` with `tweakcnId = theme.id`, `label = theme.name`
   - Returns full `TweakcnTheme` object
4. Web `onSuccess(result)`:
   - Calls `handleApply(result.tweakcnId, result.presets)` — fans out `setConfigField` for each token group (same pattern as existing `TweakcnSection`)
5. `invalidates: [listTweakcnThemes]` → parent "tweakcn" section refreshes, showing the imported theme
6. CSS vars update immediately via the config → ThemeInjector pipeline

## Dev Script

### `scripts/fetch-catalog.ts`

Single script that fetches both sources and outputs `shared/catalog.json`:

```ts
// 1. Registry: fetch from GitHub (stable)
const registry = await fetch(
  "https://raw.githubusercontent.com/jnsahaj/tweakcn/main/public/r/registry.json"
).then(r => r.json());
// Transform: add source:"registry", map name→id, title→name

// 2. Community: fetch via Server Actions (fragile — update action IDs if broken)
// Paginated loop: POST to tweakcn.com/community with Next-Action header
// 20 themes/page, ~25 pages
// Transform: add source:"community", reconstruct cssVars.theme from flat styles

// 3. Merge and deduplicate, write catalog.json
```

Run manually: `bun plugins/ui/plugins/tweakcn/plugins/community-browser/scripts/fetch-catalog.ts`
