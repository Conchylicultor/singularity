# skill-listing attachment renderer sub-plugin

## Context

The `attachment` umbrella plugin dispatches attachment JSONL events to sub-plugin renderers by matching on `subtype`. The `skill_listing` subtype (skills available at session start) currently falls through to a generic JSON dump. This plan adds a dedicated renderer following the same pattern as the three existing sub-plugins (`deferred-tools-delta`, `task-reminder`, `nested-memory`).

## Payload

```ts
interface SkillListingPayload {
  type: "skill_listing";
  content: string;       // "- name: description\n- name2: ...\n"
  skillCount: number;
  isInitial: boolean;    // always true (injected at session start)
}
```

`content` is newline-delimited, each line `- name: description`. Multi-line descriptions exist (e.g. `claude-api` has TRIGGER/SKIP blocks) — only the first line matters for the collapsed list view.
