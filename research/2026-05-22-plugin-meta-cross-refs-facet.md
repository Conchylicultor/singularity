# Cross-refs facet sub-plugin

## Context

Cross-plugin import tracking (`apiUses` for all 5 runtimes) is extracted monolithically by `parseServerApiUses()` inside `collectPlugin()` (plugin-tree.ts:704-717), and the `importedBy` reverse index is computed in `computeRelationships()` (plugin-tree.ts:788-800). These belong in a facet sub-plugin following the same pattern as the existing `commands` and `slots` facets.

This is the **first facet to use `relate()`** ‚Äî commands and slots have no cross-plugin wiring.

## Plan

### Step 1 ‚Äî Scaffold the plugin

Create `plugins/plugin-meta/plugins/facets/plugins/cross-refs/`:

```
cross-refs/
  package.json      ‚Äî @singularity/plugin-plugin-meta-facets-cross-refs
  CLAUDE.md
  facet/
    index.ts         ‚Äî the facet implementation
```

**package.json**: same shape as `commands` ‚Äî name, version 0.0.1, private.

### Step 2 ‚Äî Implement `facet/index.ts`

**Data type:**
```ts
export interface CrossRefsData {
  apiUses: Record<"server" | "central" | "web" | "core" | "shared", string[]>;
  importedBy: string[];
}
```

Normalizes the `PluginNode` asymmetry (server/central inside `RuntimeDetail` vs flat web/core/shared) into a uniform per-runtime record.

**Local helpers:**
- `walkFiles(dir, out)` ‚Äî 16-line recursive dir walker copied from plugin-tree.ts:376-392 (not exported from plugin-tree/core). Must preserve the `node_modules` + `plugins` directory skip.
- `parseApiUses(runtimeDir, selfName, runtime)` ‚Äî mirrors `parseServerApiUses` (plugin-tree.ts:419-461). Uses `readIfExists` imported from `@plugins/plugin-meta/plugins/plugin-tree/core`.

**`extract(ctx)`:** Calls `parseApiUses` for each of the 5 runtime dirs that exist. Returns `{ apiUses: { server, central, web, core, shared }, importedBy: [] }`.

**`relate(ctx)`:** Casts ctx as `{ tree: PluginTree }`. Iterates all nodes, unions each node's apiUses across runtimes, extracts plugin names via `.split(".")[0]`, and populates each target's `importedBy` array (via `getFacet`). Sorts all `importedBy` arrays. Same logic as `computeRelationships` lines 788-800 but operating on facet data.

**`renderDoc(data, ctx)`:** Returns flat markdown lines ‚Äî one per non-empty runtime Uses and a final importedBy line. Currently dead code (no caller invokes `renderDoc`), but designed for future integration.

### Step 3 ‚Äî No manual registry edit

`facet.generated.ts` is autogenerated. Running `./singularity build` will discover the new `facet/index.ts` and add the entry automatically.

### Step 4 ‚Äî No changes to monolithic code or consumers

Dual-write: the monolithic extraction in `collectPlugin()` and the importedBy computation in `computeRelationships()` stay as-is. Consumers (`docgen.ts`, `tree-handler.ts`, `compute-plugin-diff.ts`) continue reading from `PluginNode` fields unchanged. The facet independently extracts the same data into `node.facets["cross-refs"]` for future consumer migration.
