# Plan A — DataView embedded / auto-height mode

> Part 1 of 2. Pairs with **Plan B**
> (`2026-06-15-fields-tags-filter-and-community-browser-migration.md`), which
> consumes this. This plan is the primitive change only — no consumer edits.

## Context

`primitives/data-view` is a Notion-like multi-view surface. It only works as a
**full surface**: its root is `flex min-h-0 flex-1 flex-col` with an inner body
`min-h-0 flex-1 overflow-y-auto`, so it assumes a bounded-height flex parent and
owns its own scroll. Dropped into an **auto-height** context (a section stacked
among siblings in a vertically-scrolling pane), the inner `min-h-0 flex-1
overflow-y-auto` resolves to ~0px and the view collapses.

We add a generic **`mode: "surface" | "embedded"`** axis so `DataView` can be
embedded in arbitrary host surfaces: content grows to its natural height and the
host pane scrolls. This is a reusable primitive capability; the first consumer
(the tweakcn community-browser) is migrated in Plan B.

**Scope decision:** no "headless toolbar" axis. `DataView` keeps owning its
toolbar. This plan is *only* the placement/height axis plus threading it into
views for padding.

## Changes

### 1. Core types — `plugins/primitives/plugins/data-view/core/internal/types.ts`

- `DataViewProps<TRow>` (~L177): add
  ```ts
  /**
   * Placement / height. `"surface"` (default): fills a bounded-height flex
   * ancestor, owns its internal scroll, reserves the floating-action-bar gutter.
   * `"embedded"`: grows to natural content height, no internal scroll, no gutter —
   * the host pane scrolls. Use when stacked among siblings in a scrolling page.
   */
  mode?: "surface" | "embedded";
  ```
- `DataViewRenderProps<TRow>` (~L102): add
  ```ts
  /** True when the host is embedded (auto-height); views drop full-surface
   *  outer padding / forced heights. */
  embedded?: boolean;
  ```

### 2. Host — `plugins/primitives/plugins/data-view/web/components/data-view.tsx`

- Destructure `mode = "surface"`; derive `const embedded = mode === "embedded";`.
- **Both root sites** — the no-active-view fallback (~L105) and the main return
  (~L145): `flex min-h-0 flex-1 flex-col` → `cn("flex flex-col", !embedded && "min-h-0 flex-1")`.
- **Toolbar row** (~L150): drop the `pr-14` floating-action-bar gutter in embedded.
  Keep the existing `eslint-disable-next-line spacing/no-adhoc-spacing` only on the
  surface-mode `pr-14` path:
  `cn("flex shrink-0 items-center gap-sm pb-sm pl-sm", !embedded && "pr-14")`.
- **Body wrapper** (~L188): `cn("min-h-0 flex-1 overflow-y-auto")` →
  `cn(!embedded && "min-h-0 flex-1 overflow-y-auto")` (embedded → plain
  content-height div).
- Add `embedded` to the `renderProps` object (~L120-142) passed to `renderIsolated`.

The toolbar and filter-bar stay rendered in both modes (no headless axis).

### 3. Gallery view — `plugins/primitives/plugins/data-view/plugins/gallery/web/components/gallery-view.tsx`

`cn` is **not** currently imported here — add
`import { cn } from "@plugins/primitives/plugins/ui-kit/web";`.

- **Grid** (~L133): `"grid gap-lg p-xl"` →
  `cn("grid gap-lg", !props.embedded && "p-xl")` (keep the inline
  `gridTemplateColumns` style; `gap-lg` always retained).
- **Empty-state** (~L121): drop `h-full` when embedded (it collapses to 0). Use a
  content-height centered block — e.g.
  `cn("flex items-center justify-center text-muted-foreground", props.embedded ? "py-xl" : "h-full p-xl")`.

### 4. table / tree views — no change (verified)

- `table-view.tsx`: renders `<DataTable>` (a grid with a `sticky top-0` header) and
  has no `h-full`/`flex-1`/own padding to drop. In embedded mode the sticky header
  resolves against the host scroll container — correct. No change.
- `tree-view.tsx`: wraps `<TreeList>` in `<div className="px-sm">` (horizontal
  indentation, not vertical whitespace) with no `overflow`/`min-h`/`flex-1`. Grows
  to natural height; DnD overlay is fixed-position and unaffected. No change.

Documented as a known no-op: gallery is the only view that needs embed-aware
padding today; future view children should gate full-surface padding on
`!props.embedded`.

### 5. Docs — `plugins/primitives/plugins/data-view/CLAUDE.md`

Add a hand-authored `## Placement mode` section (above the autogenerated block)
documenting the two `mode` values and the `DataViewRenderProps.embedded` contract
(views gate full-surface padding on `!embedded`). Do not touch the
`<!-- AUTOGENERATED -->` block — `./singularity build` rewrites it.

## Verification

- `mode` defaults to `"surface"`; no existing consumer passes it → unchanged
  behavior. Spot-check real consumers render identically: agents list
  (`views={["tree"]}`, `plugins/conversations/plugins/agents/web/components/agents-list.tsx`),
  and a gallery/table consumer (e.g. Sonata library `views={["gallery","table"]}`).
- `./singularity build` then `./singularity check` (type-check +
  `plugins-doc-in-sync`). No new exports are introduced (`mode`/`embedded` land on
  already-exported interfaces), so the autogenerated reference blocks shouldn't drift.
- Full *visual* verification of embedded mode happens in Plan B (community-browser
  is the first `mode="embedded"` consumer).
