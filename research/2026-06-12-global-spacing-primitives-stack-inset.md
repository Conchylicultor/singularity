# Spacing primitives: `<Stack>` / `<Inset>` + density spacing scale + `no-adhoc-spacing`

**Date:** 2026-06-12 · **Category:** global (primitives + tokens/density + ui-kit + lint)

## Context

Across the app every plugin sets its own padding/gap/margin with raw Tailwind
(`p-*`, `gap-*`, `space-*`, `m-*`). Tailwind's spacing scale exposes ~20+
continuous steps per axis (`gap-0…gap-12` plus half-steps and `gap-[7px]`), so
parallel agents each pick a slightly different value and spacing drifts —
cramped, uneven, no shared rhythm. There is **no chokepoint** forcing layout
through a closed set, and **no indirection layer for gaps at all** (padding has
`p-row`/`p-chip`/`p-control`; gap has nothing — every `gap-*` is a raw Tailwind
numeric that does **not** scale with the density preset).

**What this buys us (the scale values seed 1:1 with today's dominant usage on
purpose — the rename is a neutral migration tactic, not the value):**

1. **Cardinality collapse.** ~20 continuous Tailwind steps → **8 enumerated
   roles**. `gap-1.5`/`gap-2.5`/`gap-5`/`gap-[7px]` become *illegal* and collapse
   into the nearest role. You can't drift across a value the vocabulary doesn't
   contain.
2. **Lint chokepoint = durability.** The `no-adhoc-spacing` rule makes the closed
   set the *only* legal path, so new parallel-written UI can't reintroduce raw
   spacing. This is what prevents *future* drift (the actual problem).
3. **Re-tuning seam.** Once every small gap is `gap-sm`, the design system can
   re-tune all small gaps from one token — impossible today where `gap-2`
   conflates "small-gap role" with "exactly 8px".
4. **Density-responsive gaps.** Backed by `--space-*` runtime tokens, the 1:1
   mapping holds *only at Comfortable*; at Cozy/Compact gaps tighten with controls
   and pads (today `gap-2` is `0.5rem` regardless of density, breaking rhythm).

Follows the established **primitive + token + lint** pattern:
`text`/`no-adhoc-typography`, `control-size`/`no-adhoc-control`,
`z-layers`/`no-adhoc-zindex`. The **`text` plugin is the byte-for-byte template**
(runtime component in `web/internal/` → `@utility` classes in `app.css` sourced
from a token group → `lint/no-adhoc-*` banning the raw form).

## Decisions (confirmed with user)

- **Scale shape:** scalar ramp `none | 2xs | xs | sm | md | lg | xl | 2xl`
  (mirrors `control-size`'s xs/sm/md/lg; maps cleanly onto `gap-1`/`gap-2`/`gap-4`).
- **Migration:** ship infra + **grandfather** — seed the BURNDOWN allowlist with
  every current violator so the rule bites only new/touched code. Migrate the
  ~1,181 sites later as a separate drain (how `no-adhoc-typography`/`-loading-text`
  landed). **No mass rewrite in this change.**
- **Lint coverage:** ban raw **gap + padding + margin + space** (numeric +
  arbitrary). Gaps→`<Stack>`, padding→`<Inset>`, margins discouraged (restructure).

## The spacing scale (added to the `density` token group)

New 1-D ramp on `densityGroup` so it lives under the existing Comfortable/Cozy/
Compact presets and scales with density (control heights & pads already do).
`none`/`0` needs no token (constant `0` in the utility). Comfortable seed values
chosen to coincide with today's dominant raw usage:

| Step  | CSS var       | Comfortable | Replaces (raw) |
| ----- | ------------- | ----------- | -------------- |
| `2xs` | `--space-2xs` | 0.125rem    | `*-0.5`        |
| `xs`  | `--space-xs`  | 0.25rem     | `*-1`          |
| `sm`  | `--space-sm`  | 0.5rem      | `*-2`          |
| `md`  | `--space-md`  | 0.75rem     | `*-3`          |
| `lg`  | `--space-lg`  | 1rem        | `*-4`          |
| `xl`  | `--space-xl`  | 1.5rem      | `*-6`          |
| `2xl` | `--space-2xl` | 2rem        | `*-8`          |

Cozy/Compact step the values down proportionally (same ~1-tier-per-preset cadence
as the existing pads; exact rem values finalized in impl).

## Files to change

### 1. Token scale — `plugins/ui/plugins/tokens/plugins/density/`
- `shared/group.ts` — add `space2xs … space2xl` keys to `defineTokenGroup("density", …)`
  with `default` + `label`. `DensityTokenValues` picks them up automatically.
- `web/presets.ts` — add the 7 values to **all three** presets (the `both()`
  helper covers light/dark).
- CLAUDE.md autoregenerated by build.

### 2. `@utility` bridges — `plugins/primitives/plugins/ui-kit/web/theme/app.css`
Padding utilities can't read runtime theme vars, so each class is an explicit
`@utility` (same reason as the existing `p-chip`/`p-row` block). Add families over
the 8 steps (incl. `*-none` → `0`):
- **Gap:** `gap-<step>`, `gap-x-<step>`, `gap-y-<step>` → `gap`/`column-gap`/`row-gap: var(--space-<step>)`
- **Padding:** `p-<step>`, `px-<step>`, `py-<step>`, `pt-/pr-/pb-/pl-<step>`

Margins get **no** named utilities (the stance is "no margins — restructure to
Stack/Inset"); raw numeric/arbitrary margins are banned, `mx-auto`/`my-auto`
survive naturally (word-valued). `space-x/y` is replaced by `<Stack>` (no named
`space-*` utility).

### 3. twMerge registry — `plugins/primitives/plugins/ui-kit/web/theme/custom-utilities.ts`
- Add `GAP_*`/`PAD_STEP_*` name arrays (suffix `_UTILITIES`, parsed by the sync check).
- Register each family via `extend` into its matching builtin group id (mirrors
  `px-chrome` → `extend: "px"`), giving mutual conflict with raw Tailwind + with
  the existing `sg-pad` (`p-chip`/`p-row`) for free.
- Extend `BuiltinGroupId` with the needed ids (`gap`, `gap-x`, `gap-y`, `py`,
  `pt`, `pr`, `pb`, …). `cn`/twMerge config derives automatically (`web/lib/utils.ts`).
- The `app-css-utilities-in-sync` check enforces app.css ↔ registry parity.

### 4. New primitive plugin — `plugins/primitives/plugins/spacing/`
Mirror the `text` plugin layout exactly:
```
spacing/
  package.json        # @singularity/plugin-primitives-spacing, private, 0.0.1
  CLAUDE.md
  web/index.ts        # export Stack, Inset, types; default PluginDefinition (no contributions)
  web/internal/stack.tsx
  web/internal/inset.tsx
  lint/index.ts       # default { name:"spacing", rules, ignores }
  lint/no-adhoc-spacing.ts
```
- **`<Stack>`** — `direction "col"|"row"` (default col), `gap: SpaceStep`, optional
  `align`/`justify`/`wrap`, `as`. Emits `flex` + `flex-col?` + `gap-<step>` via a
  `Record<SpaceStep,string>` (same shape as `Text`'s `VARIANT_CLASS`). align/justify
  are non-spacing (`items-*`/`justify-*`) and pass through. Replaces
  `flex flex-col gap-*` and `space-y-*`.
- **`<Inset>`** — `pad?`/`x?`/`y?`/`t?`/`r?`/`b?`/`l?`: `SpaceStep`, `as`. Maps to
  `p-/px-/py-/pt-…<step>`. Replaces `p-*`/`px-*`/`py-*`.
- `SpaceStep = "none"|"2xs"|"xs"|"sm"|"md"|"lg"|"xl"|"2xl"` exported from the barrel.

### 5. Lint rule — `lint/no-adhoc-spacing.ts`
Copy `no-adhoc-typography.ts` structure verbatim (`collectTokens` + `baseClass`,
`JSXAttribute` on `className`/`class` + `CallExpression` on `cn`/`clsx`/`twMerge`).
After stripping variant prefixes **and** a leading `-` (negative margins), ban:
- gap: `/^gap(-[xy])?-(\d|\[)/`
- padding: `/^p[xytrbl]?-(\d|\[)/`
- margin: `/^m[xytrbl]?-(\d|\[)/`  (so `mx-auto`, `m-px` are *not* matched)
- space: `/^space-[xy]-(\d|\[)/`

Named `*-<step>` classes (word-valued) never match — exactly the `z-base` vs
`z-10` distinction. No auto-fix (picking a role is a judgement). Message points to
`<Stack gap>`/`<Inset pad>` + the named-utility escape.

### 6. BURNDOWN seed — `lint/index.ts`
`ignores: { "no-adhoc-spacing": [ …every current violator… ] }`. Bootstrap the
list: wire the plugin, `./singularity build`, run `./singularity check eslint`,
capture failing file paths, paste them in, rebuild until green. ~378 files —
large but precedented; header comment marks it BURNDOWN-to-drain, "do NOT add new
entries." Single-site escapes use
`// eslint-disable-next-line spacing/no-adhoc-spacing -- <reason>`.

### 7. Registration
`./singularity build` regenerates `lint.generated.ts` (discovers `lint/index.ts`),
the web plugin registry, and docs. No manual registry edits. The `spacing` plugin
is **not** added to `web/src/plugins.ts` unless it must mount a provider — it has
no contributions (pure exports), like `text`.

## Out of scope (explicit)
- Migrating the ~1,181 existing raw-spacing sites (separate burndown effort).
- A density-customizer UI row for the new tokens (they ride the existing density
  section; can be added later).
- Negative-margin / `gap-px` exotic cases — rare, handled per-site via escape.
