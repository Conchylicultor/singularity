# primitives/persistent-draft ŌĆö Implementation Plan

## Context

Draft text for in-progress prompts is lost on page reload for any surface using plain `useState`. The conversation prompt works around this via a bespoke ~120-line React context/provider (`prompt-draft-context.tsx`) that stores all conversation drafts under a single localStorage key (`singularity:prompt-drafts-v2`). The context also shares a single live React state across three sibling components (`PromptInput`, `ForkSessionButtons`, `ForkConversationButtons`) that each need to read the same draft.

This plan replaces the bespoke context with a reusable `useDraft` primitive that:
- persists any JSON-serializable value to localStorage with TTL auto-expiry
- synchronises all `useDraft` calls sharing the same storage key within the same tab (replacing the context's state-sharing role)
- also syncs across tabs via the native `storage` event

**Scope:** new `plugins/primitives/plugins/persistent-draft/` plugin + migration of `conversation-view` off `prompt-draft-context.tsx`. `TaskDraftPopover` migration is explicitly out of scope.

## Design

### Hook signature

```ts
function useDraft<T>(
  key: string,
  initialValue: T | (() => T),
  options?: { scope?: string; ttl?: number }
): [T, Dispatch<SetStateAction<T>>, () => void]
```

- `key` ŌĆö logical name (e.g. `'conversation:prompt'`)
- `scope` ŌĆö optional entity ID appended to the storage key (e.g. a `conversationId`)
- `ttl` ŌĆö milliseconds; default 7 days; drafts older than TTL are treated as absent on read
- Returns `[value, setter, clearDraft]` ŌĆö setter has the same signature as `useState`'s setter, supporting functional updates

### Storage key format

`singularity:draft:<key>` or `singularity:draft:<key>:<scope>` when `scope` is provided.

Each key stores an envelope: `{ v: T, ts: number }` where `ts = Date.now()` at write time, enabling TTL enforcement on read.

### Cross-hook same-tab sync

The old context shared a single React state. The new primitive replaces this by dispatching a custom window event `singularity:draft-updated` with `{ detail: { storageKey } }` on every write. All other `useDraft` instances with the same `storageKey` re-read from localStorage and call `setState`. The native `storage` event handles cross-tab sync. Both are handled in a single `useEffect` with cleanup.

### Write timing

Synchronous writes (no debounce) ŌĆö matches current behaviour. Debouncing risks losing the final keystroke on abrupt tab close.

### Conversation-view simplification

`PromptDraft = { markdown: string }` is replaced by plain `string` everywhere ŌĆö removes unnecessary wrapping. `isDraftEmpty` and `draftToPlainText` are updated to accept `string` instead of `PromptDraft`. They move to a new `web/prompt-draft-utils.ts` and remain on the public barrel. `EMPTY_DRAFT` is removed; callers use `''`.

### Known data loss

The old `singularity:prompt-drafts-v2` key becomes orphaned after deploy. Any in-flight drafts are lost once. Acceptable; no migration code needed.

`use-draft.ts` implementation notes:
- Read envelope on mount; return `initialValue` if missing or `Date.now() - ts > ttl`
- `useState` initialised with that value
- Setter: calls `setState` + writes envelope to localStorage + dispatches `singularity:draft-updated` CustomEvent
- `clearDraft`: removes key from localStorage + dispatches event + resets state to `initialValue`
- `useEffect`: listens for `singularity:draft-updated` (same-tab) and `storage` (cross-tab); on match re-reads and calls `setState`
