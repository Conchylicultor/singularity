# Docgen: `docs/routes.md` generation

## Context

`./singularity build` already runs `generatePluginDocs`, which writes `docs/plugins-compact.md`, `docs/plugins-details.md`, and per-plugin `CLAUDE.md` files. Route data (`httpRoutes`, `wsRoutes`, `centralHttpRoutes`, `centralWsRoutes`) is already collected per plugin in `PluginInfo` by `parseRouteMap`. There is no dedicated cross-plugin route index — routes are buried inside the large `plugins-details.md` and the per-plugin CLAUDE.md files.

Goal: add a focused `docs/routes.md` that surfaces every `/api/…` and WebSocket route as a plugin tree, filtered to only plugins (and plugin branches) that expose at least one route.

---

## Design

### Output format — `docs/routes.md`

```markdown
# Plugin Routes

All HTTP and WebSocket routes exposed by server and central plugins.
Only plugins with at least one route (directly or in a descendant) are shown.

<!-- AUTOGENERATED:BEGIN — do not edit; regenerated by `./singularity build` -->

- **`agents`** — Named agent definitions...
  - `/api/agents` (GET, POST)
  - `/api/agents/:id` (GET, PATCH, DELETE)
  - `/api/agents/:id/launch` (POST)
  - `/api/agents/:id/launches` (GET)
  - Plugins:
    - **`auto-launch`**
      - Plugins:
        - **`toggle`** — Toggle on/off...
          - `/api/agents-ext-auto-launch/:id` (GET)
- **`auth`** — Shared authentication infrastructure...
  - `/api/auth/callback` (GET)
  - `/api/auth/logout` (POST)
  - Plugins:
    - **`google`** — Google OAuth provider...
      - `/api/auth/google` (GET)

<!-- AUTOGENERATED:END -->
```

Rules:
- **Filter**: A plugin node is included only if it has routes directly OR has a descendant with routes.
- **Structure**: Tree mirrors the `plugins/` hierarchy (same parent→children nesting as the other docs).
- **Grouping**: HTTP routes are grouped by path, with methods listed inline as `(GET, POST)`. Each distinct path = one line.
- **WS routes**: Rendered as `/path` (WS) — the stored keys have no method prefix.
- **Central routes**: Same grouping, with `_(central)_` appended to each line.
- **Indent model**: Matches `renderPluginTree` exactly — `"  ".repeat(depth)` for the bullet, +2 spaces for body, `depth + 2` passed to child recursion.

**Add tree predicate** (new helper, near other rendering helpers):

```ts
function pluginHasRoutesDeep(p: PluginInfo): boolean {
  return (
    p.httpRoutes.length > 0 ||
    p.wsRoutes.length > 0 ||
    p.centralHttpRoutes.length > 0 ||
    p.centralWsRoutes.length > 0 ||
    p.children.some(pluginHasRoutesDeep)
  );
}
```

**Add grouping helper + renderer** (new functions, after `renderPluginTree`):

```ts
/** Groups ["GET /api/foo", "POST /api/foo", "GET /api/bar"] → ["/api/foo (GET, POST)", "/api/bar (GET)"] */
function groupHttpRoutes(routes: string[]): string[] {
  const byPath = new Map<string, string[]>();
  for (const r of routes) {
    const sp = r.indexOf(" ");
    const method = r.slice(0, sp);
    const path = r.slice(sp + 1);
    (byPath.get(path) ?? (byPath.set(path, []), byPath.get(path)!)).push(method);
  }
  return [...byPath.entries()].map(([path, methods]) => `${path} (${methods.join(", ")})`);
}

function renderRoutesPluginTree(p: PluginInfo, depth: number): string[] {
  if (!pluginHasRoutesDeep(p)) return [];

  const indent = "  ".repeat(depth);
  const bodyIndent = `${indent}  `;
  const lines: string[] = [];

  const desc = p.description ? ` — ${p.description}` : "";
  lines.push(`${indent}- **\`${p.name}\`**${desc}`);

  for (const r of groupHttpRoutes(p.httpRoutes))        lines.push(`${bodyIndent}- \`${r}\``);
  for (const r of p.wsRoutes)                           lines.push(`${bodyIndent}- \`${r} (WS)\``);
  for (const r of groupHttpRoutes(p.centralHttpRoutes)) lines.push(`${bodyIndent}- \`${r}\` _(central)_`);
  for (const r of p.centralWsRoutes)                   lines.push(`${bodyIndent}- \`${r} (WS)\` _(central)_`);

  const routeChildren = p.children.filter(pluginHasRoutesDeep);
  if (routeChildren.length > 0) {
    lines.push(`${bodyIndent}- Plugins:`);
    for (const c of routeChildren)
      lines.push(...renderRoutesPluginTree(c, depth + 2));
  }

  return lines;
}
```

**Add internal renderer** (after `renderDetailsDocFromTree`):

```ts
function renderRoutesDocFromTree(tree: PluginTree, root: string): string {
  const roots = tree.roots.filter(pluginHasRoutesDeep);
  const body = roots
    .map((p) => renderRoutesPluginTree(p, 0).join("\n"))
    .join("\n\n");
  return `${ROUTES_HEADER}${BEGIN}\n\n${body}\n${END}\n`;
}
```

**Add exported render function** (after `renderDetailsDoc`, line ~1092):

```ts
export function renderRoutesDoc({ root }: GenerateDocsOptions): string {
  return renderRoutesDocFromTree(buildPluginTree(root), root);
}
```

**Wire into `generatePluginDocs`** (add one line after the two existing `writeIfChanged` calls, line ~1172):

```ts
writeIfChanged(pluginRoutesDocPath(root), renderRoutesDocFromTree(tree, root));
```

### 2. `cli/src/checks/plugins-doc-in-sync.ts`

Import the two new exports and add a third check block, mirroring the existing `compactFile`/`detailsFile` pattern:

```ts
import {
  // existing imports ...
  pluginRoutesDocPath,
  renderRoutesDoc,
} from "../docgen";

// In pluginsDocInSync.run():
const routesFile = pluginRoutesDocPath(root);
if (!existsSync(routesFile)) {
  return {
    ok: false,
    message: "docs/routes.md is missing",
    hint: "Run `./singularity build` to generate it.",
  };
}
if (readFileSync(routesFile, "utf8") !== renderRoutesDoc({ root })) {
  return {
    ok: false,
    message: "docs/routes.md is out of sync with plugin source",
    hint: "Run `./singularity build` and commit the regenerated file.",
  };
}
```

Also update the `description` string on `pluginsDocInSync` to mention `docs/routes.md`.
