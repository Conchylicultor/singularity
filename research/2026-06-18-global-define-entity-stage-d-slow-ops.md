## Context

`slow_ops` still declares its columns in **three hand-synced places**:

1. the Drizzle table — `plugins/debug/plugins/slow-ops/server/internal/tables.ts`
2. the zod wire schema — `plugins/debug/plugins/slow-ops/core/resources.ts`
3. the loader — `plugins/debug/plugins/slow-ops/server/internal/resources.ts`

The Stage-0 interim fix already deleted the loader's row projection (`db.select()`
returns rows verbatim) and added a compile-time `Equal<$inferSelect, SlowOp>` guard
so drift between the table and the schema is a loud `tsc` error rather than a
silently-dropped column (`recentSamples` was lost this way once). But the table and
schema are still **two hand-written artifacts** that the guard merely *compares* — it
doesn't make them derive from one source.

Now that `defineEntity(name, fields, meta)` exists, we can re-express `SlowOp` as a
**single field record**. The Drizzle table, the zod wire schema, and the row type all
derive from that one record, so the silent-column-drop footgun becomes structurally
**unrepresentable** — and the interim `Equal` guard becomes vacuous and is deleted.

**Hard requirement: zero schema drift.** `./singularity build` must generate **no new
migration**. The Stage C unit test (`define-entity.test.ts`) already proves
`defineEntity` reproduces the exact `slow_ops` DDL (snake_case names, `notNull`,
defaults, `double precision`, `jsonb`, `timestamptz`, the unique index), so this is
expected to hold; we verify it empirically.

## Approach

### 1. Add `uuidField` / `dateField` factories (in the field type's own `core/`)

The field record needs factories for `uuid` and `date`; only `text`/`int`/`float`/
`json`/`bool` have them today. Per the decision, these are **core-pure field-atom
constructors** added directly to each type's `core/` (alongside its identity) — *not*
new `config` sub-plugins. uuid/date have no meaningful config-render capability
(uuid is auto-generated; a date is never hand-configured), and the
`config-v2.fields.renderer` dispatch slot already falls back gracefully for any type
with no renderer (`plugins/config_v2/plugins/fields/web/internal/slots.tsx` keys on
`field.type.id` with a `fallback`), so registering none is correct, not incomplete.

Mirror the `textField`/`floatField` shape exactly (`Object.freeze`, `pickMeta(opts)`):

- **`plugins/fields/plugins/uuid/core/internal/uuid.ts`** — add:
  ```ts
  export interface UuidFieldDef extends FieldDef<string> {
    readonly type: typeof uuidFieldType;
  }
  export function uuidField(opts?: FieldMeta & { default?: string }): UuidFieldDef {
    return Object.freeze({
      type: uuidFieldType,
      schema: z.string().uuid(),         // preserves the current wire contract
      defaultValue: opts?.default ?? "",
      meta: pickMeta(opts),
    });
  }
  ```
- **`plugins/fields/plugins/date/core/internal/date.ts`** — add:
  ```ts
  export interface DateFieldDef extends FieldDef<Date> {
    readonly type: typeof dateFieldType;
  }
  export function dateField(opts?: FieldMeta & { default?: Date }): DateFieldDef {
    return Object.freeze({
      type: dateFieldType,
      schema: z.coerce.date(),           // LOAD-BEARING — see "z.coerce.date()" below
      defaultValue: opts?.default ?? new Date(0),
      meta: pickMeta(opts),
    });
  }
  ```
- Re-export both from `plugins/fields/plugins/{uuid,date}/core/index.ts`
  (`uuidField`/`UuidFieldDef`, `dateField`/`DateFieldDef`).

### 2. Field record + derived wire schema in `slow-ops/core`

`defineEntity` is server-only (imports `drizzle-orm/pg-core`), so the **field record
lives in core** — the only layer reachable by both the server (to build the table) and
web consumers (to read the schema). Core derives the wire schema from it; the server
reads the *same* record to build the table.

In **`plugins/debug/plugins/slow-ops/core/resources.ts`**:

- Keep `CallerBreakdownSchema` / `SlowOpSampleSchema` (+ their types) unchanged — these
  are **nested JSON value schemas**, not DB columns; they become the `json` fields'
  value schemas.
- Replace the hand-written `SlowOpSchema = z.object({...})` with a field record + its
  derivation:
  ```ts
  export const slowOpFields = {
    id:            uuidField(),
    worktree:      textField(),
    operationKind: textField(),
    operation:     textField(),
    count:         intField(),
    totalMs:       floatField(),
    maxMs:         floatField(),
    lastMs:        floatField(),
    thresholdMs:   floatField(),
    callers:       jsonField<CallerBreakdown[]>({ schema: z.array(CallerBreakdownSchema), default: [] }),
    recentSamples: jsonField<SlowOpSample[]>({ schema: z.array(SlowOpSampleSchema), default: [] }),
    firstSeenAt:   dateField(),
    lastSeenAt:    dateField(),
  } satisfies FieldsRecord;

  export const SlowOpSchema = fieldsToZodObject(slowOpFields);
  export type SlowOp = z.infer<typeof SlowOpSchema>;
  ```
- `slowOpsResource` is unchanged (still `resourceDescriptor<SlowOp[]>("slow-ops",
  z.array(SlowOpSchema), [])`).
- Export `slowOpFields` from `core/index.ts` so the server can read it.

Factory imports are legal core→core cross-plugin imports:
`@plugins/fields/plugins/{uuid,text,int,float,json,date}/core` and
`fieldsToZodObject` / `FieldsRecord` from `@plugins/fields/core`.

### 3. Build the table via `defineEntity` in `server/internal/tables.ts`

The `defineEntity(` call **must stay in a drizzle schema-glob file** (`tables.ts`) —
enforced by the `table-defs-in-schema-glob` check (`defineEntity` is in its
`TABLE_FACTORIES`). Replace the raw `pgTable(...)` with:

```ts
import { uniqueIndex } from "drizzle-orm/pg-core";
import { defineEntity, defaultNow, defaultRandom } from "@plugins/infra/plugins/entities/server";
import { slowOpFields } from "../../core";

const slowOps = defineEntity("slow_ops", slowOpFields, {
  primaryKey: "id",
  columns: {
    id:            { default: defaultRandom() },
    count:         { default: 0 },
    totalMs:       { default: 0 },
    maxMs:         { default: 0 },
    lastMs:        { default: 0 },
    thresholdMs:   { default: 0 },
    callers:       { default: [] },
    recentSamples: { default: [] },
    firstSeenAt:   { default: defaultNow() },
    lastSeenAt:    { default: defaultNow() },
  },
  indexes: (t) => [
    uniqueIndex("slow_ops_kind_op_worktree_idx").on(t.operationKind, t.operation, t.worktree),
  ],
});

// drizzle-kit discovery convention (matches entity-extensions / attachments):
export const _slowOps = slowOps.table;
```

This reproduces the exact existing DDL: `notNull` is derived from each field's
non-optional schema; DB defaults are opt-in via `meta.columns`; `worktree`/
`operationKind`/`operation` get `notNull` with **no** default (matching the migration).
The `_slowOps` export name is kept so `resources.ts` / `record-slow-op.ts` / the server
barrel don't churn.

### 4. Delete the interim guard; retype `SlowOp` in the loader

In **`plugins/debug/plugins/slow-ops/server/internal/resources.ts`**:

- Delete the `Equal` / `Expect` / `_SlowOpRowMatchesWire` block (superseded — the row
  type and wire schema now derive from one record, so they're identical *by
  construction*; the comparison is vacuous).
- Loader body is unchanged: `db.select().from(_slowOps).orderBy(desc(_slowOps.totalMs))`.
  `_slowOps.$inferSelect ≡ SlowOp` by construction, so the existing `Promise<SlowOp[]>`
  return type still holds.

No changes needed to `record-slow-op.ts`, `handle-cluster.ts`, the cluster endpoint, or
the pane — they import `SlowOp` / `SlowOpSchema` / `CallerBreakdown` / `SlowOpSample`
from core, whose names and runtime shapes are preserved.

## Why `z.coerce.date()` is load-bearing (not `z.date()`)

The cluster fan-out (`plugins/debug/plugins/slow-ops/plugins/cluster/server/internal/handle-cluster.ts`)
runs a **raw node-postgres** query (not drizzle ORM) and calls `SlowOpSchema.parse(...)`
on rows where `first_seen_at` / `last_seen_at` arrive as `Date | string`. The current
schema uses `z.coerce.date()` for exactly this reason. The `dateField()` factory must
therefore emit `z.coerce.date()` so the wire schema keeps coercing. `z.coerce.date()`
still infers to `Date`, so `z.infer<schema>` and the `timestamptz` `$inferSelect`
column both remain `Date` — the select-type alignment holds.

`fieldsToZodObject` wraps each field with `.default(field.defaultValue)`; this only
applies when a key is `undefined`. Every real row (loader rows from drizzle, raw cluster
rows) carries all keys, so the defaults never fire — no behavior change. The
`Entity["schema"]` *type* is keyed by the raw `F[K]["schema"]` (pre-`.default()`), so
type inference is unaffected.
