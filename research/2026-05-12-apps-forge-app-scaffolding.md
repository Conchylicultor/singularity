# Forge App — Base Scaffolding

## Context

We're adding a new "Forge" sub-app under the `apps` umbrella plugin. It will eventually house plugin graph inspection, plugin publish workflows, and other meta-tooling for the Singularity plugin system. For now we only need the base scaffolding: shell registration (app switcher entry, sidebar, toolbar), an empty main pane — all mirroring the `file-explorer` app exactly.

## Structure

Forge lives at `plugins/apps/plugins/forge/` (same depth as `agent-manager`, `deploy`, `file-explorer`). All code goes in the mandatory `shell` sub-plugin:

```
plugins/apps/plugins/forge/
├── CLAUDE.md
├── package.json
└── plugins/
    └── shell/
        ├── CLAUDE.md
        ├── package.json
        └── web/
            ├── index.ts
            ├── slots.ts
            └── components/
                └── forge-layout.tsx
```

No `server/` needed at this stage. The plugin registry (`web/src/plugins.generated.ts`) is auto-updated by `./singularity build` — no manual registration.

### `plugins/apps/plugins/forge/plugins/shell/web/index.ts`
- Plugin id: `"forge-shell"`
- Contributes `Apps.App` with `id: "forge"`, `path: "/forge"`, `tooltip: "Forge"`, icon `MdHardware` (hammer — most forge-like icon in `react-icons/md`)
- Re-exports `Forge` from `./slots` (so consumers can import from the barrel)

```ts
import type { PluginDefinition } from "@core";
import { Apps } from "@plugins/apps/web";
import { MdHardware } from "react-icons/md";
import { ForgeLayout } from "./components/forge-layout";

export { Forge } from "./slots";

export default {
  id: "forge-shell",
  name: "Forge: Shell",
  description:
    "App shell for Forge. Registers the /forge app entry and defines Forge.Sidebar/Toolbar slots.",
  contributions: [
    Apps.App({
      id: "forge",
      icon: MdHardware,
      tooltip: "Forge",
      component: ForgeLayout,
      path: "/forge",
    }),
  ],
} satisfies PluginDefinition;
```

### `plugins/apps/plugins/forge/plugins/shell/web/slots.ts`
Mirror of `file-explorer`'s `slots.ts` with `"forge.*"` slot ids and `Forge` export name:

```ts
import { defineSlot } from "@core";
import type { ComponentType } from "react";
import { Reorder } from "@plugins/reorder/web";

export const Forge = {
  Sidebar: Reorder.area(
    defineSlot<{
      title: string;
      icon: ComponentType<{ className?: string }>;
      onClick?: () => void;
      component?: ComponentType;
      group?: string;
      labelExtra?: ComponentType;
      scroll?: boolean;
    }>("forge.sidebar", { docLabel: (p) => p.title }),
    { getGroup: (item) => item.group ?? null, getLabel: (item) => item.title },
  ),

  Toolbar: Reorder.area(
    defineSlot<{
      label?: string;
      icon?: ComponentType<{ className?: string }>;
      onClick?: () => void;
      component?: ComponentType;
      group?: string;
    }>("forge.toolbar", { docLabel: (p) => p.label }),
    {
      getGroup: (item) => item.group ?? null,
      getLabel: (item) => item.label ?? item.id,
    },
  ),
};
```

### `plugins/apps/plugins/forge/plugins/shell/web/components/forge-layout.tsx`
```tsx
import { AppShellLayout } from "@plugins/primitives/plugins/app-shell/web";
import { Forge } from "../slots";

export function ForgeLayout() {
  return (
    <AppShellLayout
      sidebarSlot={Forge.Sidebar}
      toolbarSlot={Forge.Toolbar}
    />
  );
}
```

### CLAUDE.md files
Both CLAUDE.md files get a one-line hand-written description + the standard autogenerated placeholder block (filled in by `./singularity build`).

## No Files to Modify

The plugin registry (`web/src/plugins.generated.ts`) and `web/src/plugins.ts` are fully auto-generated — `./singularity build` discovers `web/index.ts` by filesystem scan and adds the entry automatically.
