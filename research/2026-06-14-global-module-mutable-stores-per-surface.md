# Audit & fix: module-level mutable stores bleeding across surfaces; widen the lint rule repo-wide

## Context

When multiple surface instances are mounted at once (desktop multi-window, or
keep-alive tabs where every tab stays mounted with `display:none`), a
module-level mutable external store — the `let x; const listeners = new Set();
useSyncExternalStore(...)` pattern — is shared by *all* mounted instances. State
that is conceptually per-surface then bleeds between windows/tabs.

Commit `3e5850411` already migrated Sonata's per-surface singletons (cursor,
audio, key-mode, transport) onto the `primitives/scoped-store` primitive and
added a lint rule, **but the rule self-limits to `/plugins/apps/plugins/`** via an
in-rule filename guard, so the anti-pattern is uncaught everywhere else in
`plugins/`.

This plan: (1) widen the rule repo-wide, (2) fix the one genuine cross-surface
correctness bug, (3) make Miller column width/collapse per-surface, and
(4) explicitly mark the genuinely-global stores as intentional so the widened
rule passes.

### Audit result (every module `let` + `useSyncExternalStore` snapshot in `plugins/`)

The rule flags **only** a module-scope `let`/`var` whose name is read by a
`useSyncExternalStore` snapshot. Stores keyed by a `const Map` (e.g.
`use-window-geometry`, notes-visibility, today's Miller hooks) are *not* matched —
so widening the rule cannot catch the keyed-Map cross-surface class; those are
handled by hand where needed.

| File | `let` returned by snapshot? | Verdict |
|---|---|---|
| `plugins/tasks/plugins/task-draft-form/web/active-relate-context.ts` | yes (`_current`) | **BUG → fix** (focus-aware registry) |
| `plugins/reorder/web/internal/edit-mode-store.ts` | yes (`editMode`) | **Intentionally global** (single chrome pen button) → exempt |

Decisions (confirmed with user): **Improve relate** → focus-aware registry,
keep the single global button. **Edit mode** → keep global. **Miller
columns** → make per-surface.

## Step 1 — Widen + relocate the lint rule (the structural fix)

**Move** `plugins/apps/lint/no-module-mutable-store.ts` →
`plugins/primitives/plugins/scoped-store/lint/no-module-mutable-store.ts`.
The `scoped-store` primitive is the sanctioned replacement, so it should own the
rule that bans the anti-pattern. The AST logic is unchanged except:

- **Delete the scope guard** `if (!context.filename.includes("/plugins/apps/plugins/")) return {};` (currently ~line 120). The rule now runs on every `**/*.{ts,tsx}` (build-lint-config already registers contributed rules repo-wide as `error`).
- Point the `messages.moduleMutableStore` text at the fix: "Module-level mutable store `{{name}}` is shared across all mounted surfaces. Use `defineScopedStore` (`@plugins/primitives/plugins/scoped-store/web`) for per-surface state, or add `// eslint-disable-next-line scoped-store/no-module-mutable-store -- <why this is intentionally page-global>` if it is genuinely a single page-global signal."

**Create** `plugins/primitives/plugins/scoped-store/lint/index.ts`:
```ts
import noModuleMutableStore from "./no-module-mutable-store";
export default {
  name: "scoped-store",
  rules: { "no-module-mutable-store": noModuleMutableStore },
};
```

**Edit** `plugins/apps/lint/index.ts`: drop the `no-module-mutable-store` import
and its `rules` entry (keep `no-raw-history-nav` and its `ignores`).

> `lint.generated.ts` is regenerated by `./singularity build` — do not hand-edit.
> The new namespace is `scoped-store/no-module-mutable-store`.

**Escape hatch = inline disable with mandatory justification.** No central
`ignores` list — co-locating the "why it's global" with the code is the clean
design (matches the repo's `no-bare-catch`/`no-floating-promises` conventions).
The rule's own closure (`let state` inside `createStore`) is function-scoped, so
scoped-store does not flag itself.

> Known limitation (document in the rule's doc comment): the rule does not catch
> the *keyed-`const`-Map by a non-surface-unique key* variant (the Miller bug in
> step 3). Statically distinguishing a surface-unique key from a shared one is
> not feasible; those are caught by review, not lint.

## Step 2 — Fix `active-relate-context` (focus-aware registry)

Today: a global singleton `let _current` + an `_owner` symbol guard. The writer
(`active-relate-sync.tsx`, mounted per conversation pane) sets it; the reader
(the single global Improve button in `FloatingBar`/`ActionBar.Item`) reads it.
With ≥2 mounted conversation surfaces the last writer wins — wrong in keep-alive
tabs and multi-window.

The reader is **global chrome**; the correct value is **the focused surface's**
context. So the store must be keyed by `surfaceId` and the global reader must
read the *focused* surface and re-render when focus changes.

**2a. Make the focused-surface signal reactive.**
`plugins/primitives/plugins/shortcuts/web/internal/focused-surface.ts` currently
exposes get/set only. Add a listener set + notify-on-set + a hook:
```ts
const listeners = new Set<() => void>();
export function setFocusedSurfaceId(id) { if (id === focusedSurfaceId) return; focusedSurfaceId = id; for (const l of listeners) l(); }
// eslint-disable-next-line scoped-store/no-module-mutable-store -- page-global: exactly one surface is focused per page; read from the window keydown handler outside any React subtree.
export function useFocusedSurfaceId(): string | undefined {
  return useSyncExternalStore((cb) => { listeners.add(cb); return () => listeners.delete(cb); }, getFocusedSurfaceId, () => undefined);
}
```
Export `getFocusedSurfaceId` + `useFocusedSurfaceId` from the shortcuts web
barrel (`setFocusedSurfaceId` is already exported).

**2b. Rewrite `active-relate-context.ts`** as a `surfaceId`-keyed registry
(`const Map<string, ActiveRelateContext>` + `const listeners` Set — no module
`let`, so the widened rule passes cleanly):
```ts
const bySurface = new Map<string, ActiveRelateContext>();
export function setActiveRelateContext(surfaceId: string, ctx: ActiveRelateContext | null): void { /* set/delete bySurface[surfaceId]; notify */ }
export function useActiveRelateContext(): ActiveRelateContext | null {
  // re-render on EITHER a registry change OR a focus change:
  const focused = useFocusedSurfaceId();
  return useSyncExternalStore(subscribe, () => (focused ? bySurface.get(focused) ?? null : null));
}
```
The `surfaceId` key replaces the `_owner` symbol entirely (each surface is
naturally unique; a stale unmounting pane deletes only its own entry).

**2c. Update the writer** `active-relate-sync.tsx`: read
`useSurfaceTabId()` (`@plugins/primitives/plugins/surface-id/web`) and pass it as
the key instead of the `Symbol()` owner. Clear its own entry on unmount.

> task-draft-form already imports the shortcuts and surface-id primitives' siblings; both are legal primitive deps. The Improve button stays a single global instance.

## Step 3 — Make Miller column width/collapse per-surface

Mirror the existing precedent `use-column-maximize.ts`
(`WeakMap<PaneStore, …>`), so each Miller surface is independent and the
WeakMap auto-GCs on unmount. Both hooks already render inside the Miller tree
where `usePaneStore()` and `useSurfaceTabId()` are available; the external hook
signatures (`useColumnWidth(paneId, default)`, `useColumnCollapse(paneId)`,
`hasStoredWidth(paneId)`) stay the same — they read `usePaneStore()` internally.

- `use-column-collapse.ts`: replace the module `const collapseState`/`subscribers`
  with a `WeakMap<PaneStore, { state: Map<paneId,boolean>; subscribers: Set }>`
  (a `stateFor(store)` helper like maximize). Persist per-surface by keying
  sessionStorage `miller.collapse.${tabId}.${paneId}` (read `useSurfaceTabId()`),
  preserving within-tab persistence while isolating surfaces.
- `use-column-widths.ts`: same `WeakMap<PaneStore>` shape; persist localStorage
  `miller.width.${tabId}.${paneId}`. `hasStoredWidth` takes the tabId-scoped key.

> Update the miller `CLAUDE.md` "Per-pane width" / "Collapse" notes to say state is now per-surface.

## Step 4 — Mark the genuinely-global stores intentional

Add `// eslint-disable-next-line scoped-store/no-module-mutable-store -- <reason>`
immediately above the flagged `let` in each (the reasons already exist as prose
in two of them):

- `plugins/config_v2/web/internal/server-paths.ts` — "server boot fact; identical for every surface."
- `plugins/primitives/plugins/shortcuts/web/internal/dynamic-registry.ts` — "registry of all surfaces' shortcuts; dispatched per surfaceId at keydown."
- `plugins/reorder/web/internal/edit-mode-store.ts` — "single global chrome pen button toggles edit mode for all surfaces (confirmed product intent)."
- `plugins/primitives/plugins/shortcuts/web/internal/focused-surface.ts` — added in step 2a.
