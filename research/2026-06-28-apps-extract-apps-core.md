# Extract `apps-core/` from `apps/` — separate the switcher infra from the apps

## Context

`plugins/apps/` today mixes **two unrelated concerns** behind one plugin:

1. **The app-switcher infrastructure** — the load-bearing core: the `Apps` slot
   contract, the tab manager, the rail / tab-bar / surface layout, the
   placement registry, theme-scope helpers (`apps/web`, `apps/core`, `apps/lint`),
   plus two infra sub-plugins that are *not* apps:
   - `surface` — the generic per-tab dispatcher (docked / floating / solo
     placements + the wallpaper subtree).
   - `app-rail-framing` — the rail / hidden chrome variant region.
2. **The actual sub-apps** — 13 of them: `agent-manager`, `browser`, `debug`,
   `deploy`, `file-explorer`, `home`, `pages`, `prototypes`, `settings`,
   `sonata`, `story`, `studio`, `workflows`.

Reading `plugins/apps/` should answer "what apps exist?", but instead the reader
must wade through switcher internals. This change **relocates the switcher
infrastructure into a sibling umbrella `plugins/apps-core/`**, leaving
`plugins/apps/` as a **pure container of the 13 apps**.

Outcome: `apps/` = the list of apps; `apps-core/` = the machine that hosts them.

### Decisions (confirmed with user)

- **Move scope:** core barrel (`web`/`core`/`lint`) **and** both infra
  sub-plugins (`surface`, `app-rail-framing`) move to `apps-core/`.
- **Slot ids unchanged:** keep `"apps.app"`, `"apps.rail-framing"`,
  `"apps.surface"`, `"apps.tab-bar-actions"`, `"apps.surface.placement"`, and the
  exported `Apps` symbol. Slot ids are arbitrary strings with no enforced tie to
  the plugin id (`defineSlot(id)` takes the literal verbatim), so keeping them
  avoids churn and orphaning any persisted `config_v2` reorder directives.
- **Pure relocation only.** No behavior change, no internal decomposition of the
  switcher barrel in this task. A follow-up task is filed to split the barrel
  into `tabs` / `rail` / `tab-bar` / `tab-surface` / `layout` sub-plugins.

### Why this is safe / low-risk

- The switcher core (`apps/web`, `apps/core`) imports **zero** sub-plugins — the
  dependency graph is strictly one-directional (`surface`/`app-rail-framing`/each
  app `shell` → the barrel, never the reverse). Relocating the barrel cannot
  create a cycle.
- **App identity is untouched.** The 13 apps stay under `plugins/apps/`, so their
  plugin ids (→ `ActiveApp.id`, the `app:<id>` theme scope, surface app id, URL
  matching) **do not change**. Only the switcher-infra ids change
  (`apps` → `apps-core`, `apps.surface` → `apps-core.surface`,
  `apps.app-rail-framing` → `apps-core.app-rail-framing`, plus their descendants).
- Registries, docs, and `dependsOn` are autogenerated from the filesystem +
  import statements — they regenerate on `./singularity build`. No manual
  registry edits; `boundary-config.ts` discovers plugin paths dynamically (no
  hardcoded `apps` path).

## Target structure

```
plugins/apps-core/              # NEW umbrella (mixed: own barrel + sub-plugins)
  package.json                  # @singularity/plugin-apps-core, private, + description
  web/        (moved from apps/web)     # the switcher: Apps slots, AppsLayout Core.Root, rail, tab-bar, tabs, etc.
  core/       (moved from apps/core)    # RailFramingProps, Placement
  lint/       (moved from apps/lint)    # no-raw-history-nav (repo-wide rule)
  plugins/
    surface/            (moved from apps/plugins/surface)
    app-rail-framing/   (moved from apps/plugins/app-rail-framing)

plugins/apps/                   # now a PURE container
  package.json                  # @singularity/plugin-apps, private, + description (umbrella doc)
  plugins/
    agent-manager/ browser/ debug/ deploy/ file-explorer/ home/ pages/
    prototypes/ settings/ sonata/ story/ studio/ workflows/
```

## Implementation steps

### 0. File the follow-up task (first, via `add_task` MCP tool)
"Decompose the `apps-core` switcher barrel into sub-plugins (`tabs` = tab
store + manager + `navigate` + placement-registry singletons; `rail`; `tab-bar`;
`tab-surface`; `layout` Core.Root). The `Apps` slot contract + app-resolution /
theme-scope helpers remain the thin `apps-core` barrel. Land after the
apps→apps-core relocation."

### 1. Move directories (use `git mv` to preserve history)
- `plugins/apps/web/`   → `plugins/apps-core/web/`
- `plugins/apps/core/`  → `plugins/apps-core/core/`
- `plugins/apps/lint/`  → `plugins/apps-core/lint/`
- `plugins/apps/plugins/surface/`          → `plugins/apps-core/plugins/surface/`
- `plugins/apps/plugins/app-rail-framing/` → `plugins/apps-core/plugins/app-rail-framing/`

### 2. package.json
- **Create** `plugins/apps-core/package.json`:
  `{ "name": "@singularity/plugin-apps-core", "description": "App-switcher core: the Apps slot contract, tab manager, rail/tab-bar/surface layout, and the surface + app-rail-framing infra.", "private": true, "version": "0.0.1" }`
- **Edit** `plugins/apps/package.json` — keep `name: "@singularity/plugin-apps"`,
  add `"description": "Container for the installed apps (agent-manager, pages, settings, …). The switcher infrastructure lives in apps-core."` so the now-barrel-less umbrella documents itself.

### 3. Rewrite import specifiers (mechanical, repo-wide)
Apply these **exact prefix** replacements (verified non-overlapping — the 13 apps
use `@plugins/apps/plugins/<app>/…` which never matches `…/web` or `…/core`, and
the two infra prefixes are specific):

| From | To | ~count |
|---|---|---|
| `@plugins/apps/web` | `@plugins/apps-core/web` | 40 files |
| `@plugins/apps/core` | `@plugins/apps-core/core` | 3 files |
| `@plugins/apps/plugins/surface` | `@plugins/apps-core/plugins/surface` | surface subtree + consumers |
| `@plugins/apps/plugins/app-rail-framing` | `@plugins/apps-core/plugins/app-rail-framing` | app-rail-framing subtree |

> **Do NOT** blanket-replace `@plugins/apps/plugins/` — that would wrongly rewrite
> the 13 apps. Only the two infra prefixes above.

Notable external consumers of `@plugins/apps/web` (must update): `ui/theme-engine`
(+ `theme-customizer`, `theme-toggle`, `theme-injector`, `use-color-mode`),
`ui/variant-region`, `config_v2/settings/scope-tabs`,
`tasks/task-draft-form/use-capture-url-default`, `shell/toaster`,
`shell/global-action-bar`, `shell/notifications/bell-button`,
`debug/reports/report-detail`. The 3 `@plugins/apps/core` importers are all inside
the moving `app-rail-framing` subtree.

### 4. Lint plugin updates (in the moved `plugins/apps-core/lint/`)
- `index.ts`: `name: "apps"` → `name: "apps-core"`; update the `ignores` glob
  `plugins/apps/web/components/apps-layout.tsx` → `plugins/apps-core/web/components/apps-layout.tsx`
  (the `plugins/primitives/plugins/pane/web/pane.ts` glob is unchanged). No live
  code uses an inline `eslint-disable …apps/no-raw-history-nav`, so the namespace
  rename is safe.
- `no-raw-history-nav.ts`: update doc/message comment text referencing
  `@plugins/apps/web` → `@plugins/apps-core/web` (comments only; no logic).

### 5. Build + regenerate
`./singularity build` regenerates `web.generated.ts`, `server.generated.ts`,
`plugins-details.md`, `plugins-compact.md`, and the per-plugin `CLAUDE.md`
reference blocks (including new `apps-core/CLAUDE.md` and the slimmed
`apps/CLAUDE.md`). No manual edits to any generated file.

## Critical files

- `plugins/apps/web/index.ts` — the barrel (moves; exports unchanged).
- `plugins/apps/web/slots.ts` — `Apps` slots; **slot id strings kept as-is**.
- `plugins/apps/web/internal/placement-registry.ts`, `internal/use-tabs.tsx` —
  hold module-level singletons (the `surface ↔ apps` bridge + tab navigator);
  they move together inside the barrel, so the single-instance invariant holds.
- `plugins/apps/lint/index.ts`, `plugins/apps/lint/no-raw-history-nav.ts`.
- `plugins/apps/package.json`, new `plugins/apps-core/package.json`.
- Autogen (no manual edit): `plugins/framework/plugins/web-sdk/core/web.generated.ts`,
  `plugins/framework/plugins/server-core/core/server.generated.ts`,
  `docs/plugins-details.md`, `docs/plugins-compact.md`.

## Verification

1. `./singularity build` — must complete; server restarts cleanly.
2. `./singularity check` — focus on `plugins-registry-in-sync`,
   `plugins-doc-in-sync`, `plugin-boundaries`, `type-check`, `eslint`. All green.
3. Load `http://<worktree>.localhost:9000` via Playwright and verify the switcher

   - App rail renders all app icons; clicking switches apps (URL + focused tab).
   - Tab bar: open/close/reorder tabs, the `+` new-tab/window button.
   - Placements: docked (default), floating window (drag/resize, wallpaper
     backdrop + picker), solo fullscreen (Esc to exit).
   - Rail framing variants (rail / hidden) switch via theme customizer.
   - Theme scope: `app:<id>` still applies (per-app theming intact).
4. Confirm `apps-core` and `apps-core.{surface,app-rail-framing,…}` appear in the
   regenerated `web.generated.ts`; confirm `apps` no longer has a web entry but
   its 13 app children still do.

## Risks / notes

- **Switcher-infra plugin-id change** (`apps`→`apps-core`, `apps.surface`→
  `apps-core.surface`, etc.). Any persisted `config_v2` reorder directive keyed on
  the *old infra contribution ids* would orphan — low risk: these are
  single-contribution infra slots users don't reorder. The 13 apps' ids are
  unchanged, so app-level config/state is unaffected.
- **Slot ids deliberately retained** as `apps.*` — keeps the `Apps` contract and
  any persisted slot config stable despite the owner plugin's rename.
- The follow-up barrel-decomposition (step 0 task) is the natural next step toward
  the project's plugin-first ethos; this task intentionally stops at relocation to
  keep the diff a reviewable pure-move.
