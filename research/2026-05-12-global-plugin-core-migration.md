# Phase 3: Move `plugin-core/` ŌåÆ `plugins/framework/plugins/web-sdk/`

## Context

Part of the plugin DAG migration (see `research/2026-05-12-global-plugin-dag-migration.md` Phase 3). `plugin-core/` is the web plugin framework library (slots, commands, contributions, plugin loader). It's web-only, has no auto-registration (no default-export barrel), and is consumed purely via the `@core` alias (~209 imports across ~135 files). Moving it into `plugins/` makes it a regular plugin in the DAG.

`framework` is a new umbrella that will eventually contain server, central, web, etc. `web-sdk` is the sub-plugin name (hierarchy ID: `framework.web-sdk`).

## Target structure

```
plugins/framework/
  package.json                    ŌåÉ umbrella
  CLAUDE.md                       ŌåÉ auto-generated by build
  plugins/
    web-sdk/
      package.json
      CLAUDE.md                   ŌåÉ moved from plugin-core/CLAUDE.md
      core/                       ŌåÉ public API (zone: core)
        index.ts                  ŌåÉ current plugin-core/index.ts
        types.ts
        slots.ts
        commands.ts
        context.tsx
        loader.ts
      internal/                   ŌåÉ private (zone: internal)
        topo.ts
```

## Steps

### 1. Create directories and move files

```
mkdir -p plugins/framework/plugins/web-sdk/core
mkdir -p plugins/framework/plugins/web-sdk/internal
```

Move public files to `core/`:
- `plugin-core/{index.ts, types.ts, slots.ts, commands.ts, context.tsx, loader.ts}` ŌåÆ `plugins/framework/plugins/web-sdk/core/`

Move internal file:
- `plugin-core/topo.ts` ŌåÆ `plugins/framework/plugins/web-sdk/internal/`

Move CLAUDE.md:
- `plugin-core/CLAUDE.md` ŌåÆ `plugins/framework/plugins/web-sdk/CLAUDE.md`

### 2. Fix internal import in `context.tsx`

`context.tsx` imports `./topo` which now lives in `../internal/topo`:

```ts

// after

```

### 6. Update boundary checker (3 files)

**`boundary.config.ts`:**
- Remove: `zone("core", { match: "plugin-core" })`
- Remove: `allow("server -> core")`, `allow("central -> core")`, `allow("cli -> core")`, `allow("tooling -> core")` ŌĆö none of these actually import from `@core`
- Change: `allow("web -> core")` ŌåÆ `allow("web -> plugin.framework.web-sdk")` ŌĆö needed for `web/src/components/plugin-load-errors.tsx` and `web/src/__tests__/plugin-render.test.tsx` (the 2 non-excluded web files that import `@core`)
- Change: `allow("plugin.** -> core")` ŌåÆ remove (covered by existing `allow("plugin.** -> plugin.**")`)

**`tooling/src/boundaries/resolve.ts`** (lines 111-113):
```ts
// before
if (specifier === "@core" || specifier.startsWith("@core/")) {
  return { zone: "core", runtime: null };
}
// after
if (specifier === "@core" || specifier.startsWith("@core/")) {
  return { zone: "plugin.framework.web-sdk", runtime: "core" };
}
```

**`tooling/src/boundaries/check.ts`** (line 14):
Remove `"plugin-core"` from `SOURCE_ROOTS`. The new location is already covered by `"plugins"`.

### 7. Delete old directory

Remove `plugin-core/` entirely (including its `package.json`).

### 8. Run `bun install`

The workspace entry changed ŌĆö regenerate the lockfile.
