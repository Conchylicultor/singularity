# Home app — design

## Goal

A new **Home** app that is the entry point when users open Singularity:

- A responsive grid of cards, one per registered app.
- The last card is a **"+"** to create a new app (no-op for now).
- `/` is no longer a top-level app; it **redirects to `/home`**.
- The Agent Manager (previously the `/` catch-all) **moves to `/agents`**.

## Key architectural finding

The app at `/` did double duty: (1) the landing, and (2) the owner of the
`basePath: ""` namespace where the entire deep-link vocabulary of the main app
resolves (`/c/:id`, `/a/:id`, `/tasks/...`, `/build/...`, `agents`, `config`,
`review`, `screenshot`, `stats`, `code`, … — none carry their own `appPath`).

Crucially, **in-app Miller pane navigation auto-adapts to the namespace**:
`setChain → buildChainUrl → applyBasePath(currentBasePath + url)` on write, and
`handleLocationChange → stripBasePath` on read. So moving Agent Manager to
`/agents` makes every pane it hosts resolve under `/agents/...` automatically —
no per-link rewrites. The namespace is hardcoded **only** at the app
registration (`path: "/agents"`).

The *only* links that don't auto-adapt are a few **server-generated** absolute
`linkTo` strings (notifications): `/c/:id` (conversations), `/build/r/:id`,
`/tasks/t/:id` (crashes). Rather than hardcode `/agents` into those server
plugins (coupling + a conversations↔agent-manager import cycle), we add a
router-level rule: **any root-relative path that matches no app is canonicalized
into the declared fallback app's namespace**. This keeps the namespace at the
app level and leaves all server links untouched.

## Routing model (apps plugin — load-bearing)

- `Apps.App` slot gains `fallback?: boolean`. Agent Manager sets it.
- `appMatchesPath`: drop the `appPath === "/" → true` magic catch-all (no app is
  at `/` anymore); matching becomes uniform exact/prefix.
- `AppsLayout` redirect effect (keyed on pathname):
  - `pathname === "/"` → `replaceState("/home")`.
  - else no match + a `fallback` app exists → `replaceState(fallback.path + pathname)`
    (e.g. `/c/456` → `/agents/c/456`).
  - render the rail always; main area empty for the one frame before redirect.
- No infinite loops: `/`→`/home`(match), `/c/x`→`/agents/c/x`(match), matched paths untouched.

## File changes

Routing + remap (Agent A):
- `plugins/apps/web/slots.ts` — add `fallback?: boolean`.
- `plugins/apps/web/internal/use-active-app.ts` — uniform `appMatchesPath`; export `usePathname`.
- `plugins/apps/web/components/apps-layout.tsx` — redirect/canonicalize effect.
- `plugins/apps/plugins/agent-manager/plugins/shell/web/index.ts` — `path: "/agents"`, `fallback: true`.
- `plugins/apps/plugins/agent-manager/plugins/shell/web/components/agent-manager-layout.tsx` — logo link `/` → `/agents`.
- `plugins/welcome/web/panes.tsx` — `appPath: "/"` → `"/agents"`.

Home app (Agent B):
- `plugins/apps/plugins/home/` — empty namespace plugin (mirror pages).
- `plugins/apps/plugins/home/plugins/shell/` — registers `Apps.App({ id:"home", path:"/home", icon: MdHome, tooltip:"Home", component: HomeLayout })`; defines `Home.Section` render slot; `HomeLayout` = full-screen scroll container (top-right gutter for floating bar) rendering `<Home.Section.Render/>`.
- `plugins/apps/plugins/home/plugins/app-cards/` — contributes `Home.Section` = responsive card grid over `Apps.App.useContributions()`, filtered by `app.id !== useCurrentAppId()` (generic API, no contributor naming). Each card: tinted icon + name, navigates to `app.onClick ?? app.path`. Final cell = dashed **"+" New app** card (no-op).

## Tradeoffs / decisions

- **Fallback canonicalization** chosen over editing server `linkTo` strings: keeps
  the namespace single-sourced at the app level (user's explicit constraint) and
  avoids an import cycle. Cost: a small generic addition to the load-bearing apps router.
- **"+" card lives inside the app grid** (not its own sub-plugin) so it's
  guaranteed to be the literal last cell. Can be promoted to a `Home.Section`
  contribution later if extensibility is wanted.
- `/` → `/home` is a redirect (per request), so the canonical home URL is `/home`.
- Agent Manager keeps `hostsToolbar: true`; Home does not, so the floating bar
  (theme/health) is available on Home.
