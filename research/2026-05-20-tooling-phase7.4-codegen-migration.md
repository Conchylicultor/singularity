# Phase 7.4: Move codegen (docgen + registry-gen) to tooling sub-plugin

## Context

Part of the Phase 7 tooling-to-plugin migration (`research/2026-05-19-phase7-tooling-to-plugin.md`). The goal is to move `tooling/src/docgen.ts` and `tooling/src/plugin-registry-gen.ts` into a self-contained sub-plugin at `plugins/framework/plugins/tooling/plugins/codegen/`, following the same pattern as the existing `boundaries`, `lint`, and `guards` sub-plugins.

Phase 7.5 (checks) depends on this being done first, since checks files import from docgen and plugin-registry-gen.

## Import updates

### CLI consumers (switch from `@tooling/*` to `@plugins/...` barrel)

