# Remove ConfigStore, inline ConfigWatcher

## Context

The config_v2 `store` sub-plugin provides a `ConfigStore` interface (`read/write/watch/list`) backed by `JsoncConfigStore`. However, the registry — the sole external consumer — only calls `store.watch()`, and even discards the value it receives. All actual I/O goes through `jsoncConfigProxy` (a separate sync layer with `// @hash` awareness). The store's `read()`, `write()`, `list()` are dead code. The `ConfigStore` abstraction (designed as a seam for future cloud backends) was superseded by `ConfigProxy` when the hash-chain design landed.

Goal: delete the store sub-plugin entirely, extract the watcher logic into a thin module inside `config_v2/server/internal/`.

Extract the `@parcel/watcher` + debounce/ceiling/reconcile logic from `JsoncConfigStore`. Drop everything unused (`read`, `write`, `list`, `atomicWrite`, `walkJsonc`, `enqueueWrite`, seed read).

```ts
  // clear all timers, await subscription.unsubscribe()

export function watchFileChange(absPath: string, cb: () => void): Disposable
  // register cb for absPath, return { dispose() }
```

Key differences from `JsoncConfigStore`:
- Callback is `() => void` (no value) — the registry ignores it anyway
- No seed read — registry already reads via `jsoncConfigProxy` before registering watchers
- Eager init (`await parcel.subscribe`) instead of lazy `ensureSubscription` — no TOCTOU window
- Paths are absolute (registry already computes them as `join(CONFIG_DIR, ...)`)
- Same constants: `DEBOUNCE_MS=100`, `CEILING_MS=1000`, `RECONCILE_MS=30_000`

### 3. Update `registry.ts`

Replace:
```ts
import { getConfigStore } from "@plugins/config_v2/plugins/store/server";
...
const store = getConfigStore();
const userOverwritesStorePath = `${hierarchyPath}/${descriptor.name}.jsonc`;
const userOriginStorePath = `${hierarchyPath}/${descriptor.name}.origin.jsonc`;
disposables.push(store.watch(userOverwritesStorePath, onFileChange));
disposables.push(store.watch(userOriginStorePath, onFileChange));
```

With:
```ts
import { watchFileChange } from "./config-watcher";
...
disposables.push(watchFileChange(userOverwritesPath, onFileChange));
disposables.push(watchFileChange(userOriginPath, onFileChange));
```

The intermediate relative-path variables (`userOverwritesStorePath`, `userOriginStorePath`) are deleted — registry already has the absolute paths.

### 4. Update `config_v2/server/index.ts` lifecycle

```ts
import { initConfigWatcher, shutdownConfigWatcher } from "./internal/config-watcher";

async onReady() {
  await initConfigWatcher();  // must complete before registry
  initRegistry();
},
async onShutdown() {
  shutdownRegistry();          // dispose per-path callbacks first
  await shutdownConfigWatcher(); // then tear down subscription
},
```

### 5. Delete store sub-plugin

Remove entire `plugins/config_v2/plugins/store/` directory (7 files). After this, `plugins/config_v2/plugins/` contains only `fields/`.

`plugins.generated.ts` regenerates automatically on `./singularity build` — no manual edits needed.

### 6. Update `plugins/config_v2/CLAUDE.md`

Update the "Internal architecture" section to describe `ConfigWatcher` instead of `ConfigStore`. The autogenerated block will regenerate on build.
